## Direct Connect

### Características Principales

| Característica | Descripción |
|----------------|-------------|
| **Velocidad** | 1Gbps, 10Gbps, 100Gbps |
| **Latencia** | Consistente y predecible |
| **Costo** | Tarifa por hora + tarifa por tráfico |
| **Seguridad** | Tráfico no pasa por internet pública |
| **VLANs** | Soporta 802.1Q (trunking) |

### Tipos de Conexión

```bash
# Dedicated Connection (1-100 Gbps)
# Puerto dedicado físicamente al cliente
aws directconnect create-connection \
    --location "EqDC2" \
    --bandwidth 1Gbps \
    --connection-name "My-Dedicated-Connection"

# Hosted Connection (50Mbps-10Gbps)
# Proveído por AWS Partners
aws directconnect create-connection \
    --location "Partner/Equinix" \
    --bandwidth 100Mbps \
    --connection-name "My-Hosted-Connection"
```

### Virtual Interfaces

| Tipo VIF | Uso | Route Target |
|----------|-----|--------------|
| **Private VIF** | Acceder VPCs privadas | VPC CIDR |
| **Public VIF** | Servicios públicos AWS (S3, DynamoDB) | Public IP ranges |
| **Transit VIF** | Conectar a Transit Gateway | TGW |

## Site-to-Site VPN

### Características Principales

| Característica | Descripción |
|----------------|-------------|
| **Velocidad** | Limitado por internet (hasta 1.25 Gbps) |
| **Latencia** | Variable (internet) |
| **Costo** | Solo por hora de conexión |
| **Implementación** | Rápida (horas vs semanas) |
| **Throughput** | IPSec隧道 |

### Componentes

| Componente | Descripción |
|------------|-------------|
| **Customer Gateway (CGW)** | Dispositivo on-premises (hardware o software) |
| **Virtual Private Gateway (VPG)** | Endpoint AWS (attachado a VPC) |
| **VPN Connection** | El túnel IPSec entre CGW y VPG |

### Flujo de Configuración VPN

```bash
# 1. Crear Customer Gateway
aws ec2 create-customer-gateway \
    --type ipsec.1 \
    --public-ip 203.0.113.10 \
    --bgp-asn 65001 \
    --customer-gateway-name "My-CGW"

# 2. Crear Virtual Private Gateway
aws ec2 create-vpn-gateway \
    --type ipsec.1 \
    --amazon-side-asn 64512

# 3. Attach VPG a VPC
aws ec2 attach-vpn-gateway \
    --vpn-gateway-id vpg-xxxxxxxx \
    --vpc-id vpc-xxxxxxxx

# 4. Crear VPN Connection
aws ec2 create-vpn-connection \
    --customer-gateway-id cgw-xxxxxxxx \
    --vpn-gateway-id vpg-xxxxxxxx \
    --type ipsec.1 \
    --options "{\"StaticRoutesOnly\":false}"
```

### Configuración de Tunnel IPSec

```bash
# Opciones avanzadas de VPN con tunnel configs
aws ec2 create-vpn-connection \
    --customer-gateway-id cgw-xxxxxxxx \
    --vpn-gateway-id vpg-xxxxxxxx \
    --type ipsec.1 \
    --options '{
        "TunnelOptions": [
            {
                "TunnelInsideCidr": "169.254.0.0/30",
                "PreSharedKey": "MySecretKey1!",
                "Phase1LifetimeSeconds": 28800,
                "Phase2LifetimeSeconds": 3600,
                "RekeyFuzzPercentage": 100,
                "DPDTimeoutSeconds": 30
            },
            {
                "TunnelInsideCidr": "169.254.1.0/30",
                "PreSharedKey": "MySecretKey2!",
                "Phase1LifetimeSeconds": 28800,
                "Phase2LifetimeSeconds": 3600
            }
        ]
    }'
```

### Tunnel Inside CIDR

```bash
# Rango /30 reservado para túneles VPN
# No puede superponerse con ninguna red existente

# Tunnel 1: 169.254.0.0/30
#   - 169.254.0.1 = AWS end
#   - 169.254.0.2 = Customer end

# Tunnel 2: 169.254.1.0/30
#   - 169.254.1.1 = AWS end
#   - 169.254.1.2 = Customer end
```

## BGP (Border Gateway Protocol)

### Conceptos BGP

| Concepto | Descripción |
|----------|-------------|
| **ASN (Autonomous System Number)** | Número único que identifica tu red |
| **Peer** | Vecino BGP con el que intercambia rutas |
| **Prefix** | Red que se anuncia (ej: 10.0.0.0/16) |
| **MED** | Multi-Exit Discriminator (preferência de entrada) |
| **AS Path** | Camino de ASs que atraviesa la ruta |

### ASN en AWS

```bash
# ASN para Virtual Private Gateway
# AWS reserva: 101 - 65534 (16-bit), 4200000000 - 4294967294 (32-bit)
# AWS usa 64512 por defecto

aws ec2 create-vpn-gateway \
    --type ipsec.1 \
    --amazon-side-asn 64512  # ASN de AWS side

# Customer side ASN (tu router on-prem)
# Puede ser cualquier ASN público o privado válido
# ASN privado: 64512 - 65534
```

### Configuración BGP en Router (Ejemplo)

```bash
# Configuración en router Cisco
router bgp 65001
 ! Tu ASN
 neighbor 169.254.0.1 remote-as 64512
 ! AWS ASN
 neighbor 169.254.0.1 description AWS-VPN-Tunnel1
 !
 address-family ipv4
  network 172.16.0.0 mask 255.255.0.0
  ! Tu red on-premises
  neighbor 169.254.0.1 activate
  neighbor 169.254.0.1 prefix-list YOUR_PREFIXES in
  ! Solo recibir ciertas rutas de AWS
 exit-address-family
```

### Route Propagation

```bash
# Habilitar route propagation en route table
aws ec2 enable-vgw-route-propagation \
    --route-table-id rtb-xxxxxxxx \
    --gateway-id vgw-xxxxxxxx

# Verificar rutas propagadas
aws ec2 describe-route-tables \
    --route-table-id rtb-xxxxxxxx \
    --query 'RouteTables[0].Routes'
```

## Alta Disponibilidad

### Arquitectura Redundante

```
┌─────────────────────────────────────────────────────────────┐
│                    On-Premises                             │
│                                                              │
│  ┌─────────────┐                  ┌─────────────┐          │
│  │  Router 1   │                  │  Router 2   │          │
│  │  (Primary)  │                  │  (Backup)   │          │
│  └──────┬──────┘                  └──────┬──────┘          │
│         │                                │                  │
│    ┌────▼────┐                      ┌────▼────┐            │
│    │ BGP     │                      │ BGP     │            │
│    │ Session │                      │ Session │            │
│    └────┬────┘                      └────┬────┘            │
└─────────┼────────────────────────────────┼──────────────────┘
          │                                │
    ┌─────▼───────────────────────────────▼─────┐
    │         Direct Connect Location A          │
    │              (Primary)                      │
    └───────────────────────────────────────────┘
          │
    ┌─────▼───────────────────────────────▲─────┐
    │         Direct Connect Location B        │
    │              (Secondary)                  │
    └───────────────────────────────────────────┘
          │
    ┌─────▼─────────────────────────────────────▼─────┐
    │                   AWS Cloud                      │
    │                                                     │
    │  ┌─────────────────────────────────────────────┐  │
    │  │     Virtual Private Gateway (VPG)           │  │
    │  │     con BGP failover automático             │  │
    │  └─────────────────────────────────────────────┘  │
    └────────────────────────────────────────────────────┘
```

### VPN como Backup de Direct Connect

```bash
# Crear VPN Connection
aws ec2 create-vpn-connection \
    --customer-gateway-id cgw-xxxxxxxx \
    --vpn-gateway-id vgw-xxxxxxxx \
    --type ipsec.1 \
    --options '{"StaticRoutesOnly": false}'

# AWS automáticamente hace failover si Direct Connect cae
# Se necesita BGP para route propagation dinámico
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| VPN túnel DOWN | NAT o Firewall bloqueando 500/4500 UDP | Abrir puertos UDP 500 y 4500 |
| BGP no establece | ASN incorrecto o filtro de rutas | Verificar ASN coincide en ambos lados |
| Rutas no aparecen | Route propagation deshabilitado | Habilitar `enable-vgw-route-propagation` |
| Latencia alta en DX | Congestión en POP | Usar Direct Connect location más cercano |
| Direct Connect down | falla de hardware | Implementar segunda conexión en otro POP |

## Cuándo Usar Direct Connect vs VPN

| Criterio | Direct Connect | VPN |
|----------|----------------|-----|
| **Latencia** | Mínima y predecible | Variable (internet) |
| **Ancho de banda** | Hasta 100 Gbps | Hasta 1.25 Gbps por túnel |
| **Costo** | Hora + tráfico | Solo hora |
| **Implementación** | Semanas (requiere physical) | Horas |
| **SLA** | 99.9% | Basado en internet |
| **Ideal para** | Mission-critical, alta吞吐量 | Quick setup, menor требования |
