## Route Tables en AWS

### Tipos de Route Tables

```
┌──────────────────────────────────────────────────────────────┐
│                    Route Tables en AWS                        │
│                                                               │
│   Main Route Table                                             │
│   └─ Asociada a VPC por defecto                              │
│   └─ Todas las subnets usan esta si no tienen RT propia      │
│                                                               │
│   Custom Route Tables                                          │
│   └─ Creadas por usuario                                      │
│   └─ Pueden asociarse a subnets específicas                  │
│                                                               │
│   Edge Association                                            │
│   └─ Para VPC con Internet Gateway + VPN                     │
│   └─ Traffic destined to remote network via IGW              │
└──────────────────────────────────────────────────────────────┘
```

### Estructura de una Ruta

```bash
# Ruta típica
Destination:     10.0.0.0/16
Target:          local
State:           active
Propagated:      -

# Ruta a Internet (IGW)
Destination:     0.0.0.0/0
Target:          igw-xxxxxxxx

# Ruta a NAT Gateway
Destination:     0.0.0.0/0
Target:          nat-xxxxxxxx

# Ruta a VPC Peering
Destination:     10.1.0.0/16
Target:          pcx-xxxxxxxx

# Ruta a Transit Gateway
Destination:     10.0.0.0/8
Target:          tgw-xxxxxxxx
```

## Border Gateway Protocol (BGP)

### Conceptos Fundamentales

```
┌──────────────────────────────────────────────────────────────┐
│                       BGP Session                            │
│                                                               │
│   On-Premises Router                 AWS VPG/TGW            │
│                                                               │
│   ┌─────────────┐                  ┌─────────────┐          │
│   │   ASN:      │◄─────────────────►│   ASN:      │          │
│   │   65001     │     BGP Peer     │   64512     │          │
│   └─────────────┘                  └─────────────┘          │
│                                                               │
│   Anuncia: 172.16.0.0/16             Anuncia: 10.0.0.0/16     │
│   (tu red on-prem)                  (tu VPC)                  │
└──────────────────────────────────────────────────────────────┘
```

### Atributos BGP Importantes

| Atributo | Descripción | Uso |
|----------|-------------|-----|
| **AS_PATH** | Lista de ASs por donde pasa la ruta | Prevenir loops, preferir shortest path |
| **NEXT_HOP** | IP del next hop para llegar a la ruta | Routing decision |
| **LOCAL_PREF** | Preferencia para rutas salientes | Control outbound traffic |
| **MED (Multi-Exit Discriminator)** | Preferencia para rutas entrantes | Control inbound traffic |
| **ORIGIN** | Origen de la ruta (IGP, EGP, INCOMPLETE) | Preferência |

### Route Selection en AWS

```
Orden de prioridad cuando hay múltiples rutas al mismo destino:
1. Longest prefix match (más específica)
2. BGP Learned routes vs Static routes
3. Lowest metric (para VPN)
4. Static routes (menor priority que BGP)
```

## Static Routes vs Dynamic (BGP)

### Static Routes

```bash
# Agregar ruta estática
aws ec2 create-route \
    --route-table-id rtb-xxxxxxxx \
    --destination-cidr-block 10.1.0.0/16 \
    --vpc-peering-connection-id pcx-xxxxxxxx

# Reemplazar ruta
aws ec2 replace-route \
    --route-table-id rtb-xxxxxxxx \
    --destination-cidr-block 10.1.0.0/16 \
    --vpc-peering-connection-id pcx-xxxxxxxx

# Eliminar ruta
aws ec2 delete-route \
    --route-table-id rtb-xxxxxxxx \
    --destination-cidr-block 10.1.0.0/16
```

### Dynamic Routes (BGP)

```bash
# Habilitar route propagation (VPG)
aws ec2 enable-vgw-route-propagation \
    --route-table-id rtb-vpc-main \
    --gateway-id vgw-xxxxxxxx

# Deshabilitar
aws ec2 disable-vgw-route-propagation \
    --route-table-id rtb-vpc-main \
    --gateway-id vgw-xxxxxxxx

# Las rutas BGP aparecen como "Propagated" en el route table
```

## Route Priority

### Firewall/VPN/NAT Priority

```
Tráfico destined to 0.0.0.0/0:

1. Route Table local routes (VPC CIDR) - PRIORITY 1
2. NAT Gateway (más específica que 0.0.0.0/0) - PRIORITY 2
3. Egress Internet Gateway - PRIORITY 3
4. VPC Peering - PRIORITY 4
5. Transit Gateway - PRIORITY 5
6. VPN Connection - PRIORITY 6
```

### Longest Prefix Match

```bash
# Si tienes estas rutas:
# 10.0.0.0/8     → IGW
# 10.1.0.0/16     → NAT
# 10.1.1.0/24     → TGW

# Traffic a 10.1.1.100 usa /24 (TGW) - más específica
# Traffic a 10.2.0.100 usa /8 (IGW)
```

## Routing para VPC Peering

### Configuración Completa

```bash
# 1. Crear VPC Peering
aws ec2 create-vpc-peering-connection \
    --vpc-id vpc-a \
    --peer-vpc-id vpc-b

# 2. Aceptar en cuenta другая (si cross-account)
aws ec2 accept-vpc-peering-connection \
    --vpc-peering-connection-id pcx-xxxxxxxx

# 3. Route en VPC A (requester)
aws ec2 create-route \
    --route-table-id rtb-vpc-a-private \
    --destination-cidr-block 10.1.0.0/16 \
    --vpc-peering-connection-id pcx-xxxxxxxx

# 4. Route en VPC B (accepter)
aws ec2 create-route \
    --route-table-id rtb-vpc-b-private \
    --destination-cidr-block 10.0.0.0/16 \
    --vpc-peering-connection-id pcx-xxxxxxxx

# 5. Security Groups - permitir tráfico
aws ec2 authorize-security-group-ingress \
    --group-id sg-vpc-b \
    --source-group sg-vpc-a \
    --protocol all
```

### VPC Peering - Limitaciones

```
⚠️ No Transitivo!
   VPC A <---> VPC B (peering)
   VPC B <---> VPC C (peering)

   ❌ A NO puede alcanzar C via B
   ✅ Necesitas A <---> C peering directo

⚠️ No es Regional
   Peering entre regiones usa IPsec VPN (no direct)
```

## Routing para NAT Gateway

### Arquitectura

```
┌──────────────────────────────────────────────────────────────┐
│                    NAT Gateway Routing                        │
│                                                               │
│   Public Subnet:                                              │
│   ┌──────────────────────────────────────────────────────┐   │
│   │  Route Table:                                         │   │
│   │  10.0.0.0/16 → local                                 │   │
│   │  0.0.0.0/0    → igw-xxxxxxxx                         │   │
│   │                                                        │   │
│   │  NAT Gateway                                          │   │
│   │  └─ Elastic IP attached                               │   │
│   └──────────────────────────────────────────────────────┘   │
│                                                               │
│   Private Subnet:                                             │
│   ┌──────────────────────────────────────────────────────┐   │
│   │  Route Table:                                         │   │
│   │  10.0.0.0/16 → local                                 │   │
│   │  0.0.0.0/0    → nat-xxxxxxxx                         │   │
│   └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

### NAT vs IGW en Public Subnet

| Componente | Ubicación | Routing |
|------------|-----------|---------|
| **IGW** | Auto-creado,Attached a VPC | Traffic to 0.0.0.0/0 |
| **NAT Gateway** | Public Subnet | Traffic from private subnets to internet |
| **Internet Gateway** | VPC-level | Enables internet access (both directions) |

## Routing para VPN

### Virtual Private Gateway (VPG)

```bash
# Crear VPG
aws ec2 create-vpn-gateway \
    --type ipsec.1 \
    --amazon-side-asn 64512

# Attach a VPC
aws ec2 attach-vpn-gateway \
    --vpn-gateway-id vgw-xxxxxxxx \
    --vpc-id vpc-xxxxxxxx

# Habilitar route propagation
aws ec2 enable-vgw-route-propagation \
    --route-table-id rtb-main \
    --gateway-id vgw-xxxxxxxx
```

### Customer Gateway (CGW)

```bash
# Crear CGW
aws ec2 create-customer-gateway \
    --type ipsec.1 \
    --public-ip 203.0.113.10 \
    --bgp-asn 65001 \
    --tag-specifications "ResourceType=customer-gateway,Tags=[{Key=Name,Value=My-CGW}]"

# Crear VPN Connection
aws ec2 create-vpn-connection \
    --customer-gateway-id cgw-xxxxxxxx \
    --vpn-gateway-id vgw-xxxxxxxx \
    --type ipsec.1 \
    --options '{"StaticRoutesOnly":false}'
```

## Troubleshooting de Routing

| Problema | Diagnóstico | Solución |
|----------|------------|----------|
| Ping no responde | `Traceroute` para ver dónde muere | Verificar routes en cada hop |
| VPC unreachable | Ver route table de cada subnet | Agregar ruta faltante |
| VPN no conecta | `describe-vpn-connections` | Verificar CGW IP, BGP ASN |
| TGW traffic drops | Ver route tables de TGW | Verificar propagation/association |
| NAT no funciona | IGW en public RT, NAT en private RT | Mover NAT a public subnet |

### Comandos de Diagnóstico

```bash
# Ver routes de una subnet
aws ec2 describe-route-tables \
    --filters Name=association.subnet-id,Values=subnet-xxxxxxxx \
    --query 'RouteTables[0].Routes'

# Ver si route está propagada
aws ec2 describe-vpn-connections \
    --vpn-connection-id vpn-xxxxxxxx \
    --query 'VpnConnections[0].Routes'

# Ver Traffic Mirror source
aws ec2 describe-traffic-mirror-sources

# Test routing (desde EC2)
traceroute -I 10.1.0.10
```
