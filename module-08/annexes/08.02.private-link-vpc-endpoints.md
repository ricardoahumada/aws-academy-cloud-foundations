## Conceptos Fundamentales

### ¿Qué es PrivateLink?

```
┌──────────────────────────────────────────────────────────────┐
│                    Conectividad Privada                     │
│                                                               │
│   SIN PrivateLink:           CON PrivateLink:               │
│   Internet ────► NAT ────►    └──► PrivateLink ────►       │
│            Gateway                (ENI en tu VPC)           │
│                                                               │
│   Tráfico pasa por              Tráfico permanece            │
│   internet pública              dentro de AWS               │
└──────────────────────────────────────────────────────────────┘
```

### Componentes

| Componente | Descripción |
|------------|-------------|
| **VPC Endpoint** | Entry point para el servicio (ENI) |
| **Endpoint Service** | Tu servicio expuesto via PrivateLink |
| **Private DNS** | Resolución DNS automática |

```
│   │  Gateway Endpoint │      │ Interface Endpoint│        │
```

### Gateway Endpoint (S3 y DynamoDB)

```bash
# Crear Gateway Endpoint para S3
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.s3 \
    --vpc-endpoint-type Gateway \
    --route-table-ids rtb-xxxxxxxx

# Crear Gateway Endpoint para DynamoDB
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.dynamodb \
    --vpc-endpoint-type Gateway \
    --route-table-ids rtb-xxxxxxxx
```

### Interface Endpoint (Otros Servicios)

```bash
# Crear Interface Endpoint para SNS
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.sns \
    --vpc-endpoint-type Interface \
    --subnet-ids subnet-xxxxxxxx subnet-yyyyyyyy \
    --security-group-ids sg-xxxxxxxx

# Crear Interface Endpoint para Secrets Manager

    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.secretsmanager \
    --vpc-endpoint-type Interface \
```

### Prefix List en Route Table

```bash
# Ver prefix lists (S3 y DynamoDB)
aws ec2 describe-managed-prefix-lists \
    --filters Name=owner-id,Values=amazon

# Output incluye:
# - pl-xxxxxxxx (S3) - prefix list ID para S3
# - pl-yyyyyyyy (DynamoDB) - prefix list ID para DynamoDB

# La ruta usa el prefix list, no CIDR
# AWS manage las IPs automáticamente
```

## Interface Endpoint - Detalle

### Arquitectura

```
┌──────────────────────────────────────────────────────────────┐
│                         VPC                                  │
│                                                               │
│   ┌──────────────────────────────────────────────────────┐   │
│   │  Interface Endpoint (ENI)                            │   │
│   │                                                       │   │
│   │   ┌──────────────────────────────────────────────┐   │   │
│   │   │  Security Group: 443 (HTTPS)                  │   │   │
│   │   │  IP: 10.0.1.56 (del subnet privado)          │   │   │
│   │   │  DNS: vpce-xxx.sns.us-east-1.vpce.amazonaws.com│   │   │
│   │   └──────────────────────────────────────────────┘   │   │
│   │                                                       │   │
│   └──────────────────────────────────────────────────────┘   │
│                                                               │
│   ┌──────────────────────────────────────────────────────┐   │
│   │  EC2 Instance                                        │   │
│   │   # curl https://vpce-xxx.sns.us-east-1.vpce.amazonaws.com│   │
│   │                                                       │   │
│   └──────────────────────────────────────────────────────┘   │
└──────────────────────────────────────────────────────────────┘
```

### Private DNS

```bash
# Enable private DNS cuando creas el endpoint
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.sns \
    --vpc-endpoint-type Interface \
    --subnet-ids subnet-xxxxxxxx \
    --security-group-ids sg-xxxxxxxx \
    --private-dns-enabled

# Resultado:
# - Tu instancia puede acessar sns.us-east-1.amazonaws.com
# - DNS resuelve a IP privada del endpoint
# - No requiere cambiar código
```

### Endpoint vs Público Internet

| Aspecto | Endpoint Privado | Internet Público |
|---------|-----------------|-----------------|
| **IP Source** | Private ENI | Internet Gateway |
| **DNS** | vpce-xxx.vpce.amazonaws.com | amazonaws.com |
| **Firewall** | Security Group | N/A (requiere SG también) |
| **Ruta** | Directa | NAT/IGW |
| **Costo** | Por hora + tráfico | Normal tráfico |

## Endpoint Services (PrivateLink para SaaS)

### Crear Servicio Propio

```bash
# Crear Network Load Balancer (pre-requisito)
aws elbv2 create-load-balancer \
    --name my-nlb \
    --type network \
    --subnets subnet-xxxxxxxx subnet-yyyyyyyy

# Crear Endpoint Service
aws ec2 create-vpc-endpoint-service-configuration \
    --service-name com.amazonaws.vpce.us-east-1.my-service \
    --network-load-balancer-arns arn:aws:elasticloadbalancing:us-east-1:123456789:loadbalancer/net/my-nlb/xxxxxxxx \
    --acceptance-required

# Require acceptance para controlar quién puede acceder
```

### Permisos de Acceso

```bash
# Permitir cuenta específica
aws ec2 modify-vpc-endpoint-service-permissions \
    --service-id vpce-svc-xxxxxxxx \
    --add-allowed-principals arn:aws:iam::123456789012:root

# Allow IAM users/roles específicos
aws ec2 modify-vpc-endpoint-service-permissions \
    --service-id vpce-svc-xxxxxxxx \
    --add-allowed-principals arn:aws:iam::123456789012:user/my-user

# Listar permisos
aws ec2 describe-vpc-endpoint-service-permissions \
    --service-id vpce-svc-xxxxxxxx
```

### Consumer - Conectar al Servicio

```bash
# En la cuenta del consumidor
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-yyyyyyyy \
    --service-name com.amazonaws.vpce.us-east-1.my-service \
    --vpc-endpoint-type Interface \
    --subnet-ids subnet-yyyyyyyy \
    --security-group-ids sg-yyyyyyyy

# Verificar conexión
aws ec2 describe-vpc-endpoint-connections \
    --service-id vpce-svc-xxxxxxxx
```

## Casos de Uso

### 1. Acceso Privado a S3 desde Lambda

```bash
# Crear Gateway Endpoint para Lambda (sin internet)
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.s3 \
    --vpc-endpoint-type Gateway \
    --route-table-ids rtb-lambda

# Lambda en VPC privada ahora puede acceder S3
```

### 2. Private API con API Gateway

```bash
# Crear Interface Endpoint para API Gateway
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-xxxxxxxx \
    --service-name com.amazonaws.us-east-1.execute-api \
    --vpc-endpoint-type Interface \
    --subnet-ids subnet-xxxxxxxx \
    --security-group-ids sg-xxxxxxxx

# Tu API solo es accessible desde VPC
```

### 3. Cross-Account Database Access

```
Account A (Producer):
  └── RDS Instance
  └── Endpoint Service (NLB + PrivateLink)

Account B (Consumer):
  └── VPC
  └── Interface Endpoint → Service A
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| Timeout en access S3 | Ruta no configurada | Verificar route table con prefix list |
| DNS no resuelve | Private DNS disabled | Habilitar `--private-dns-enabled` |
| Connection refused | Security group no permite 443 | Abrir puerto 443 en SG |
| Access denied | Policy del endpoint | Verificar VPC endpoint policy |
| Latencia alta | Interface endpoint en una AZ | Crear en múltiples subnets |
