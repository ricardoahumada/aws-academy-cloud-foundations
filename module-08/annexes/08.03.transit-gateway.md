# Transit Gateway - Hub Central de Networking

> **Tiempo de lectura:** ~10 min | **Referencia:** [AWS Transit Gateway](https://docs.aws.amazon.com/vpc/latest/tgw/what-is-transit-gateway.html)

---

## Conceptos Fundamentales

### El Problema: Topología de Malla

```
SIN Transit Gateway (VPC Peering):
┌─────────┐         ┌─────────┐
│  VPC A  │◄───────►│  VPC B  │
└────┬────┘         └────┬────┘
     │                    │
     ◄────────┬───────────►
              │
     ┌────────┴────┐
     │             │
     ▼             ▼
┌─────────┐   ┌─────────┐
│  VPC C  │◄──│  VPC D  │
└─────────┘   └─────────┘

Con 4 VPCs = 6 conexiones Peering (N*(N-1)/2)
```

### La Solución: Transit Gateway

```
CON Transit Gateway:
┌─────────┐
│  VPC A  │──────┐
└─────────┘      │
┌─────────┐      │       ┌───────────────────┐
│  VPC B  │──────┼──────►│                   │
└─────────┘      │       │  Transit Gateway  │
                 ├───────►│     (Hub)         │
┌─────────┐      │       │                   │
│  VPC C  │──────┤       │     Routes:       │
└─────────┘      │       │  - VPC A ←local  │
                  │       │  - VPC B ←attach │
┌─────────┐      │       │  - VPC C ←attach │
│  VPC D  │──────┘       │  - VPN   ←attach │
└─────────┘              └───────────────────┘

Con 4 VPCs + VPN = 5 attachments
Simplifica de O(N²) a O(N) conexiones
```

## Componentes de Transit Gateway

| Componente | Descripción |
|------------|-------------|
| **Transit Gateway** | Hub central que conecta VPCs y VPN |
| **Transit Gateway Attachment** | Conexión al TGW (VPC, VPN, Direct Connect) |
| **Transit Gateway Route Table** | Rutas que controlan el tráfico |
| **Transit Gateway ARN** | Identificador único (resource) |
| **Association** | Asocia un attachment a una route table |
| **Propagation** | Auto-aprende rutas de un attachment |

## Transit Gateway Route Tables

### Estructura de Rutas

```bash
# Route Table del Transit Gateway
Destination              Target
----------------------------------
10.0.0.0/16             local (VPC A)
10.1.0.0/16             tgw-attach-vpc-b (VPC B)
10.2.0.0/16             tgw-attach-vpc-c (VPC C)
172.16.0.0/16           tgw-attach-vpn (VPN)
```

### Crear Transit Gateway

```bash
# Crear TGW básico
aws ec2 create-transit-gateway \
    --description "Main Transit Gateway" \
    --amazon-side-asn 64512 \
    --options "AutoAcceptSharedAttachments=enable,DefaultRouteTableAssociation=enable,DefaultRouteTablePropagation=enable"

# Output:
# TransitGatewayId: tgw-xxxxxxxx
```

### Tipos de Route Tables

| Tipo | Descripción | Uso |
|------|-------------|-----|
| **Association** | Define qué attachment usa qué route table | mapping attachment → route table |
| **Propagation** | De dónde aprende rutas un attachment | auto-learn routes |

### Route Tables Múltiples

```bash
# Crear Transit Gateway con múltiples route tables
aws ec2 create-transit-gateway \
    --description "TGW with custom RTs" \
    --amazon-side-asn 64512 \
    --options "AutoAcceptSharedAttachments=disable"

# Crear route tables
aws ec2 create-transit-gateway-route-table \
    --transit-gateway-id tgw-xxxxxxxx \
    --tag-specifications "ResourceType=transit-gateway-route-table,Tags=[{Key=Name,Value=Production-RT}]"

aws ec2 create-transit-gateway-route-table \
    --transit-gateway-id tgw-xxxxxxxx \
    --tag-specifications "ResourceType=transit-gateway-route-table,Tags=[{Key=Name,Value=Development-RT}]"
```

## Attachments

### VPC Attachment

```bash
# Crear attachment para VPC
aws ec2 create-transit-gateway-vpc-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --vpc-id vpc-xxxxxxxx \
    --subnet-ids subnet-xxxxxxxx subnet-yyyyyyyy

# Output:
# TransitGatewayAttachmentId: tgw-attach-xxxxxxxx
```

### VPN Attachment (Site-to-Site)

```bash
# Primero crear Virtual Private Gateway
aws ec2 create-vpn-gateway \
    --type ipsec.1 \
    --amazon-side-asn 65001

aws ec2 attach-vpn-gateway \
    --vpn-gateway-id vgw-xxxxxxxx \
    --vpc-id vpc-xxxxxxxx

# Crear Transit Gateway Attachment para VPN
aws ec2 create-transit-gateway-vpc-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --vpn-gateway-id vgw-xxxxxxxx \
    --subnet-ids subnet-xxxxxxxx
```

### Transit VIF (Direct Connect)

```bash
# Crear Direct Connect Gateway primero

    --direct-connect-gateway-name my-dx-gateway \
    --amazon-side-asn 64513

# Crear Transit Gateway Attachment
aws ec2 create-transit-gateway-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --direct-connect-gateway-id dxg-xxxxxxxx \
    --attachment-type transit
```

## Associations y Propagations

### Association (Attachment → Route Table)

```bash
# Asociar VPC A a Production route table
aws ec2 associate-transit-gateway-route-table \
    --transit-gateway-route-table-id tgw-rtb-xxxxxxxx \
    --transit-gateway-attachment-id tgw-attach-vpc-a \
    --transit-gateway-route-table-id tgw-rtb-prod

# Asociar VPC B a Production route table
aws ec2 associate-transit-gateway-route-table \
    --transit-gateway-route-table-id tgw-rtb-xxxxxxxx \
    --transit-gateway-attachment-id tgw-attach-vpc-b \
    --transit-gateway-route-table-id tgw-rtb-prod
```

### Propagation (Attachment aprende rutas)

```bash
# Habilitar propagation para que VPC B aprenda rutas de VPN
aws ec2 enable-transit-gateway-route-table-propagation \
    --transit-gateway-route-table-id tgw-rtb-prod \
    --transit-gateway-attachment-id tgw-attach-vpn \
    --transit-gateway-route-table-id tgw-rtb-prod

# Disable propagation
aws ec2 disable-transit-gateway-route-table-propagation \
    --transit-gateway-route-table-id tgw-rtb-prod \
    --transit-gateway-attachment-id tgw-attach-vpn
```

### Configuración Completa

```bash
# 1. Crear TGW
aws ec2 create-transit-gateway \
    --description "Main TGW"

# 2. Crear attachments
aws ec2 create-transit-gateway-vpc-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --vpc-id vpc-a \
    --subnet-ids subnet-a1,subnet-a2

aws ec2 create-transit-gateway-vpc-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --vpc-id vpc-b \
    --subnet-ids subnet-b1,subnet-b2

# 3. Habilitar propagation para todas
aws ec2 enable-transit-gateway-route-table-propagation \
    --transit-gateway-route-table-id tgw-rtb-default \
    --transit-gateway-attachment-id tgw-attach-vpc-a

aws ec2 enable-transit-gateway-route-table-propagation \
    --transit-gateway-route-table-id tgw-rtb-default \
    --transit-gateway-attachment-id tgw-attach-vpc-b

# 4. VPC Route Tables deben tener ruta al TGW
aws ec2 create-route \
    --route-table-id rtb-vpc-a-private \
    --destination-cidr-block 10.0.0.0/8 \
    --transit-gateway-id tgw-xxxxxxxx
```

## Route Server (EC2 Transit Gateway Route Server)

### Concepto

```
┌──────────────────────────────────────────────────────────────┐
│                  Transit Gateway Route Server                 │
│                                                               │
│  VPC A ──► TGW RT ──► VPC B                                  │
│  VPC A ──► TGW RT ──► VPC C                                  │
│  VPC B ──► TGW RT ──► VPC A (via VPC A's propagated route)  │
│  VPC B ──► TGW RT ──► VPC C                                  │
│                                                               │
│  Route Server:                                                │
│  - aprende rutas estáticas de attachments                    │
│  - Propaga rutas aprendidas a TODOS los attachments          │
│  - Spoke-to-spoke via TGW (sin VPN adicional)              │
└──────────────────────────────────────────────────────────────┘
```

### Limitación Importante

```
⚠️ IMPORTANTE: Transit Gateway Route Server NO es BGP route server

El Route Server de TGW:
✅ Aprende rutas estáticas configuradas en attachments
✅ Aprende rutas propagadas de VPN
✅ Propaga a todos los attachments
❌ NO aprende rutas dinámicas de VPC (no BGP con VPC)

Para VPC-to-VPC transitive routing:
- Las VPCs deben tener rutas pointing to TGW
- TGW aprende via propagation
- VPCs aprenden via static route pointing to TGW
```

## Transit Gateway with RAM (Resource Access Manager)

### Compartir TGW entre Cuentas

```bash
# En cuenta Owner (cuenta 111111111111)
aws ram create-resource-share \
    --name "Shared-TGW" \
    --resource-arns arn:aws:ec2:us-east-1:111111111111:transit-gateway/tgw-xxxxxxxx \
    --principals 222222222222

# En cuenta Consumer (cuenta 222222222222)
aws ec2 accept-transit-gateway-vpc-attachment \
    --transit-gateway-attachment-id tgw-attach-xxxxxxxx

# Ahora ambas cuentas pueden usar el mismo TGW
```

### Arquitectura Multi-Cuenta

```
Account A (Owner):
  └── Transit Gateway (tgw-xxxxxxxx)
  
Account B (Shared):
  └── Can create: VPC Attachment → tgw-xxxxxxxx
  └── Cannot modify TGW route tables

Account C (Shared):
  └── Can create: VPC Attachment → tgw-xxxxxxxx
```

## Blackhole Routes

```bash
# Crear blackhole route (drop traffic)
aws ec2 create-transit-gateway-route \
    --transit-gateway-route-table-id tgw-rtb-xxxxxxxx \
    --destination-cidr-block 10.100.0.0/24 \
    --transit-gateway-attachment-id tgw-attach-0aa65c9b01example

# Verificar
aws ec2 search-transit-gateway-routes \
    --transit-gateway-route-table-id tgw-rtb-xxxxxxxx \
    --filters Name=state,Values=active \
    --query 'Route[{RouteTableId} == "tgw-rtb-xxxxxxxx"].{DestinationCidrBlock: destination-cidr-block,Type: type}'
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| VPC no可达 other VPC | Falta route en VPC RT | Agregar `10.0.0.0/8 → tgw-id` en RT |
| TGW attachment stuck | `associate` requiere que RT esté asociado | Verificar associations |
| Rutas no aparecen | Propagation no habilitada | `enable-transit-gateway-route-table-propagation` |
| Cross-account attachment stuck | Acceptance requerido | Accept en cuenta owner |
| Traffic dropping | Blackhole route conflicts | Verificar route table priorities |

## Peering Transit Gateway (TGW Peering)

```bash
# Crear peering attachment (cross-region o cross-account)
aws ec2 create-transit-gateway-peering-attachment \
    --transit-gateway-id tgw-xxxxxxxx \
    --peer-transit-gateway-id tgw-yyyyyyyy \
    --peer-account-id 123456789012 \
    --peer-region us-west-2

# Aceptar en la otra cuenta
aws ec2 accept-transit-gateway-peering-attachment \
    --transit-gateway-attachment-id tgw-attach-peering-xxxxxxxx

# Ahora el tráfico puede cruzar entre TGWs
```
