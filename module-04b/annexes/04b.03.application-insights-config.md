# Anexo 03b.03: Application Insights Configuration

## Conceptos Fundamentales

CloudWatch Application Insights automatiza el monitoreo de aplicaciones, detectando problemas comunes sin configuración manual de umbrales. Usa machine learning para identificar anomalías.

### Componentes Detectados Automáticamente

| Tipo de Recurso | Monitoreo Automático |
|-----------------|---------------------|
| EC2 | CPU, memoria, disco, red, procesos |
| Lambda | Errores, duración, invocations |
| ALB | Request count, latency, errors |
| RDS | Connections, queries, buffer cache |
| ASG | instances, health status |
| SQS | Messages visible, age of oldest message |
| ECS | CPU, memory utilization |

## Configuración Paso a Paso

### 1. Habilitar Application Insights

#### Via Consola

1. Ir a CloudWatch > Application Insights
2. Click "Create application"
3. Seleccionar resource group
4. Confirmar componentes detectados
5. Click "Create"

#### Via AWS CLI

```bash
aws application-insights create-application \
    --resource-group-name "MiResourceGroup" \
    --ops-center-enabled \
    --tags '[{"Key": "Environment", "Value": "Production"}]'
```

---

### 2. Configurar Monitoreo de Componentes

#### Ver componentes detectados

```bash
aws application-insights list-components \
    --resource-group-name "MiResourceGroup" \
    --output table
```

#### Habilitar monitoreo específico

```bash
aws application-insights create-component \
    --resource-group-name "MiResourceGroup" \
    --component-name "mi-lambda-function" \
    --component-type AWS_Lambda
```

---

### 3. Configurar Monitoreo de Logs

#### Patrones de errores a monitorear

```bash
aws application-insights create-log-pattern \
    --resource-group-name "MiResourceGroup" \
    --pattern-set-name "CommonErrors" \
    --pattern-name "JavaException" \
    --pattern "Exception.*at.*" \
    --rank 1
```

#### Patrones predefinidos comunes

| Patrón | Regex | Severidad |
|--------|-------|-----------|
| Exception | `Exception|Error|Caught` | High |
| Timeout | `timed out|Timeout` | Medium |
| HTTP Error | `HTTP/\d.\d [45]\d\d` | Medium |
| OutOfMemory | `OutOfMemory|OOM` | Critical |
| Connection Refused | `Connection refused|refused to connect` | High |

### 4. Integrar con CloudWatch Metrics

#### Usar EMF (Embedded Metric Format)

Application Insights soporta EMF para enviar métricas personalizadas:

```json
{
    "_aws": {
        "Timestamp": "2026-04-20T10:30:00.000Z",
        "CloudWatchMetrics": [{
            "Namespace": "MyApp/Checkout",
            "Dimensions": [["Service", "Operation"]],
            "Metrics": [{
                "Name": "RequestLatency",
                "Unit": "Milliseconds"
            }, {
                "Name": "RequestCount",
                "Unit": "Count"
            }]
        }]
    },
    "Service": "checkout",
    "Operation": "processPayment",
    "RequestLatency": 145,
    "RequestCount": 1
}
```

#### Enviar métricas vía CloudWatch Agent

```bash
# Configurar agent para enviar métricas custom
sudo vi /etc/awslogs/cloudwatch.conf

[cloudwatch]
region = us-east-1

[metrics]
namespace = MyApp/Custom
[[metrics]]
metric_name = queue_depth
value = current_queue_size
```

### 5. Configurar Alertas

#### Crear alarma desde Application Insights

1. En CloudWatch > Application Insights
2. Seleccionar aplicación
3. Click en "Alarms" tab
4. Click "Create alarm"
5. Configurar threshold

#### Alertas preconfiguradas

| Problema Detectado | Umbral Automático |
|-------------------|-------------------|
| High CPU | > 85% por 5 min |
| Memory | < 15% disponible |
| Lambda Errors | > 1% de errores |
| ALB Target Errors | > 5% de 5xx |
| Lambda Duration | p99 > umbral normal + 2 std |

## Integración con Systems Manager OpsCenter

Habilitar OpsCenter para crear tickets automáticos:

```bash
aws application-insights create-application \
    --resource-group-name "MiResourceGroup" \
    --ops-center-enabled true \
    --ops-item-sns-topic-arn arn:aws:sns:us-east-1:123456789012:ops-items
```

### Ver OpsItems creados

```bash
aws ssm describe-ops-items \
    --ops-item-filters '[{"Key": "OperationalData", "Values": ["Application Insights"]}]'
```

## Diagnóstico de Problemas

### Problema: Application Insights no detecta componentes

**Solución:**
1. Verificar que los recursos están en el resource group
2. Confirmar que CloudWatch Agent está instalado
3. Verificar permisos IAM del rol asignado

### Problema: No hay datos de logs

**Solución:**
1. Confirmar que los log groups están en CloudWatch
2. Verificar que el patrón de errores coincide con los logs
3. Revisar que el agente está enviando logs

### Problema: Dashboard no muestra problemas

**Solución:**
1. Confirmar que el timeframe es correcto
2. Verificar que la aplicación está generando logs
3. Revisar que el componente está soportado

```bash

    --resource-group-name "MiResourceGroup"
```

```bash
aws application-insights update-application \
    --resource-group-name "MiResourceGroup" \
    --ops-center-enabled true
```

## Mejores Prácticas

1. **Usar Resource Groups** para organizar recursos por aplicación
2. **Habilitar OpsCenter** para tracking de problemas
3. **Configurar tags** para identificar aplicaciones fácilmente
4. **Revisar dashboards** diariamente durante primeros días para calibrar
5. **Crear patrones custom** para errores específicos de tu aplicación

## Troubleshooting Checklist

- [ ] Resource Group existe y contiene los recursos
- [ ] IAM role tiene permisos de CloudWatch
- [ ] CloudWatch Agent instalado (para EC2)
- [ ] Logs están siendo enviados a CloudWatch
- [ ] Patrones de errores definidos
- [ ] Alertas configuradas con SNS topic
