# Anexo 03b.01: CloudWatch CLI Commands

## Dashboards

### Crear Dashboard

```bash
aws cloudwatch put-dashboard \
    --dashboard-name "MiDashboard" \
    --dashboard-body '{
        "widgets": [
            {
                "type": "metric",
                "properties": {
                    "metrics": [
                        ["AWS/EC2", "CPUUtilization", "InstanceId", "i-1234567890abcdef0"],
                        ["AWS/EC2", "NetworkIn", "InstanceId", "i-1234567890abcdef0"]
                    ],
                    "period": 300,
                    "stat": "Average",
                    "region": "us-east-1",
                    "title": "EC2 Metrics",
                    "view": "timeSeries"
                }
            },
            {
                "type": "log",
                "properties": {
                    "title": "Errores Recientes",
                    "query": "fields @timestamp, @message | filter @message like /error/i | sort @timestamp desc | limit 20",
                    "region": "us-east-1",
                    "timeRange": {
                        "mode": "relative",
                        "relative": 900
                    }
                }
            }
        ]
    }'
```

### Listar Dashboards

```bash
# Listar todos los dashboards
aws cloudwatch list-dashboards

# Filtrar por prefijo
aws cloudwatch list-dashboards --dashboard-name-prefix "Production"
```

### Obtener Detalles de Dashboard

```bash
aws cloudwatch get-dashboard --dashboard-name "MiDashboard"
```

### Eliminar Dashboard

```bash
aws cloudwatch delete-dashboards --dashboard-names "MiDashboard" "OtroDashboard"
```

## CloudWatch Logs

### Crear Grupo de Logs

```bash
aws logs create-log-group \
    --log-group-name "/aws/lambda/mi-funcion"
```

### Put Log Events

```bash
aws logs put-log-events \
    --log-group-name "/aws/lambda/mi-funcion" \
    --log-stream-name "2026/04/20/[$LATEST]abc123" \
    --log-events '[{"timestamp": 1713600000000, "message": "INFO: Application started"}]'
```

### Query con Logs Insights

```bash
# Iniciar query
QUERY_ID=$(aws logs insights start-query \
    --log-group-names "/aws/lambda/mi-funcion" \
    --start-time 1713500000 \
    --end-time 1713600000 \
    --query-string 'fields @timestamp, @message | filter @message like /error/i | limit 20' \
    --query 'QueryId' \
    --output text)

echo "Query ID: $QUERY_ID"

# Esperar y obtener resultados
sleep 10
aws logs insights get-query-results --query-id "$QUERY_ID"
```

### Métricas Basadas en Logs

```bash
# Crear métrica a partir de un patrón de log
aws logs put-metric-filter \
    --log-group-name "/aws/lambda/mi-funcion" \
    --filter-name "ErrorCount" \
    --filter-pattern '{"error"}' \
    --metric-transformations '[{"metricName": "Errors", "metricNamespace": "MyApp", "metricValue": "1"}]'
```

## CloudWatch Contributor Insights

### Crear Regla

```bash
aws cloudwatch put-insight-rule \
    --rule-name "TopErrorContributors" \
    --rule-state ENABLED \
    --log-group-name "/aws/lambda/mi-funcion" \
    --filter-pattern '{"message": ["ERROR"]}' \
    --aggregation-dimension '{\"Keys\": [\"remoteAddr\"], \"AggregateWith\": \"Sum\"}'
```

### Listar Reglas

```bash
aws cloudwatch list-insight-rules
```

### Habilitar/Deshabilitar Regla

```bash
aws cloudwatch put-insight-rule \
    --rule-name "TopErrorContributors" \
    --rule-state DISABLED
```

### Obtener Insight Contributors

```bash
aws cloudwatch get-insight-rule-report \
    --rule-name "TopErrorContributors" \
    --start-time 1713500000 \
    --end-time 1713600000 \
    --period 3600
```

## CloudWatch Application Insights

### Crear Application

```bash
aws application-insights create-application \
    --resource-group-name "MiResourceGroup" \
    --tags '[{"Key": "Environment", "Value": "Production"}]'
```

### Listar Applications

```bash
aws application-insights list-applications
```

### Configurar Componentes

```bash
aws application-insights list-components \
    --resource-group-name "MiResourceGroup"
```

### Eliminar Application

```bash
aws application-insights delete-application \
    --resource-group-name "MiResourceGroup"
```

## CloudWatch alarms

### Crear Alarma de Métrica

```bash
aws cloudwatch put-metric-alarm \
    --alarm-name "CPU-High-Alarm" \
    --metric-name "CPUUtilization" \
    --namespace "AWS/EC2" \
    --statistic "Average" \
    --period 300 \
    --threshold 80 \
    --comparison-operator "GreaterThanThreshold" \
    --evaluation-periods 2 \
    --alarm-actions arn:aws:sns:us-east-1:123456789012:mi-topic
```

### Crear Alarma de Logs Insights

```bash
aws cloudwatch put-metric-alarm \
    --alarm-name "HighErrorRate" \
    --metric-name "Errors" \
    --namespace "MyApp" \
    --statistic "Sum" \
    --period 300 \
    --threshold 10 \
    --comparison-operator "GreaterThanThreshold" \
    --evaluation-periods 2
```

### Listar Alarmas

```bash
aws cloudwatch describe-alarms
```

### Eliminar Alarma

```bash
aws cloudwatch delete-alarms --alarm-names "CPU-High-Alarm"
```

## CloudWatch Synthetics

### Crear Canary

```bash
aws synthetics create-canary \
    --name "mi-canary" \
    --code "{\"handler\": \"pageHandler\", \"script\": \"async function pageHandler(page) { await page.goto(\\\"https://mi-sitio.com\\\"); }\"}" \
    --execution-role-arn arn:aws:iam::123456789012:role/CloudWatchSyntheticsRole \
    --schedule-expression "rate(5 minutes)" \
    --runtime-version "syn-nodejs-3.0" \
    --vpc-config '{"SubnetIds": ["subnet-12345"], "SecurityGroupIds": ["sg-12345"]}'
```

2. **Configurar retenciones** apropiadas para logs (por defecto nunca expiran)
3. **Usar métricas compuestas** para reducir costos de almacenamiento
4. **Crear dashboards tácticos** para incidentes específicos
5. **Automatizar** la creación de dashboards con CloudFormation
