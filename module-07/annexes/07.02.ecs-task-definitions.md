# ECS Task Definitions - Guía de Configuración

> **Tiempo de lectura:** ~10 min | **Referencia:** [AWS ECS Task Definitions](https://docs.aws.amazon.com/AmazonECS/latest/developerguide/task_definitions.html)

## Conceptos Fundamentales

### Task Definition - Blueprint de Contenedor

```
┌─────────────────────────────────────────────────────────────┐
│              Task Definition (Family: webapp)               │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Container: webapp                                   │   │
│  │  ├── Image: 123456.dkr.ecr.us-east-1.amazonaws.com  │   │
│  │  │              /webapp:latest                       │   │
│  │  ├── Memory: 512 MB                                 │   │
│  │  ├── CPU: 256                                       │   │
│  │  ├── Port: 8000                                     │   │
│  │  └── Environment: ENV=production                    │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Network Mode: awsvpc                                       │
│  Requires Compatibilities: FARGATE                          │
│  Execution Role: ecsTaskExecutionRole                       │
└─────────────────────────────────────────────────────────────┘
```

### Task vs Service

| Concepto | Descripción |
|----------|-------------|
| **Task** | Una instancia de Task Definition (como un "contenedor corriendo") |
| **Service** | Mantiene un número específico de Tasks corriendo (alta disponibilidad) |
| **Cluster** | Grupo lógico de recursos (EC2 o Fargate) |

## Estructura JSON de Task Definition

```json
{
  "family": "webapp-task",
  "cpu": "256",
  "memory": "512",
  "networkMode": "awsvpc",
  "requiresCompatibilities": ["FARGATE"],
  "executionRoleArn": "arn:aws:iam::123456789:role/ecsTaskExecutionRole",
  "containerDefinitions": [
    {
      "name": "webapp",
      "image": "123456789.dkr.ecr.us-east-1.amazonaws.com/webapp:latest",
      "portMappings": [
        {
          "containerPort": 8000,
          "protocol": "tcp"
        }
      ],
      "essential": true,
      "environment": [
        {
          "name": "ENV",
          "value": "production"
        }
      ],
      "logConfiguration": {
        "logDriver": "awslogs",
        "options": {
          "awslogs-group": "/ecs/webapp",
          "awslogs-region": "us-east-1",
          "awslogs-stream-prefix": "ecs"
        }
      }
    }
  ]
}
```

## Execution Role vs Task Role

### Execution Role (ecsTaskExecutionRole)

Permite a ECS pulling imágenes y escribir logs:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "ecr:GetAuthorizationToken",
        "ecr:BatchCheckLayerAvailability",
        "ecr:GetDownloadUrlForLayer",
        "ecr:BatchGetImage"
      ],
      "Resource": "*"
    },
    {
      "Effect": "Allow",
      "Action": [
        "logs:CreateLogStream",
        "logs:PutLogEvents"
      ],
      "Resource": "arn:aws:logs:us-east-1:123456789:log-group:/ecs/*"
    }
  ]
}
```

### Task Role (app-specific)

Permite al contenedor acceder a otros servicios AWS:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "dynamodb:Scan",
        "sns:Publish"
      ],
      "Resource": "*"
    }
  ]
}
```

## Puertos y Networking

### awsvpc Mode (Fargate)

```json
{
  "containerDefinitions": [
    {
      "name": "webapp",
      "image": "my-image",
      "portMappings": [
        {
          "containerPort": 8000,
          "protocol": "tcp"
        }
      ]
    }
  ]
}
```

**Importante:** En awsvpc mode:
- El contenedor recibe su propia ENI
- No hay NAT ni port mapping
- El Security Group se asigna al nivel de Task
- La aplicación debe escuchar en `containerPort`

### Port Mappings Válidos

```json
{
  "portMappings": [
    {
      "containerPort": 80,
      "hostPort": 80,
      "protocol": "tcp"
    }
  ]
}
```

## Dependencias entre Contenedores

### Condiciones de Salud

```json
{
  "containerDefinitions": [
    {
      "name": "webapp",
      "dependsOn": [
        {
          "containerName": "nginx",
          "condition": "HEALTHY"
        }
      ]
    },
    {
      "name": "nginx",
      "healthCheck": {
        "command": ["CMD-SHELL", "curl -f http://localhost/ || exit 1"],
        "interval": 30,
        "timeout": 5,
        "retries": 3,
        "startPeriod": 10
      }
    }
  ]
}
```

### Condiciones de Start

| Condición | Descripción |
|-----------|-------------|
| **HEALTHY** | Espera a que el contenedor dependiente esté HEALTHY |
| **START** | Espera a que el contenedor comece |
| **COMPLETE** | Espera a que el contenedor complete |

## Recursos y Límites

### Combinaciones Válidas CPU/Memory (Fargate)

| CPU | Memoria Mínima | Memoria Máxima |
|-----|----------------|----------------|
| 0.25 vCPU | 512 MB | 1 GB |
| 0.5 vCPU | 1 GB | 3 GB |
| 1 vCPU | 2 GB | 8 GB |
| 2 vCPU | 4 GB | 16 GB |
| 4 vCPU | 8 GB | 30 GB |

### Ejemplo Multi-Contenedor

```json
{
  "family": "webapp-with-proxy",
  "cpu": "1024",
  "memory": "2048",
  "networkMode": "awsvpc",
  "requiresCompatibilities": ["FARGATE"],
  "containerDefinitions": [
    {
      "name": "nginx",
      "image": "nginx:latest",
      "essential": true,
      "portMappings": [
        {"containerPort": 80, "protocol": "tcp"}
      ],
      "dependsOn": [
        {"containerName": "webapp", "condition": "START"}
      ]
    },
    {
      "name": "webapp",
      "image": "my-webapp:latest",
      "essential": true,
      "portMappings": [
        {"containerPort": 8000, "protocol": "tcp"}
      ],
      "environment": [
        {"name": "PORT", "value": "8000"}
      ]
    }
  ]
}
```

## Comandos AWS CLI

```bash
# Registrar Task Definition
aws ecs register-task-definition \
    --cli-input-json file://task-definition.json

# Listar Task Definitions (familias)
aws ecs list-task-definitions \
    --family-prefix webapp

# Describir Task Definition específica
aws ecs describe-task-definition \
    --task-definition webapp:1

# Actualizar servicio con nueva revisión
aws ecs update-service \
    --cluster my-cluster \
    --service my-service \
    --task-definition webapp:2

# Deregistrar Task Definition
aws ecs deregister-task-definition \
    --task-definition webapp:3
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| `ResourceInitializationError` | Permisos ECR faltantes | Verificar execution role |
| `InvalidParameterException` | Combinación CPU/Memory inválida | Verificar tabla de combinaciones |
| `Essential container exited` | Error en contenedor esencial | `docker logs` o CloudWatch |
| `CannotPullContainerError` | Imagen no existe en ECR | Verificar nombre de imagen |
| `No Container Instances` | Cluster sin capacidad | Verificar Fargate o EC2 instances |

## Register vs Run Task

```bash
# Solo registrar (no ejecuta)
aws ecs register-task-definition --cli-input-json file://td.json

# Ejecutar UNA task
aws ecs run-task \
    --cluster my-cluster \
    --task-definition webapp:1 \
    --network-configuration "awsvpcConfiguration={subnets=[subnet-xxx],securityGroups=[sg-xxx]}"

# Ejecutar task en Fargate
aws ecs run-task \
    --cluster my-cluster \
    --task-definition webapp:1 \
    --launch-type FARGATE \
    --network-configuration "awsvpcConfiguration={subnets=[subnet-xxx],securityGroups=[sg-xxx]}"
```
