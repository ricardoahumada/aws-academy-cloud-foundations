## Imágenes Docker

### Listar y Buscar

```bash
# Listar imágenes locales
docker images

# Listar imágenes con formato
docker images --format "table {{.Repository}}\t{{.Tag}}\t{{.Size}}"

# Buscar imágenes en Docker Hub
docker search nginx

# Ver histórico de una imagen (capas)
docker history nginx:latest
```

### Construir Imagen

```bash
# Build con tag
docker build -t my-webapp:latest .

# Build con tag y path específico
docker build -t my-webapp:1.0.0 ./my-app

# Build sin cache
docker build --no-cache -t my-webapp:latest .

# Build con argumentos
docker build --build-arg VERSION=1.0.0 -t my-webapp:latest .
```

### Taggear y Push

```bash
# Taggear imagen
docker tag my-webapp:latest 123456789.dkr.ecr.us-east-1.amazonaws.com/my-webapp:latest

# Push a ECR
docker push 123456789.dkr.ecr.us-east-1.amazonaws.com/my-webapp:latest

# Pull desde ECR
docker pull 123456789.dkr.ecr.us-east-1.amazonaws.com/my-webapp:latest
```

### Eliminar

```bash
# Eliminar imagen específica
docker rmi my-webapp:latest

# Eliminar imágenes sin tag
docker image prune

# Eliminar todas las imágenes no usadas
docker image prune -a
```

## Contenedores

### Crear y Ejecutar

```bash
# Ejecutar contenedor en modo interactivo
docker run -it ubuntu:latest /bin/bash

# Ejecutar en modo detach (background)
docker run -d nginx:latest

# Ejecutar con nombre específico
docker run --name my-nginx nginx:latest

# Ejecutar con puerto publicado
docker run -d -p 8080:80 nginx:latest

# Ejecutar con variables de entorno
docker run -d -e ENV=production -e DEBUG=false my-app:latest

# Ejecutar con volumen montado
docker run -d -v /host/path:/container/path nginx:latest

# Ejecutar con límites de recursos
docker run -d --memory=512m --cpus=0.5 nginx:latest
```

### Listar y Gestionar

```bash
# Listar contenedores activos
docker ps

# Listar TODOS los contenedores (incluyendo parados)
docker ps -a

# Listar solo IDs
docker ps -q

# Ver logs de contenedor
docker logs my-container

# Ver logs en tiempo real
docker logs -f my-container

# Ver últimos N líneas
docker logs --tail 100 my-container

# Restart contenedor
docker restart my-container

# Stop/Start
docker stop my-container
docker start my-container
```

### Ejecutar Comandos dentro del Contenedor

```bash
# Abrir shell interactivo
docker exec -it my-container /bin/bash

# Ejecutar comando específico
docker exec my-container ls -la /app

# Ejecutar como usuario específico
docker exec -u root my-container apt-get update
```

### Eliminar

```bash
# Eliminar contenedor específico
docker rm my-container

# Eliminar contenedor corriendo (force)
docker rm -f my-container

# Eliminar todos los contenedores parados
docker container prune

# Eliminar TODOS los contenedores
docker rm -f $(docker ps -aq)
```

## Redes Docker

### Listar y Crear Redes

```bash
# Listar redes
docker network ls

# Ver información de red
docker network inspect bridge

# Crear red
docker network create --driver bridge my-network

# Conectar contenedor a red
docker network connect my-network my-container

# Desconectar
docker network disconnect my-network my-container
```

### Tipos de Redes

| Driver | Descripción | Caso de uso |
|--------|-------------|-------------|
| **bridge** | Red por defecto | Desarrollo local |
| **host** | Comparte red con host | Production (cuidado!) |
| **overlay** | Multi-host networking | Docker Swarm |
| **none** | Sin red | Aislamiento total |

## Volúmenes

### Listar y Crear

```bash
# Listar volúmenes
docker volume ls

# Crear volumen
docker volume create my-data

# Inspect volumen
docker volume inspect my-data

# Eliminar volumen no usado
docker volume prune
```

### Usar Volúmenes

```bash
# Volumen persistente
docker run -d -v my-data:/app/data my-app:latest

# Bind mount (path del host)
docker run -d -v /host/data:/container/data my-app:latest

# Volumen de solo lectura
docker run -d -v my-data:/app/data:ro my-app:latest
```

## Docker Compose

### Ejemplo docker-compose.yml

```yaml
version: '3.8'

services:
  web:
    build: .
    ports:
      - "8080:8000"
    environment:
      - ENV=production
    volumes:
      - ./data:/app/data
    depends_on:
      - db
    networks:
      - app-network

  db:
    image: postgres:14
    environment:
      POSTGRES_PASSWORD: secret
      POSTGRES_DB: myapp
    volumes:
      - db-data:/var/lib/postgresql/data
    networks:
      - app-network

volumes:
  db-data:

networks:
  app-network:
    driver: bridge
```

### Comandos Compose

```bash
# Iniciar servicios
docker-compose up -d

# Ver estado
docker-compose ps

# Ver logs
docker-compose logs -f web

# Escalar servicio
docker-compose up -d --scale web=3

# Detener y eliminar
docker-compose down

# Rebuild images
docker-compose up -d --build
```

## Dockerfile - Mejores Prácticas

```dockerfile
# ❌ BAD - Orden ineficiente
FROM ubuntu
COPY . /app
RUN apt-get update && apt-get install -y python3
RUN pip install -r /app/requirements.txt
EXPOSE 8000

# ✅ GOOD - Orden eficiente, capas optimizadas
FROM python:3.11-slim

WORKDIR /app

# Copiar solo requirements primero (mejor caché)
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

# Luego copiar código
COPY . .

EXPOSE 8000

CMD ["python", "app.py"]
```

### Tips de Optimización

```dockerfile
# Usar imagen slim/minimal
FROM python:3.11-slim

# Limpiar caches en la misma capa
RUN apt-get update && \
    apt-get install -y --no-install-recommends curl && \
    rm -rf /var/lib/apt/lists/*

# No usar ADD para archivos simples
COPY config.json /app/config.json

# Multi-stage build para reducir tamaño
FROM golang:1.21 AS builder
WORKDIR /app
COPY . .
RUN go build -o myapp

FROM alpine:latest
COPY --from=builder /app/myapp /myapp
CMD ["/myapp"]
```

## ECR - Login y Comandos

```bash
# Login a ECR (requerido antes de push/pull)
aws ecr get-login-password --region us-east-1 | \
    docker login --username AWS \
    --password-stdin 123456789.dkr.ecr.us-east-1.amazonaws.com

# Pull imagen pública a ECR (mirroring)
aws ecr create-repository \
    --repository-name mirror/nginx \
    --image-tag-mutability MUTABLE

docker pull nginx:latest
docker tag nginx:latest 123456789.dkr.ecr.us-east-1.amazonaws.com/mirror/nginx:latest
docker push 123456789.dkr.ecr.us-east-1.amazonaws.com/mirror/nginx:latest
```

## Troubleshooting

| Problema | Solución |
|----------|----------|
| `docker: permission denied` | Usar `sudo` o agregar usuario al grupo `docker` |
| `port already allocated` | Cambiar puerto `-p 8081:80` |
| `no space left on device` | `docker system prune -a` |
| `image not found` | Verificar nombre y tag correctos |
| `connection refused` | Verificar que el servicio dentro del contenedor escucha en el puerto correcto |
| Container exits immediately | `docker logs` para ver errores |

## Comandos Útiles para Debug

```bash
# Ver procesos dentro del contenedor
docker top my-container

# Ver uso de recursos
docker stats

# Ver información completa
docker inspect my-container

# Ver mapeo de puertos
docker port my-container

# Copiar archivos desde contenedor
docker cp my-container:/app/logs ./local-logs

# Ver diferencias con imagen base
docker diff my-container
```
