# Fargate vs EC2 - Comparativa para ECS

> **Tiempo de lectura:** ~8 min | **Referencia:** [AWS ECS Launch Types](https://docs.aws.amazon.com/AmazonECS/latest/developerguide/launch_types.html)

---

## Comparativa General

```
┌─────────────────────────┬─────────────────────────┐
│         FARGATE          │          EC2            │
├─────────────────────────┼─────────────────────────┤
│ Serverless              │ Gestiona tus propias    │
│ No EC2 instances        │ EC2 instances          │
│ Pay per task runtime    │ Pay per instance hour   │
│ Aislamiento por task    │ Comparte instancias      │
│ Menos ops               │ Más control y flexibilidad│
└─────────────────────────┴─────────────────────────┘
```

---

## Características Detalladas

| Característica | Fargate | EC2 |
|----------------|---------|-----|
| **Infraestructura** | AWS gestiona | Tú gestionas |
| **Scaling** | Automático por task | Manual o ASG |
| **Pricing** | vCPU + GB/hour | Instance hour |
| **Isolation** | Task-level | Instance-level |
| **Persistent storage** | EFS mount (solo) | EBS + EFS |
| **Port mapping** | No (awsvpc) | Sí (bridge/host) |
| **GPU support** | No | Sí |
| **Windows containers** | No | Sí |
| **Host volume access** | No | Sí |

## Casos de Uso

### Fargate - Ideal para:

```
✅ Microservicios con tráfico variable
✅ Aplicaciones serverless
✅ Entornos de desarrollo/testing
✅ Batch jobsInfrequent workloads
✅ No quieres gestionar infraestructura
✅ Compliance - aislamiento fuerte
```

### EC2 - Ideal para:

```
✅ Workloads constants de alta performance
✅ GPU workloads (ML/AI)
✅ Windows containers
✅ Cost optimization en producción
✅ Persistent local storage (EBS)
✅ Custom networking (host mode)
✅ Long-running applications
```

## Pricing Examples

### Fargate Pricing (us-east-1)

```bash
# Task con 1 vCPU, 2GB RAM corriendo 24/7 por 30 días
vCPU: 1 * $0.04048/hour * 24 * 30 = $29.15
Memory: 2GB * $0.004445/hour * 24 * 30 = $6.40
Total Fargate: ~$35.55/month

# Misma task con Fargate Spot (60-70% descuento)
Fargate Spot: ~$12-15/month
```

### EC2 Pricing (t3.medium example)

```bash
# t3.medium: 2 vCPU, 4GB RAM
On-Demand: $0.0416/hour * 24 * 30 = $29.95/month
# En una instancia puedo correr 4-8 tasks de 0.25 vCPU
# Costo efectivo por task: ~$4-7/month

# Reserved Instance (1 year)
RI: ~$18/month = ~$0.025/hour
# Costo efectivo por task: ~$2-4/month
```

### Comparativa de Costos

| Scenario | Fargate | EC2 (On-Demand) | EC2 (RI) |
|----------|---------|-----------------|-----------|
| 1 task (1vCPU, 2GB) | ~$35.55/mo | ~$30/mo | ~$20/mo |
| 4 tasks mismo spec | ~$142/mo | ~$30/mo | ~$20/mo |
| 10 tasks | ~$355/mo | ~$75/mo | ~$50/mo |

**Conclusión:** EC2 es más económico para múltiples tasks en la misma instancia. Fargate es más económico cuando tienes pocas tasks o tráfico muy variable.

### Fargate: awsvpc Mode

```
┌─────────────────────────────────────────────────────────────┐
│                        Fargate                             │
│                                                             │
│   Task 1              Task 2              Task 3           │
│   ┌──────┐            ┌──────┐            ┌──────┐          │
│   │ ENI  │            │ ENI  │            │ ENI  │          │
│   │eth0  │            │eth0  │            │eth0  │          │
│   └──┬───┘            └──┬───┘            └──┬───┘          │
│      │                   │                   │              │
│      └───────────────────┴───────────────────┘              │
│                         │                                   │
│                    Security Group                          │
│                    (asignado a Task)                        │
└─────────────────────────────────────────────────────────────┘
```

### EC2: bridge/host Mode

```
┌─────────────────────────────────────────────────────────────┐
│                     EC2 Instance                            │
│                                                             │
│   ┌─────────┐   ┌─────────┐   ┌─────────┐                │
│   │ Docker  │   │ Docker  │   │ Docker  │                │
│   │ Bridge  │   │ Bridge  │   │ Bridge  │                │
│   │  net    │   │  net    │   │  net    │                │

│        │              │              │                     │
│        └──────────────┴──────────────┘                     │
│                     │                                       │
│              Host Network                                  │
│              (eth0 - instance ENI)                         │
│                                                             │

│   - Task1: 80:8000                                         │
│   - Task2: 8080:8000                                       │

```

## Storage - Diferencias

### Fargate

| Storage Type | Descripción |
|-------------|-------------|
| **Ephemeral** | /tmp - 20GB default, hasta 200GB |
| **EFS** | Solo opção - montaje de volumes EFS |

```json
{
  "name": "webapp",
  "image": "my-image",
  "mountPoints": [
    {
      "sourceVolume": "efs-volume",
      "containerPath": "/data",
      "readOnly": false
    }
  ],
  "volumesFrom": []
}
```

### EC2

| Storage Type | Descripción |
|-------------|-------------|
| **Ephemeral** | /tmp - 50GB (instance store) |
| **EBS** | Volúmenes EBS attachados |
| **EFS** | Compartido entre instancias |
| **FSx** | Windows file systems |

## Auto Scaling

### Fargate - Target Tracking

```bash
aws ecs put-account-setting-default \
    --name serviceLongArnFormat \
    --value enabled

# Crear Service con Auto Scaling
aws ecs create-service \
    --cluster my-cluster \
    --service-name my-service \
    --task-definition webapp \
    --desired-count 2 \
    --launch-type FARGATE \
    --network-configuration "awsvpcConfiguration={subnets=[subnet-xxx]}"

# Configurar Auto Scaling
aws application-autoscaling register-t ScalableTarget \
    --namespace aws:ecs \
    --resource service/my-cluster/my-service \
    --scalable-dimension ecs:service:DesiredCount \
    --service-namespace ecs \
    --min-capacity 2 \
    --max-capacity 10

# Target tracking policy
aws application-autoscaling put ScalingPolicy \
    --policy-name my-target-tracking \
    --resource-id service/my-cluster/my-service \
    --scalable-dimension ecs:service:DesiredCount \
    --service-namespace ecs \
    --policy-type TargetTrackingScaling \
    --target-tracking-configuration file://target-tracking.json
```

```json
// target-tracking.json
{
  "TargetTrackingScalingPolicyConfiguration": {
    "TargetValue": 50,
    "ScaleInCooldown": 60,
    "ScaleOutCooldown": 60,
    "PredefinedMetricSpecification": {
      "PredefinedMetricType": "ECSServiceAverageCPUUtilization"
    }
  }
}
```

### EC2 - Auto Scaling Group

```bash
# Crear ASG
aws autoscaling create-auto-scaling-group \
    --auto-scaling-group-name my-ecs-asg \
    --min-size 1 \
    --max-size 10 \
    --desired-capacity 2 \
    --vpc-zone-identifier "subnet-xxx,subnet-yyy" \
    --launch-template "LaunchTemplateName=my-ecs-template"

# Configurar scaling policies basadas en CPU/memory de EC2
# O métricas custom de container insights
```

## Cuándo Migrar de Fargate a EC2

### Señales para Migrar a EC2

1. **Costos altos** - Más de 10-15 tasks persistent
2. **GPU needed** - ML/AI workloads
3. **Windows containers** - No hay Fargate para Windows
4. **Persistent local storage** - Necesitas EBS volumes
5. **Custom networking** - host mode networking
6. **Compliance requirements** - Control de infra específico

### Estrategia de Optimización

```
┌─────────────────────────────────────────────────────────────┐
│                 Estrategia Híbrida                          │
│                                                             │
│  Fargate: Dev/Test, Staging, Batch jobs                    │
│  EC2 (RI): Production microservices constants              │
│  Fargate Spot: Non-critical background tasks                │
│                                                             │
│  Ejemplo:                                                  │
│  - 5 microservices constants → EC2 (1-2 instancias)       │
│  - Dev/Test environments → Fargate                         │
│  - Job processor (spot) → Fargate Spot                    │
└─────────────────────────────────────────────────────────────┘
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| `InsufficientInstanceCapacity` (EC2) | AZ sin capacidad | Usar múltiples AZs o diferentes instance types |
| Tasks tardan en start (Fargate) | Cold start | Usar provisioned concurrency o mantener min tasks |
| `ExecFormatError` (EC2) | ARM vs x86 mismatch | Verificar arquitectura de imagen |
| High CPU on EC2 | Many tasks en few instances | Configurar service at max capacity, ASG scaling |
| Port conflicts (EC2) | Host mode port mapping | Usar diferentes host ports o bridge mode |
