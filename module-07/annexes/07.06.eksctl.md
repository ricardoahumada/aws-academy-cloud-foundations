# Mini-Guía de eksctl

## Instalación

```bash
# Linux
curl --silent --location "https://github.com/eksctl-io/eksctl/releases/download/v0.225.0/eksctl_Linux_amd64.tar.gz" | tar xz -C /tmp
sudo mv /tmp/eksctl /usr/local/bin

# macOS
brew install eksctl

# Verificar
eksctl version
```

## Operaciones con Clusters

### Crear cluster desde archivo YAML
```bash
eksctl create cluster -f cluster.yaml
```

### Crear cluster interactivo
```bash
eksctl create cluster --name my-cluster --region us-east-1 --version 1.30
```

### Crear cluster con node group
```bash
eksctl create cluster \
  --name my-cluster \
  --region us-east-1 \
  --nodegroup-name workers \
  --node-type t3.small \
  --nodes 2 \
  --nodes-min 1 \
  --nodes-max 3
```

### Eliminar cluster
```bash
eksctl delete cluster -f cluster.yaml
# o
eksctl delete cluster --name my-cluster --region us-east-1
```

### Listar clusters
```bash
eksctl get cluster --region us-east-1
```

### Describir cluster
```bash
eksctl describe cluster --name my-cluster --region us-east-1
```

## Node Groups

### Crear node group en cluster existente
```bash
eksctl create nodegroup \
  --cluster my-cluster \
  --name workers \
  --instance-type t3.small \
  --nodes 2 \
  --nodes-min 1 \
  --nodes-max 3 \
  --node-ami-family AmazonLinux2
```

### Escalar node group
```bash
eksctl scale nodegroup \
  --cluster my-cluster \
  --name workers \
  --nodes 4
```

### Eliminar node group
```bash
eksctl delete nodegroup \
  --cluster my-cluster \
  --name workers
```

###-etiquetas y tags en nodos
```bash
eksctl create nodegroup \
  --cluster my-cluster \
  --name workers \
  --instance-type t3.small \
  --labels role=worker \
  --tags Environment=development
```

## IAM y OIDC

### Habilitar OIDC en cluster
```bash
eksctl utils update-iam-service-with-oidc \
  --cluster my-cluster \
  --region us-east-1
```

### Crear service account con IAM role
```bash
eksctl create iamserviceaccount \
  --cluster my-cluster \
  --namespace kube-system \
  --name aws-load-balancer-controller \
  --attach-policy-arn arn:aws:iam::123456789012:policy/MyPolicy \
  --override-existing-serviceaccounts \
  --approve
```

## Addons

### Instalar addon (vpc-cni, coredns, kube-proxy)
```bash
eksctl create addon \
  --cluster my-cluster \
  --name vpc-cni \
  --region us-east-1
```

### Actualizar addon
```bash
eksctl update addon \
  --cluster my-cluster \
  --name vpc-cni \
  --version latest
```

### Eliminar addon
```bash
eksctl delete addon \
  --cluster my-cluster \
  --name vpc-cni
```

## Actualizar kubeconfig
```bash
aws eks update-kubeconfig --name my-cluster --region us-east-1
```

## Archivo YAML de Ejemplo

```yaml
apiVersion: eksctl.io/v1alpha5
kind: ClusterConfig

metadata:
  name: my-eks-cluster
  region: us-east-1
  version: "1.30"

vpc:
  id: vpc-xxxxxxxx
  subnets:
    private:
      us-east-1a: { id: subnet-xxxxxxxx }
      us-east-1b: { id: subnet-xxxxxxxx }
  nat:
    gateway: Single

managedNodeGroups:
  - name: workers
    instanceType: t3.small
    desiredCapacity: 2
    minSize: 1
    maxSize: 3
    privateNetworking: true
    labels:
      role: worker
    tags:
      Environment: development

iam:
  withOIDC: true

addons:
  - name: vpc-cni
  - name: coredns
  - name: kube-proxy
```

## Troubleshooting

### Cluster no se crea
- Verificar permisos IAM
- Verificar límites de recursos (VPCs, EIPs)
- Revisar CloudFormation console

### Node group falla
- Verificar que el IAM role existe y tiene políticas correctas
- Verificar que las subnets tienen tags correctos
- Revisar logs: `eksctl get nodegroup --cluster my-cluster`

### Permission denied en OIDC
```bash
# Verificar que el cluster tiene OIDC habilitado
eksctl describe cluster --name my-cluster --region us-east-1

# Habilitar si no existe
eksctl utils associate-iam-oidc-provider --cluster my-cluster --approve
```
