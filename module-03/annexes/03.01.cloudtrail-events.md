# CloudTrail - Análisis de Eventos

> **Tiempo de lectura:** ~8 min | **Referencia:** [CloudTrail User Guide](https://docs.aws.amazon.com/awscloudtrail/latest/userguide/)

---

## ¿Qué es CloudTrail?

AWS CloudTrail registra llamadas API realizadas en tu cuenta AWS. Cada evento registrado incluye:
- **Quién** hizo la llamada (identidad)
- **Qué** acción realizó
- **Cuándo** ocurrió
- **Dónde** se hizo (IP, región)

```
┌─────────────────────────────────────────────────────────────┐
│  CloudTrail Event                                            │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  Who: user@example.com (ARN, principal)             │    │
│  │  What: ec2:TerminateInstances                       │    │
│  │  When: 2024-01-15T10:30:00Z                        │    │
│  │  Where: 203.0.113.45 / us-east-1                   │    │
│  │  Result: Success / AccessDenied                    │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

## Tipos de Eventos

### Por Categoría

| Tipo | Descripción | Retention |
|------|-------------|-----------|
| **Management** | Operaciones en recursos (Create, Delete, Modify) | 90 días |
| **Data** | Operaciones en datos (S3 object, Lambda invoke) | 90 días |
| **Insights** | Anomalías en patrones de llamadas API | 90 días |

## Consultar Event History

### AWS Console

1. CloudTrail > Event history
2. Filtrar por:
   - Attribute: Event name, Resource name, User name, etc.
   - Time range: Last 15 minutes, 1 hour, 7 days, Custom

### AWS CLI

```bash
# Últimos eventos (por defecto, últimas 2 horas)
aws cloudtrail lookup-events

# Filtrar por nombre de evento
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=ConsoleLogin

# Filtrar por usuario
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=Username,AttributeValue=admin-juan

# Filtrar por recurso
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=ResourceName,AttributeValue=i-1234567890abcdef0

# Filtrar por rango de tiempo
aws cloudtrail lookup-events \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-02T00:00:00Z
```

## Trails (Registros Persistentes)

### Crear Trail

```bash
# Trail básico
aws cloudtrail create-trail \
  --name security-audit-trail \
  --s3-bucket-name mi-cloudtrail-logs-123456789012 \
  --is-multi-region \
  --include-global-service-events

# Habilitar validación de logs (tamper-evident)
aws cloudtrail update-trail \
  --name security-audit-trail \
  --enable-log-file-validation

# Habilitar CloudWatch Logs
aws cloudtrail update-trail \
  --name security-audit-trail \
  --cloudwatch-logs-log-group-arn arn:aws:logs:us-east-1:123456789012:log-group:/aws/cloudtrail:* \
  --cloudwatch-logs-role-arn arn:aws:iam::123456789012:role/CloudTrail_CloudWatchLogs_Role
```

### Estructura de Logs en S3

```
s3://mi-cloudtrail-logs-123456789012/
└── AWSLogs/
    └── 123456789012/
        └── CloudTrail/
            └── us-east-1/
                └── 2024/
                    └── 01/
                        └── 15/
                            └── 123456789012_CloudTrail_us-east-1_20240115T103000Z_xxxxxxxx.json.gz
```

### Decodificar Log Files

```bash
# Descargar y descomprimir
aws s3 cp s3://mi-cloudtrail-logs-123456789012/AWSLogs/123456789012/CloudTrail/us-east-1/2024/01/15/123456789012_CloudTrail_us-east-1_20240115T103000Z_xxxxxxxx.json.gz /tmp/

# Ver contenido (gzipped - usar zcat o gunzip)
zcat /tmp/123456789012_CloudTrail_us-east-1_20240115T103000Z_xxxxxxxx.json.gz
```

## CloudWatch Logs Integration

### Buscar Eventos en CloudWatch

```bash
# Listar grupos de logs de CloudTrail
aws logs describe-log-groups \
  --log-group-name-prefix /aws/cloudtrail

# Ver streams
aws logs describe-log-streams \
  --log-group-name /aws/cloudtrail/security-audit

# Filtrar eventos por patrón
aws logs filter-log-events \
  --log-group-name /aws/cloudtrail/security-audit \
  --start-time 1704067200000 \
  --filter-pattern "{$.eventName=ConsoleLogin}"

# Filtrar eventos fallidos
aws logs filter-log-events \
  --log-group-name /aws/cloudtrail/security-audit \
  --filter-pattern "{$.errorCode=AccessDenied}"
```

### Patrones de Filtro Útiles

```json
// Evento específico
{$.eventName=TerminateInstances}

// Eventos de un servicio
{$.eventSource=iam.amazonaws.com}

// IPs específicas
{$.sourceIPAddress=192.168.1.100}

// Accesos fallidos
{$.errorCode=AccessDenied}

// Todos los eventos excepto lecturas
{$.eventType!=AwsApiCall || $.readOnly=false}
```

## Insights Events

Detectan anomalías en patrones de API:

```bash
# Habilitar Insights
aws cloudtrail update-trail \
  --name security-audit-trail \
  --enable-legacy-header \
  --insight-selector '[{"InsightType":"ApiCallRateInsight"}]'

# Ver eventos de Insights
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventSource,AttributeValue=cloudtrail.amazonaws.com
```

**Insights comunes:**
- Aumento dramático en llamadas API
- Calls desde ubicaciones inusuales
- Uso de APIs que normalmente no se usan

## Validación de Logs

Verificar que los logs no han sido manipulados:

```bash
# Verificar integridad de archivo
aws cloudtrail validate-logs \
  --trail-arn arn:aws:cloudtrail:us-east-1:123456789012:trail/security-audit \
  --start-time 2024-01-01T00:00:00Z

# O usando la CLI de CloudTrail directamente
cloudtrail validate-logs \
  --trail-arn arn:aws:cloudtrail:us-east-1:123456789012:trail/security-audit \
  --start-time 2024-01-01T00:00:00Z
```

## Eventos de Seguridad Clave

### Autenticación

```bash
# Console logins exitosos
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=ConsoleLogin \
  --query 'Events[?contains(eventJson, `"ConsoleLogin": "Success"`)]'

# Logins fallidos (possible brute force)
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=ConsoleLogin \
  --query 'Events[?contains(eventJson, `"ConsoleLogin": "Failure"`)]'
```

### IAM

```bash
# Crear usuario
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=CreateUser

# Cambios de política
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=PutUserPolicy

# Uso de Access Keys
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=CreateAccessKey
```

### EC2

```bash
# Instancias creadas
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=RunInstances

# Instancias terminadas
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=TerminateInstances

# Cambios de Security Groups
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=AuthorizeSecurityGroupIngress
```

### S3

```bash
# Bucket creado
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=CreateBucket

# Cambios de policy
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=PutBucketPolicy

# Objetos eliminados
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=DeleteObject
```

## Troubleshooting

### No veo eventos

1. Verificar que CloudTrail está habilitado
```bash
aws cloudtrail describe-trails
```

2. Verificar que el trail está aktif
```bash
aws cloudtrail get-trail-status --name security-audit-trail
```

3. Verificar permisos de S3 bucket
4. Verificar latencia (puede haber delay de 5-15 minutos)

### Logs no se entregan a S3

1. Verificar bucket policy permite CloudTrail
2. Verificar que el trail tiene `include-global-service-events` para IAM
3. Revisar CloudTrail delivery en CloudWatch Logs

### Error de permisos

```bash
# Verificar role de CloudTrail
aws cloudtrail get-event-selectors \
  --trail-name security-audit-trail
```

## Comandos Relacionados

```bash
# Listar todos los trails
aws cloudtrail describe-trails

# Ver selectores de eventos
aws cloudtrail get-event-selectors --trail-name security-audit-trail

# Listar Trails por región
aws cloudtrail list-trails --region us-east-1

# Ver trail status
aws cloudtrail get-trail-status --name security-audit-trail

# Stop/Start logging
aws cloudtrail stop-logging --name security-audit-trail
aws cloudtrail start-logging --name security-audit-trail
```
