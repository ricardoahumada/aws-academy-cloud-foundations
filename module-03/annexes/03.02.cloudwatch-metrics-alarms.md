## Operaciones con AWS CLI

### Listar Métricas

```bash
# Listar todas las métricas disponibles
aws cloudwatch list-metrics

# Filtrar por namespace
aws cloudwatch list-metrics --namespace AWS/EC2

# Filtrar por nombre de métrica
aws cloudwatch list-metrics \
  --namespace AWS/EC2 \
  --metric-name CPUUtilization

# Filtrar por dimensión
aws cloudwatch list-metrics \
  --namespace AWS/EC2 \
  --dimensions Name=InstanceId,Value=i-1234567890abcdef0
```

### Obtener Métricas (GetMetricData)

```bash
# Obtener uso de CPU de última hora
aws cloudwatch get-metric-statistics \
  --namespace AWS/EC2 \
  --metric-name CPUUtilization \
  --dimensions Name=InstanceId,Value=i-1234567890abcdef0 \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-01T01:00:00Z \
  --period 300 \
  --statistics Average,Maximum
```

### GetMetricWidgetImage

```bash
# Generar imagen PNG de gráfico de métricas
aws cloudwatch get-metric-widget-image \
  --metric-widget '{
    "metrics": [["AWS/EC2","CPUUtilization","InstanceId","i-1234567890abcdef0"]],
    "period":300,
    "stat":"Average",
    "width":600,
    "height":200
  }'
```

## Alarmas (CloudWatch Alarms)

### Crear Alarma Simple

```bash
# Alarma: CPU > 80% por 3 períodos consecutivos de 5 minutos
aws cloudwatch put-metric-alarm \
  --alarm-name HighCPU-Instances \
  --alarm-description "Alarma cuando CPU supera 80%" \
  --namespace AWS/EC2 \
  --metric-name CPUUtilization \
  --dimensions Name=InstanceId,Value=i-1234567890abcdef0 \
  --statistic Average \
  --period 300 \
  --threshold 80 \
  --comparison-operator GreaterThanThreshold \
  --evaluation-periods 3 \
  --alarm-actions arn:aws:sns:us-east-1:123456789012:mi-topic
```

### Tipos de Estatísticas

| Estadística | Descripción |
|-------------|-------------|
| `SampleCount` | Número de muestras |
| `Average` | Promedio |
| `Sum` | Suma de valores |
| `Minimum` | Valor mínimo |
| `Maximum` | Valor máximo |

### Operadores de Comparación

| Operador | Descripción |
|----------|-------------|
| `GreaterThanThreshold` | Mayor que |
| `GreaterThanOrEqualToThreshold` | Mayor o igual |
| `LessThanThreshold` | Menor que |
| `LessThanOrEqualToThreshold` | Menor o igual |
| `LessThanLowerOrGreaterThanUpperThreshold` | Fuera de rango (para métricas con 2 umbrales) |

## Acciones de Alarmas

### SNS (Simple Notification Service)

```bash
# Crear alarma con acción de SNS
aws cloudwatch put-metric-alarm \
  --alarm-name "Critical-CPU-Alarm" \
  --alarm-actions arn:aws:sns:us-east-1:123456789012:critical-alerts \
  --ok-actions arn:aws:sns:us-east-1:123456789012:alerts-resolved \
  # ... resto de parámetros
```

### Auto Scaling

```bash
# Agregar acción de ASG
aws cloudwatch put-metric-alarm \
  --alarm-name "ScaleUp-OnHighCPU" \
  --alarm-actions arn:aws:autoscaling:us-east-1:123456789012:scalingPolicy::autoScalingGroupName:mi-asg:policyName:mi-policy
```

### EC2 Actions

```bash
# Recover instance (solo para instancias con CloudWatch Agent)
aws cloudwatch put-metric-alarm \
  --alarm-name "Instance-Recover" \
  --alarm-actions arn:aws:automate:us-east-1:ec2:recover
```

## Dashboards

### Crear Dashboard

```bash
aws cloudwatch put-dashboard \
  --dashboard-name "Mi-Dashboard" \
  --dashboard-body '{
    "widgets": [
      {
        "type": "metric",
        "properties": {
          "metrics": [
            ["AWS/EC2","CPUUtilization","InstanceId","i-1234567890abcdef0"],
            [".","NetworkIn",".","."],
            [".","NetworkOut",".","."]
          ],
          "period": 300,
          "stat": "Average",
          "region": "us-east-1",
          "title": "EC2 Metrics"
        }
      },
      {
        "type": "text",
        "properties": {
          "markdown": "# My Dashboard"
        }
      }
    ]
  }'
```

### Listar Dashboards

```bash
aws cloudwatch list-dashboards

# Eliminar dashboard
aws cloudwatch delete-dashboards --dashboard-names Mi-Dashboard
```

## Logs (CloudWatch Logs)

### Conceptos

```
Log Group (ej: /aws/lambda/mi-funcion)
  └── Log Stream (ej: ip-172-31-1-1)
        └── Log Events (mensajes individuales con timestamp)
```

### Comandos Básicos

```bash
# Listar log groups
aws logs describe-log-groups

# Crear log group
aws logs create-log-group --log-group-name /mi/app/logs

# Ver streams
aws logs describe-log-streams \
  --log-group-name /aws/ec2/instance-health

# Filtrar eventos
aws logs filter-log-events \
  --log-group-name /aws/cloudtrail \
  --start-time 1704067200000 \
  --filter-pattern "{$.eventName=ConsoleLogin}"
```

### Metric Filters

Extraer métricas de logs:

```bash
# Crear filtro de métrica (contar errores 500)
aws logs put-metric-filter \
  --log-group-name /aws/ec2/app \
  --filter-name HttpServerErrors \
  --metric-transformations '{
    "metricName": "Http5xxErrors",
    "metricNamespace": "MyApp",
    "metricValue": "1"
  }' \
  --filter-pattern '{$.statusCode=500}'
```

## Troubleshooting

### Alarma no se dispara

1. Verificar que la métrica está siendo publicada
```bash
aws cloudwatch get-metric-statistics --namespace AWS/EC2 ...
```

2. Verificar umbrales y períodos
3. Verificar que hay datos suficientes (mínimo 1 punto para evaluation period)

### Sin datos de métricas EC2

1. Verificar que CloudWatch Agent está instalado (si son métricas custom)
2. Verificar que Instance ID是对的
3. Verificar que la instancia tiene permisos de CloudWatch

### Dashboard no carga

```bash
# Verificar errores
aws cloudwatch get-dashboard \
  --dashboard-name Mi-Dashboard
```

## Integración con CloudTrail

CloudWatch puede recibir eventos de CloudTrail:

```bash
# Buscar eventos de API en CloudWatch Logs
aws logs filter-log-events \
  --log-group-name /aws/cloudtrail \
  --start-time 1704067200000 \
  --filter-pattern "{$.eventSource=ec2.amazonaws.com}"
```

## Best Practices

1. **Usar períodos adecuados**: 5 min para métricas de negocio, 1 min para críticos
2. **Establecer alarm沉默了**: No alarmar por spikes únicos
3. **Usar SNS para notificaciones**: Centralizar en un topic
4. **Utilizar dashboards**: Para vista consolidada
5. **Configurar retención**: 30 días para logs operativos, más para compliance
