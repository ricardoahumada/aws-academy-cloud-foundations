# AWS KMS - Conceptos y Operaciones

> **Tiempo de lectura:** ~8 min | **Referencia:** [AWS KMS Developer Guide](https://docs.aws.amazon.com/kms/latest/developerguide/)

## ¿Qué es AWS KMS?

AWS Key Management Service (KMS) es un servicio gestionado de creación y control de claves de cifrado. Permite crear y gestionar claves criptográficas integradas con la mayoría de servicios AWS.

**Características principales:**
- Claves de cifrado gestionadas por el usuario o AWS
- Integrado con +90 servicios de AWS
- Compliant con HIPAA, PCI DSS, FedRAMP, SOC
- Cifrado simétrico (AES-256) y asimétrico (RSA, ECC)

## Arquitectura de KMS

```
┌─────────────────────────────────────────────────────────────┐
│                     AWS KMS                                  │
│  ┌─────────────────────────────────────────────────────┐   │
│  │           Customer Master Key (CMK)                  │   │
│  │  ┌─────────────────────────────────────────────┐     │   │
│  │  │  Key Material (material criptográfico)     │     │   │
│  │  │  - AWS Managed (generado por AWS)           │     │   │
│  │  │  - Customer Managed (generado por usuario)  │     │   │
│  │  │  - Custom (importado por usuario)           │     │   │
│  │  └─────────────────────────────────────────────┘     │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
                          │
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                   Data Keys (DEK)                            │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Generadas por CMK para cifrar datos específicos    │   │
│  │  - GenerateDataKey → Plaintext + CiphertextBlob     │   │
│  │  - Plaintext se descarta después de uso              │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

## Tipos de CMK

| Tipo | Description | Rotación | Precio |
|------|-------------|----------|--------|
| **AWS Managed** | Creada automáticamente por AWS | Automática (1 año) | Gratis |
| **Customer Managed** | Creada por usuario | Manual/Automática | $1/mes + $0.03/10k operaciones |
| **AWS Owned** | Gestionadas por AWS (no visibles) | Variable | Gratis |
| **Custom Key Store** | En CloudHSM dedicado | Manual | $1.45/hora |

### AWS Managed vs Customer Managed

**AWS Managed:**
```bash
# Lista de AWS Managed keys (visibles en cuenta)
aws kms list-aliases --query 'Aliases[?contains(AliasName, `aws/`)]'
```

**Customer Managed:**
```bash
# Crear CMK personalizada
aws kms create-key \
  --description "Mi clave de cifrado" \
  --key-usage ENCRYPT_DECRYPT \
  --origin AWS_KMS \
  --tags TagKey=Purpose,TagValue=Education
```

## Operaciones con CMK

### Create Key

```bash
aws kms create-key \
  --description "Clave para datos financieros" \
  --key-usage ENCRYPT_DECRYPT \
  --origin AWS_KMS \
  --region us-east-1
```

**Respuesta:**
```json
{
  "KeyMetadata": {
    "KeyId": "1234abcd-12ab-34cd-56ef-1234567890ab",
    "Arn": "arn:aws:kms:us-east-1:123456789012:key/1234abcd-...",
    "KeyState": "Enabled",
    "KeyUsage": "ENCRYPT_DECRYPT",
    "Origin": "AWS_KMS"
  }
}
```

### Encrypt

```bash
# Cifrar datos (datos limitados a 4KB)
aws kms encrypt \
  --key-id alias/mi-clave \
  --plaintext fileb://datos-para-cifrar.txt \
  --output text --query CiphertextBlob | base64 --decode > cifrado.bin
```

### Decrypt

```bash
# Descifrar datos
aws kms decrypt \
  --ciphertext-blob fileb://cifrado.bin \
  --output text --query Plaintext | base64 --decode > datos-descifrados.txt
```

### Generate Data Key

**Útil para cifrar datos grandes (más de 4KB):**

```bash
# Generar par de claves (una en texto claro, otra cifrada con CMK)
aws kms generate-data-key \
  --key-id alias/mi-clave \
  --key-spec AES_256 \
  --encryption-context App=MiApp,Version=1
```

**Respuesta:**
```json
{
  "CiphertextBlob": "base64-encoded-encrypted-key",
  "Plaintext": "base64-encoded-plaintext-key",
  "KeyId": "arn:aws:kms:us-east-1:..."
}
```

**Flujo completo:**
```bash
# 1. Generar DEK
 aws kms generate-data-key --key-id alias/mi-clave --key-spec AES_256

# 2. Usar Plaintext DEK para cifrar archivo grande
openssl enc -aes256 -salt -in mi-archivo-grande.zip -out mi-archivo-grande.zip.enc -k $PLAIN_TEXT_DEK

# 3. Descartar Plaintext DEK inmediatamente

# 4. Guardar CiphertextBlob + archivo cifrado

# Para descifrar:
# 1. Extraer CiphertextBlob
# 2. Decrypt con KMS para obtener Plaintext DEK
# 3. Usar Plaintext DEK para descifrar archivo
```

### Re-encrypt

```bash
# Recifrar datos con una clave diferente
aws kms re-encrypt \
  --ciphertext-blob fileb://datos-cifrados.bin \
  --source-key-id alias/clave-origen \
  --destination-key-id alias/clave-destino
```

## Alias

Los alias facilitan referencia a CMKs sin usar el KeyId.

```bash
# Crear alias
aws kms create-alias \
  --alias-name alias/mi-clave-cifrado \
  --target-key-id 1234abcd-12ab-34cd-56ef-1234567890ab

# Usar alias en lugar de key ID
aws kms encrypt --key-id alias/mi-clave-cifrado --plaintext "hola"
```

## Key Policies

Cada CMK tiene una política de acceso (key policy).

### Key Policy Mínima

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "Enable IAM User Permissions",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::123456789012:root"},
      "Action": "kms:*",
      "Resource": "*"
    },
    {
      "Sid": "Allow Use of Key",
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::123456789012:user/my-user"},
      "Action": [
        "kms:Encrypt",
        "kms:Decrypt",
        "kms:GenerateDataKey*"
      ],
      "Resource": "*"
    }
  ]
}
```

### Policy para Cross-Account Access

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Principal": {"AWS": "arn:aws:iam::111122223333:root"},
      "Action": [
        "kms:Encrypt",
        "kms:Decrypt"
      ],
      "Resource": "*"
    }
  ]
}
```

## Integración con S3

### SSE-KMS en Bucket

```bash
# Habilitar cifrado por defecto en bucket
aws s3api put-bucket-encryption \
  --bucket mi-bucket-cifrado \
  --server-side-encryption-configuration '{
    "Rules": [
      {
        "ApplyServerSideEncryptionByDefault": {
          "SSEAlgorithm": "aws:kms",
          "KMSMasterKeyID": "alias/mi-clave"
        }
      }
    ]
  }'

# Subir con SSE-KMS específico
aws s3 cp archivo.txt s3://mi-bucket/ \
  --sse aws:kms \
  --sse-kms-key-id alias/mi-clave

# Verificar cifrado
aws s3api head-object --bucket mi-bucket --key archivo.txt
```

### Bucket Policy para Requerir KMS Encryption

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Deny",
      "Principal": "*",
      "Action": "s3:PutObject",
      "Resource": "arn:aws:s3:::mi-bucket/*",
      "Condition": {
        "StringNotEquals": {
          "s3:x-amz-server-side-encryption": "aws:kms"
        }
      }
    }
  ]
}
```

## Rotación de Claves

### Customer Managed Keys

```bash
# Habilitar rotación automática (cada 365 días)
aws kms enable-key-rotation --key-id alias/mi-clave

# Deshabilitar rotación
aws kms disable-key-rotation --key-id alias/mi-clave

# Ver estado de rotación
aws kms get-key-rotation-status --key-id alias/mi-clave
```

## Integración con CloudTrail

KMS registra todas las operaciones en CloudTrail:

```bash
# Buscar eventos de uso de clave específica
aws cloudtrail lookup-events \
  --lookup-attributes AttributeKey=EventName,AttributeValue=Encrypt

# Eventos relevantes:
# - GenerateDataKey
# - Encrypt
# - Decrypt
# - ReEncrypt
# - DisableKey
# - ScheduleKeyDeletion
```

## Troubleshooting

### Error: "Access Denied" al cifrar

1. Verificar que el principal tiene permiso `kms:Encrypt`
2. Verificar key policy permite acceso al usuario
3. Verificar que la clave está `Enabled`

```bash
# Describir estado de la clave
aws kms describe-key --key-id alias/mi-clave
```

### Error: "Key does not exist"

```bash
# Listar todas las claves disponibles
aws kms list-keys

# Verificar alias
aws kms list-aliases
```

### Cifrado no funciona en S3

1. Verificar que el bucket tiene cifrado habilitado
2. Verificar que la key existe y está habilitada
3. Verificar permisos IAM del usuario para usar la key
