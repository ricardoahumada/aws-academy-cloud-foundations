# IAM Policies JSON - Guía de Referencia

> **Tiempo de lectura:** ~10 min | **Referencia:** [IAM JSON Policy Reference](https://docs.aws.amazon.com/IAM/latest/UserGuide/reference_policies.html)

---

## Estructura Básica de una Política IAM

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow|Deny",
      "Action": ["servicio:acción"],
      "Resource": "arn:aws:servicio:región:cuenta:recurso",
      "Condition": { ... }
    }
  ]
}
```

### Elementos Principales

| Elemento | Descripción | Obligatorio |
|----------|-------------|-------------|
| `Version` | Versión del lenguaje de políticas | Sí |
| `Statement` | Bloque principal con uno o más permisos | Sí |
| `Sid` | Identificador opcional del statement | No |
| `Effect` | `Allow` o `Deny` | Sí |
| `Action` | Lista de acciones permitidas/denegadas | Sí |
| `Resource` | Recursos a los que aplica | Sí* |
| `Condition` | Condiciones bajo las cuales aplica | No |

*Excepto para políticas basadas en recursos donde Resource es obligatorio.

---

## Version

Solo existen dos valores válidos:

- **`2012-10-17`** - Versión actual del lenguaje
- **`2008-10-17`** - Versión legacy (no usar)

```json
"Version": "2012-10-17"
```

**Nota:** Siempre usar `2012-10-17`. Esta versión es requerida para:
- Condiciones con múltiples valores (`ForAnyValue`, `ForAllValues`)
- Variables de política como `aws:RequestTag`
- Permisos a nivel de recursos

---

## Effect

Solo dos valores posibles:

```json
"Effect": "Allow"    // Otorgar permiso
"Effect": "Deny"     // Denegar permiso
```

**Reglas importantes:**
- `Deny` override a `Allow`: Si un usuario tiene `Allow` en un recurso pero `Deny` explícito, prevalece `Deny`
- implícito `Deny`: Todo usuario nuevo empieza con permisos negados por defecto

## Action

### Formato

```
servicio:acción
```

Ejemplos:
```json
"Action": [
  "s3:GetObject",           // Acción individual
  "s3:List*",              // Con wildcard
  "ec2:DescribeInstances", // EC2 usa Describe* no Get*
  "iam:*AccessKey*"         // Wildcard en medio
]
```

### Categorías de Acciones

| Prefijo | Servicio |
|---------|----------|
| `s3:*` | Amazon S3 |
| `ec2:*` | Amazon EC2 |
| `iam:*` | AWS IAM |
| `kms:*` | AWS KMS |
| `cloudtrail:*` | AWS CloudTrail |
| `logs:*` | CloudWatch Logs |

### Acciones Comunes por Servicio

**S3:**
```json
"Action": [
  "s3:GetObject",           // Leer objetos
  "s3:PutObject",           // Subir objetos
  "s3:DeleteObject",        // Eliminar objetos
  "s3:ListBucket",          // Listar contenido
  "s3:GetBucketLocation"    // Obtener región del bucket
]
```

**EC2:**
```json
"Action": [
  "ec2:DescribeInstances",  // Ver instancias
  "ec2:StartInstances",      // Iniciar
  "ec2:StopInstances",       // Detener
  "ec2:TerminateInstances"   // Eliminar
]
```

## Resource

### Formato ARN

```
arn:aws:servicio:región:cuenta:recurso
```

### Ejemplos

```json
"Resource": "arn:aws:s3:::mi-bucket"                    // Bucket específico
"Resource": "arn:aws:s3:::mi-bucket/*"                  // Objetos en bucket
"Resource": "arn:aws:ec2:us-east-1:123456789012:instance/*"  // Todas las instancias en región
"Resource": "*"                                         // Todos los recursos
```

### Variables en ARNs

```json
"Resource": "arn:aws:s3:::mi-bucket/${aws:username}/*"
"Resource": "arn:aws:s3:::mi-bucket/${aws:PrincipalTag/department}/*"
```

## Condition

### Operadores de Condición

| Operador | Descripción | Ejemplo |
|----------|-------------|---------|
| `StringEquals` | Cadena exacta | `"aws:RequestedRegion": "us-east-1"` |
| `Bool` | Booleano | `"aws:MultiFactorAuthPresent": true` |
| `IpAddress` | Rango IP (CIDR) | `"aws:SourceIp": "192.168.1.0/24"` |
| `NotIpAddress` | IP fuera de rango | |
| `Null` | ¿Valor existe? | `"aws:PrincipalTag/team": {"false": ""}` |
| `Date` | Fecha/Hora | `"aws:CurrentTime": "2024-01-01T00:00:00Z"` |

### Condiciones Comunes

**Requerir MFA:**
```json
"Condition": {
  "Bool": {
    "aws:MultiFactorAuthPresent": false
  }
}
```

**Limitar por IP:**
```json
"Condition": {
  "IpAddress": {
    "aws:SourceIp": ["192.168.1.0/24", "10.0.0.0/16"]
  }
}
```

**Requerir tag específico:**
```json
"Condition": {
  "StringEquals": {
    "aws:RequestTag/Environment": "Production"
  }
}
```

**Restringir región:**
```json
"Condition": {
  "StringEquals": {
    "aws:RequestedRegion": ["us-east-1", "us-west-2"]
  }
}
```

**Tiempo de expiración (sesiones temporales):**
```json
"Condition": {
  "NumericLessThan": {
    "aws:TokenIssueTime": "2024-01-01T00:00:00Z"
  }
}
```

### Múltiples Condiciones

```json
"Condition": {
  "StringEquals": {
    "aws:RequestedRegion": "us-east-1",
    "aws:PrincipalTag/Department": "IT"
  },
  "IpAddress": {
    "aws:SourceIp": "192.168.1.0/24"
  }
}
```

## Statements Múltiples

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowReadS3",
      "Effect": "Allow",
      "Action": ["s3:GetObject", "s3:ListBucket"],
      "Resource": "arn:aws:s3:::mi-bucket"
    },
    {
      "Sid": "DenyDelete",
      "Effect": "Deny",
      "Action": "s3:DeleteObject",
      "Resource": "arn:aws:s3:::mi-bucket/*"
    }
  ]
}
```

## Ejemplo Completo: Policy con Múltiples Restricciones

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Sid": "AllowDevAccess",
      "Effect": "Allow",
      "Action": [
        "s3:GetObject",
        "s3:PutObject",
        "s3:ListBucket"
      ],
      "Resource": [
        "arn:aws:s3:::dev-bucket",
        "arn:aws:s3:::dev-bucket/*"
      ],
      "Condition": {
        "StringEquals": {
          "aws:RequestTag/Environment": "Development"
        }
      }
    },
    {
      "Sid": "DenyRootAccess",
      "Effect": "Deny",
      "Action": "*",
      "Resource": "*",
      "Condition": {
        "StringLike": {
          "aws:PrincipalARN": "arn:aws:iam::*:root"
        }
      }
    }
  ]
}
```

## Permission Boundaries

Limitan los permisos máximos que puede tener un usuario o rol.

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:*",
        "ec2:Describe*"
      ],
      "Resource": "*"
    }
  ]
}
```

**Uso típico:**
- Developer puede crear usuarios con máximo de permisos de lectura
- Prevenir que alguien se otorge a sí mismo permisos de admin

## Troubleshooting

### Error: "Access Denied"
1. Verificar que `Effect` es `Allow`
2. Verificar que `Resource` incluye el recurso objetivo
3. Verificar que `Action` incluye la acción deseada
4. Revisar si otra policy con `Deny` explícito está bloqueando

### Error: "Invalid principal"
- En roles: verificar el `Principal` en la trust policy
- En policies basadas en recursos: verificar formato ARN

### Policy no surte efecto
- IAM es global, pero resource policies pueden ser regionales
- Verificar que la policy está attached a usuario/grupo/rol

## Comandos AWS CLI Útiles

```bash
# Validar JSON de política
aws iam simulate-principal-policy \
  --policy-source-arn arn:aws:iam::123456789012:user/my-user \
  --action-names s3:GetObject \
  --resource-arns arn:aws:s3:::mi-bucket/file.txt

# Listar políticas attachadas a usuario
aws iam list-attached-user-policies --user-name my-user

# Generar policy desde accesos existentes
aws iam generate-service-last-accessed-details \
  --arn arn:aws:iam::123456789012:user/my-user
```
