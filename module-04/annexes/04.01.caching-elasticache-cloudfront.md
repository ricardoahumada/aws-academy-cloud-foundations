```
┌─────────────────────────────────────────────────────────────┐
│                 Caching Decision Matrix                     │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐    │
│  │  USE CACHE                                          │    │
│  │  ✅ Datos frecuentemente accedidos                  │    │
│  │  ✅ Datos que cambian raramente                     │    │
│  │  ✅ Latencia crítica                                │    │

│  ├─────────────────────────────────────────────────────┤    │
│  │  DON'T USE CACHE                                    │    │

│  │  ❌ Datos únicos/irreemplazables                    │    │
│  │  ❌ Datos sensibles (no cache banking transactions) │    │
│  └─────────────────────────────────────────────────────┘    │
└─────────────────────────────────────────────────────────────┘
```

## Amazon ElastiCache

Servicio fully managed para Redis o Memcached en memoria.

### Redis vs Memcached

| Característica | Redis | Memcached |
|----------------|-------|-----------|
| **Data structures** | Strings, Lists, Sets, Hashes, Sorted Sets | Key-value simple |
| **Persistence** | Opcional (RDB, AOF) | No persistence |
| **Replication** | Master-replica, clusters | No replication |
| **High availability** | Sí (Cluster mode,Replica) | No |
| **Pub/Sub** | Sí | No |
| **Transactions** | Sí (MULTI/EXEC) | No |
| **Sorted sets** | Sí | No |
| **Use case** | Cache avanzado, sessions, leaderboards | Simple caching, image cache |

### CLI: Crear ElastiCache Cluster

```bash
# 1. Crear subnet group
aws elasticache create-cache-subnet-group \
  --cache-subnet-group-name mi-subnet-group \
  --cache-subnet-group-description "Subnet group for ElastiCache" \
  --subnet-ids subnet-xxxx subnet-yyyy

# 2. Crear parameter group (configuración)
aws elasticache create-cache-parameter-group \
  --cache-parameter-group-name mi-redis-params \
  --cache-parameter-group-family redis7 \
  --description "Custom Redis parameters"

# 3. Crear Redis cluster
aws elasticache create-cache-cluster \
  --cluster-id mi-redis-cluster \
  --cache-node-type cache.t3.micro \
  --engine redis \
  --engine-version 7.0 \
  --num-cache-nodes 2 \
  --cache-subnet-group-name mi-subnet-group \
  --security-group-ids sg-xxxx \
  --preferred-availability-zones us-east-1a us-east-1b \
  --cache-parameter-group-name mi-redis-params

# 4. Para Memcached
aws elasticache create-cache-cluster \
  --cluster-id mi-memcached-cluster \
  --cache-node-type cache.t3.micro \
  --engine memcached \
  --engine-version 1.6.6 \
  --num-cache-nodes 3 \
  --cache-subnet-group-name mi-subnet-group \
  --security-group-ids sg-xxxx
```

## Patrones de Caching

### Cache-Aside (Lazy Loading)

La app verifica cache primero; si no existe, lee de DB y actualiza cache.

```python
def get_user(user_id):
    # Step 1: Check cache
    cache_key = f"user:{user_id}"
    user = redis.get(cache_key)
    
    if user:
        return json.loads(user)  # Cache HIT
    
    # Step 2: Cache MISS - get from DB
    user = db.query("SELECT * FROM users WHERE id = ?", user_id)
    
    if user:
        # Step 3: Store in cache with TTL
        redis.setex(cache_key, 3600, json.dumps(user))  # 1 hour
    
    return user
```

**Ventajas:** Solo cachea lo que se consulta, tolerancia a fallos de cache
**Desventajas:** Primer acceso siempre es lento (cache miss)

### Write-Through

Escribe en cache al mismo tiempo que en DB.

```python
def create_user(user_id, data):
    # Step 1: Write to DB
    db.execute("INSERT INTO users VALUES (?, ?)", user_id, data)
    
    # Step 2: Write to cache immediately
    cache_key = f"user:{user_id}"
    redis.setex(cache_key, 3600, json.dumps(data))
    
    return data
```

**Ventajas:** Datos siempre新鲜 en cache
**Desventajas:** Write más lento, puede cachear datos que nunca se leen

### Write-Behind

Escribe a cache asincrónicamente, DB se actualiza después.

```python
def update_user(user_id, data):
    # Step 1: Write to cache immediately
    cache_key = f"user:{user_id}"
    redis.setex(cache_key, 3600, json.dumps(data))
    
    # Step 2: Queue DB update for later
    sqs_queue.send(MessageBody={
        'action': 'UPDATE_USER',
        'user_id': user_id,
        'data': data
    })
```

## TTL (Time To Live)

```bash
# TTL recomendaciones:
# - Session data: 15-30 minutes
# - User profiles: 1-4 hours
# - Product catalog: 4-24 hours
# - Static content references: 24 hours+

# Redis: set with expiration
redis.setex("user:123", 3600, "user_data")  # 1 hour TTL
```

## Amazon CloudFront

CDN (Content Delivery Network) con 450+ edge locations globalmente.

### Conceptos

```
┌─────────────────────────────────────────────────────────────┐
│                     Amazon CloudFront                        │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Edge Locations (450+)                               │   │
│  │  • Cache static/dynamic content                     │   │
│  │  • TLS termination                                   │   │
│  │  • DDoS protection (Shield)                         │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  Origins                                             │   │
│  │  • S3 bucket (static content)                       │   │
│  │  • ALB/EC2 (dynamic content)                        │   │
│  │  • Custom origin (any HTTP server)                  │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### CLI: Crear CloudFront Distribution

```bash
# 1. Crear S3 origin (simple)
aws cloudfront create-distribution \
  --origin-domain-name mi-bucket.s3.amazonaws.com \
  --default-root-object index.html

# 2. Crear distribution completa via JSON
aws cloudfront create-distribution \
  --distribution-config '{
    "CallerReference": "my-dist-$(date +%s)",
    "Comment": "CloudFront for mi-app",
    "Enabled": true,
    "DefaultRootObject": "index.html",
    "Origins": {
      "Quantity": 1,
      "Items": [
        {
          "Id": "S3-mi-bucket",
          "DomainName": "mi-bucket.s3.amazonaws.com",
          "S3OriginConfig": {
            "OriginAccessIdentity": ""
          }
        }
      ]
    },
    "DefaultCacheBehavior": {
      "TargetOriginId": "S3-mi-bucket",
      "ViewerProtocolPolicy": "redirect-to-https",
      "AllowedMethods": {
        "Quantity": 2,
        "Items": ["GET", "HEAD"],
        "CachedMethods": {
          "Quantity": 2,
          "Items": ["GET", "HEAD"]
        }
      },
      "Compress": true,
      "CachePolicyId": "658327ea-f89d-4fab-a63e-50ba6b1dtt6bb"  # Managed-CachingOptimized
    }
  }'
```

### Multiple Origins

```bash

  --distribution-config '{
    "CallerReference": "multi-origin-$(date +%s)",

      "Quantity": 2,
      "Items": [
        {
          "Id": "ALB-web-app",
          "DomainName": "mi-alb.amazonaws.com",
          "CustomOriginConfig": {
            "HTTPPort": 80,
            "HTTPSPort": 443,
            "OriginProtocolPolicy": "https-only"
          }
        },
        {
          "Id": "S3-static",
          "DomainName": "mi-static.s3.amazonaws.com",
          "S3OriginConfig": {}
        }
      ]
    },
    "CacheBehaviors": {
      "Quantity": 2,
      "Items": [
        {
          "PathPattern": "/static/*",
          "TargetOriginId": "S3-static",
          "ViewerProtocolPolicy": "https-only",

        },
        {
```

### Cache Policies

| Policy | Uso |
|--------|-----|
| **CachingOptimized** | Static content (imágenes, CSS, JS) |
| **CachingOptimizedForUncompressed** | Uncompressed content |
| **CachingDisabled** | Dynamic content (API, personalized) |
| **Elemental-MediaStore** | Video streaming |

## Invalidations

Invalidar contenido cacheado (cuando actualizas en origin).

```bash
# Invalidar archivo específico
aws cloudfront create-invalidation \
  --distribution-id EDFDVBD6EXAMPLE \
  --paths "/images/logo.png"

# Invalidar todos los archivos
aws cloudfront create-invalidation \
  --distribution-id EDFDVBD6EXAMPLE \
  --paths "/*"

# Invalidar múltiples archivos
aws cloudfront create-invalidation \
  --distribution-id EDFDVBD6EXAMPLE \
  --paths "/images/*.jpg" "/css/*.css"
```

## Lambda@Edge

Ejecutar funciones Lambda en edge locations para request/response manipulation.

```bash
# Crear función Lambda (Node.js)
aws lambda create-function \
  --function-name cloudfront-redirect \
  --runtime nodejs18.x \
  --role arn:aws:iam::123456789012:role/lambda-role \
  --handler index.handler \
  --zip-file fileb://function.zip

# Agregar trigger de CloudFront
aws lambda add-permission \
  --function-name cloudfront-redirect \
  --statement-id "cloudfront-redirect" \
  --action "lambda:GetFunction" \
  --principal "edgelambda.amazonaws.com"

# Configurar en CloudFront (via console)
```

**Usos comunes:**
- A/B testing
- Geolocation redirects
- JWT validation at edge
- Custom error pages

## Troubleshooting

### Cache Miss Alto

```bash
# Ver命中率 (hit ratio) en CloudFront
aws cloudfront get-stats \
  --distribution-id EDFDVBD6EXAMPLE

# En ElastiCache: verificar memoria
aws elasticache list-metric-detections \
  --cache-cluster-id mi-redis-cluster

# Posibles causas:
# - TTL muy bajo
# - Cache keys diferentes (case sensitivity)
# - Cache warming insuficiente
```

### Redis Connection Issues

```bash
# Ver estado del cluster
aws elasticache describe-cache-clusters \
  --cache-cluster-id mi-redis-cluster \
  --show-cache-node-info

# Ver métricas en CloudWatch
aws cloudwatch get-metric-statistics \
  --namespace AWS/ElastiCache \
  --metric-name CurrConnections \
  --dimensions Name=CacheClusterId,Value=mi-redis-cluster \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-01T01:00:00Z \
  --period 300 \
  --statistics Average
```

### CloudFront No Actualiza Contenido

```bash
# 1. Verificar origin es accesible
curl -I https://www.mi-dominio.com/static/image.png

# 2. Forzar invalidation
aws cloudfront create-invalidation \
  --distribution-id EDFDVBD6EXAMPLE \
  --paths "/static/image.png"

# 3. Verificar Cache-Control headers
curl -I https://cloudfront-url/static/image.png
# Headers importantes:
# - Cache-Control: max-age=xxxx
# - ETag
# - Last-Modified
```
