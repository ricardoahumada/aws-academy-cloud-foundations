## Tipos de Record

### Records Comunes

| Type | Purpose | Ejemplo |
|------|---------|---------|
| **A** | IPv4 address | `198.51.100.1` |
| **AAAA** | IPv6 address | `2001:db8::1` |
| **CNAME** | Alias para otro dominio | `www.example.com → example.com` |
| **ALIAS** | AWS-specific, evita lookup | `www.example.com → ELB DNS` |
| **MX** | Mail servers | `10 mail.example.com` |
| **TXT** | Texto arbitrario | SPF records |
| **NS** | Name servers | Delegar zona |

### Diferencia CNAME vs ALIAS

```bash
# CNAME: Solo para non-root domains (www, api)
# ✅ www.example.com → www.example.org
# ❌ example.com → elb.amazonaws.com (NO válido)

# ALIAS: AWS-specific, para root domains y AWS resources
# ✅ example.com → mi-alb.amazonaws.com (CloudFront, ELB, S3)
```

## Políticas de Routing

| Política | Descripción | Caso de Uso |
|----------|-------------|-------------|
| **Simple** | Un solo record, sin lógica | Single server, testing |
| **Weighted** | Distribuye por porcentaje | A/B testing, blue/green |
| **Latency** | Basado en latencia | Multi-region optimization |
| **Failover** | Primary + secondary | Alta disponibilidad |
| **Geolocation** | Basado en ubicación | Content localization |
| **Geoproximity** | Traffic flow bias | Route traffic globally |
| **IP-based** | Basado en CIDR de cliente | Custom routing rules |

## Simple Routing

Único record, sin condiciones. Para testing o single server.

```bash
# Crear record simple
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "test.mi-dominio.com",
        "Type": "A",
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.1"}]
      }
    }]
  }'
```

**Limitaciones:**
- No health checks
- No múltiples valores (devuelve todos, cliente elige)
- No geolocation

## Weighted Routing

Distribuye tráfico según porcentaje definido.

```bash
# Record 1: 80% tráfico a v1
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "ab.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "version-1",
        "Weight": 80,
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.1"}]
      }
    }]
  }'

# Record 2: 20% tráfico a v2
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "ab.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "version-2",
        "Weight": 20,
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.2"}]
      }
    }]
  }'
```

**Casos de uso:**
- A/B testing
- Blue/Green deployments
- Canary releases
- Testing en producción (% limitado)

## Latency-Based Routing

Redirige al endpoint con menor latencia para el usuario.

```bash
# US East (N. Virginia)
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "latency.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "us-east-1",
        "Region": "us-east-1",
        "LatencyRoutingPolicy": {},
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.1"}]
      }
    }]
  }'

# EU West (Ireland)
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "latency.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "eu-west-1",
        "Region": "eu-west-1",
        "LatencyRoutingPolicy": {},
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.2"}]
      }
    }]
  }'
```

**Cómo funciona:**
1. Route 53 mide latencia desde usuario a cada región
2. Elige el endpoint con menor latencia
3. Si latencias similares, puede elegir cualquiera

## Failover Routing

Active-Passive: Primary responde si está healthy, secondary es backup.

### Crear Health Check

```bash
# Health check para primary
aws route53 create-health-check \
  --caller-reference "hc-primary-$(date +%s)" \
  --health-check-config '{
    "Type": "HTTPS",
    "FullyQualifiedDomainName": "www.mi-dominio.com",
    "Port": 443,
    "ResourcePath": "/health",
    "RequestInterval": 10,
    "FailureThreshold": 3
  }'
```

### Crear Failover Records

```bash
# Record primario (Primary)
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "www.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "primary-alb",
        "Failover": "PRIMARY",
        "TTL": 60,
        "ResourceRecords": [{"Value": "alb-dns-name.amazonaws.com"}],
        "HealthCheckId": "xxxxx-health-check-id"
      }
    }]
  }'

# Record secundario (Secondary) - siempre healthy
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "www.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "secondary-s3",
        "Failover": "SECONDARY",
        "TTL": 60,
        "ResourceRecords": [{"Value": "s3-website-us-east-1.amazonaws.com"}]
      }
    }]
  }'
```

**Flujo de failover:**
```
User Request
     │
     ▼
Route 53 Health Check
     │
     ├─── Healthy ──▶ Return Primary (ALB)
     │
     └─── Unhealthy ──▶ Return Secondary (S3 Static)
```

## Geolocation Routing

Redirige basado en ubicación geográfica del cliente.

```bash
# Record para Estados Unidos
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "geo.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "us-users",
        "GeoLocation": {"CountryCode": "US"},
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.1"}]
      }
    }]
  }'

# Record para Europa
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "geo.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "eu-users",
        "GeoLocation": {"CountryCode": "DE"},
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.2"}]
      }
    }]
  }'

# Record default (cuando no hay coincidencia)
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "geo.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "default",
        "GeoLocation": {"CountryCode": "*"},
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.3"}]
      }
    }]
  }'
```

**Jerarquía de Geolocation:**
1. Continente (North America, Europe, etc.)
2. País (US, MX, BR)
3. Estado/Provincia (solo US)

## Geoproximity Routing

Routing basado en ubicación geográfica con bias.

```bash
# Requiere Route 53 Traffic Flow y Traffic Policies
# CLI no soporta crear directamente
# Usar AWS Console o Traffic Policies
```

**Bias de tráfico:**
- Negativo (-50 a -1): Repeler tráfico del recurso
- Positivo (1 a 50): Atraer tráfico al recurso
- 0: Sin bias

## IP-Based Routing

Routing basado en rango CIDR del cliente.

```bash
# Routing basado en IPs de cliente conocidas
aws route53 change-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --change-batch '{
    "Changes": [{
      "Action": "CREATE",
      "ResourceRecordSet": {
        "Name": "office.mi-dominio.com",
        "Type": "A",
        "SetIdentifier": "corp-office",
        "Location": {
          "CountryCode": "*",
          "CIDR": "203.0.113.0/24"
        },
        "TTL": 300,
        "ResourceRecords": [{"Value": "198.51.100.10"}]
      }
    }]
  }'
```

## Hosted Zones

### Crear Hosted Zone

```bash
# Public hosted zone
aws route53 create-hosted-zone \
  --name mi-dominio.com \
  --caller-reference "hz-$(date +%s)" \
  --hosted-zone-config '{
    "Comment": "Hosted zone for mi-dominio.com",
    "PrivateZone": false
  }'

# Private hosted zone (VPC internal)
aws route53 create-hosted-zone \
  --name internal.mi-dominio.com \
  --caller-reference "hz-internal-$(date +%s)" \
  --hosted-zone-config '{"PrivateZone": true}' \
  --vpc '{"VPCRegion": "us-east-1", "VPCId": "vpc-xxxx"}'
```

### Listar y Ver Records

```bash
# Listar hosted zones
aws route53 list-hosted-zones

# Listar records de una zona
aws route53 list-resource-record-sets \
  --hosted-zone-id ZXXXXX

# Ver solo A records
aws route53 list-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --query 'ResourceRecordSets[?Type==`A`]'
```

## TTL (Time To Live)

```bash
# TTL bajo: Cambios rápidos (~60 seconds)
# TTL alto: Menos queries DNS (~86400 seconds = 24h)

# TTL para diferentes scenarios:
# - ALB/ELB: 60 segundos (IPs pueden cambiar)
# - Static content: 86400 segundos
# - Services críticos: 300 segundos
# - Testing: 30 segundos
```

## Troubleshooting

### DNS no resuelve

```bash
# Verificar que el record existe
aws route53 list-resource-record-sets \
  --hosted-zone-id ZXXXXX \
  --query 'ResourceRecordSets[?Name==`www.mi-dominio.com`]'

# Probar con dig/nslookup
nslookup www.mi-dominio.com
dig www.mi-dominio.com

# Verificar propagated
aws route53 get-change --id XXXXX
# Status: INSYNC cuando está propagado
```

### Health check falla

```bash
# Ver estado del health check
aws route53 get-health-check \
  --health-check-id xxxxx

# Ver último status
aws route53 get-health-check-status \
  --health-check-id xxxxx
```

### Latency routing no funciona

- Verificar que los recursos existen y están accesibles
- Route 53 requiere que al menos un endpoint tenga latencia medida
- Probar con `dig +subnet=<client_ip>` para simular diferentes ubicaciones
