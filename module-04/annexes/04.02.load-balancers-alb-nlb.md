# Load Balancers - ALB vs NLB vs CLB

> **Tiempo de lectura:** ~8 min | **Referencia:** [AWS Load Balancer Guide](https://docs.aws.amazon.com/elasticloadbalancing/latest/application/)

---

## Tipos de Balanceadores

AWS ofrece tres tipos de balanceadores de carga:

```
┌─────────────────────────────────────────────────────────────┐
│                  AWS Load Balancers                          │
│                                                             │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────┐        │
│  │     ALB      │  │     NLB      │  │     CLB      │        │
│  │ Application  │  │   Network    │  │   Classic    │        │
│  │   Layer 7    │  │   Layer 4    │  │  Layer 4/7   │        │
│  └─────────────┘  └─────────────┘  └─────────────┘        │
│        │                │                  │                │
│    HTTP/HTTPS         TCP/UDP           HTTP/TCP           │
└─────────────────────────────────────────────────────────────┘
```

| Característica | ALB | NLB | CLB |
|----------------|-----|-----|-----|
| **Capa OSI** | 7 (Application) | 4 (Transport) | 4 y 7 |
| **Protocolos** | HTTP, HTTPS | TCP, UDP, TLS | HTTP, TCP, SSL |
| **Routing avanzado** | Sí (path, host, query) | No | No |
| **SSL Termination** | Sí | Sí | Sí |
| **Static IP** | No | Sí | No |
| **Zona única** | No | Sí | Sí |
| **Integración with WAF** | Sí | No | No |
| **Target Groups** | Sí | Sí | No |
| **Use case principal** | Web apps, microservices | High perf, gaming, IoT | Legacy, migration |

## ALB (Application Load Balancer)

### Conceptos Clave

- **Layer 7:** Puede inspeccionar contenido HTTP/HTTPS
- **Content-based routing:** Direcciona basado en URL path, host header
- **Target Groups:** Agrupa targets (EC2, Lambda, IP)
- **Rules:** Evalúa en orden para direccionar tráfico

### Componentes

```
ALB
├── Listeners (port, protocol, rules)
│   └── Rules (condition → action)
│       ├── Rule 1: /api/* → Target Group: tg-api
│       ├── Rule 2: /static/* → Target Group: tg-static
│       └── Default: → Target Group: tg-default
└── Target Groups
    └── Targets (instances, IPs, Lambda)
```

### CLI: Crear ALB

```bash
# 1. Crear Security Group para ALB
aws ec2 create-security-group \
  --group-name alb-sg \
  --description "Security group for ALB"

aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxx \
  --protocol http \
  --port 80 \
  --cidr 0.0.0.0/0

# 2. Crear ALB
aws elbv2 create-load-balancer \
  --name mi-alb \
  --type application \
  --scheme internet-facing \
  --ip-address-type ipv4 \
  --subnets subnet-xxxx subnet-yyyy subnet-zzzz \
  --security-groups sg-xxxx

# 3. Crear Target Group
aws elbv2 create-target-group \
  --name tg-web-servers \
  --protocol HTTP \
  --port 80 \
  --target-type instance \
  --vpc-id vpc-xxxx \
  --health-check-path /index.html \
  --health-check-interval-seconds 30 \
  --healthy-threshold-count 2 \
  --unhealthy-threshold-count 3

# 4. Registrar targets (instancias)
aws elbv2 register-targets \
  --target-group-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web-servers/xxxx \
  --targets Id=i-xxxxx Id=i-yyyyy

# 5. Crear Listener (HTTP :80 → forward a TG)
aws elbv2 create-listener \
  --load-balancer-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:loadbalancer/app/mi-alb/xxxx \
  --protocol HTTP \
  --port 80 \
  --default-actions Type=forward,TargetGroupArn=arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web-servers/xxxx
```

### Path-Based Routing

```bash
# Crear rule para path específico
aws elbv2 create-rule \
  --listener-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:listener/app/mi-alb/xxxx/yyyy \
  --priority 100 \
  --conditions Field=path-pattern,Values="/api/*","/health*" \
  --actions Type=forward,TargetGroupArn=arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-api/zzzz
```

## NLB (Network Load Balancer)

### Conceptos Clave

- **Layer 4:** Balancea TCP/UDP/TLS
- **Ultra-high performance:** Millones de requests/segundo
- **Static IP:** Direcciones IP fijas por zona
- **Preserva source IP:** Para aplicaciones que requieren IP original

### Casos de Uso

- Gaming / UDP
- IoT / MQTT
- High-performance computing
- VoIP
- Banking / Financial services

### CLI: Crear NLB

```bash
# 1. Crear NLB
aws elbv2 create-load-balancer \
  --name mi-nlb \
  --type network \
  --scheme internet-facing \
  --ip-address-type ipv4 \
  --subnets subnet-xxxx subnet-yyyy

# 2. Crear Target Group (tipo IP para NLB)
aws elbv2 create-target-group \
  --name tg-game-servers \
  --protocol TCP \
  --port 7777 \
  --target-type ip \
  --vpc-id vpc-xxxx \
  --health-check-protocol TCP \
  --health-check-interval-seconds 10 \
  --healthy-threshold-count 2

# 3. Registrar por IP (no por instance ID)
aws elbv2 register-targets \
  --target-group-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-game-servers/xxxx \
  --targets Id=10.0.1.100 Id=10.0.2.100

# 4. Crear Listener
aws elbv2 create-listener \
  --load-balancer-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:loadbalancer/net/mi-nlb/xxxx \
  --protocol TCP \
  --port 7777 \
  --default-actions Type=forward,TargetGroupArn=arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-game-servers/xxxx
```

### Obtener Static IP

```bash
# NLB proporciona IPs estáticas por zona
aws elbv2 describe-load-balancers \
  --names mi-nlb \
  --query 'LoadBalancers[].AvailabilityZones'

# Las IPs pueden ser usadas en:
# - Whitelisting en Security Groups
# - DNS con failover
```

## SSL/TLS en ALB

### Configurar HTTPS Listener

```bash
# 1. Obtener certificado de ACM
aws acm list-certificates \
  --query 'CertificateSummaryList[?DomainName==`mi-dominio.com`].CertificateArn'

# 2. Crear HTTPS listener
aws elbv2 create-listener \
  --load-balancer-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:loadbalancer/app/mi-alb/xxxx \
  --protocol HTTPS \
  --port 443 \
  --certificates CertificateArn=arn:aws:acm:us-east-1:123456789012:certificate/xxxx \
  --ssl-policy ELBSecurityPolicy-2016-08 \
  --default-actions Type=forward,TargetGroupArn=arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web/yyyy

# 3. Redirigir HTTP → HTTPS
aws elbv2 create-listener \
  --load-balancer-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:loadbalancer/app/mi-alb/xxxx \
  --protocol HTTP \
  --port 80 \
  --default-actions Type=redirect,RedirectConfig='{Protocol=HTTPS,Port=443,StatusCode=HTTP_301}'
```

## Health Checks

### Configuración

```bash
# Modificar health check de un target group
aws elbv2 modify-target-group \
  --target-group-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web/xxxx \
  --health-check-interval-seconds 30 \
  --health-check-path /health \
  --healthy-threshold-count 2 \
  --unhealthy-threshold-count 3 \
  --timeout-seconds 5 \
  --matcher HttpCode=200
```

### Troubleshooting Health Checks

```bash
# Ver estado de targets
aws elbv2 describe-target-health \
  --target-group-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web/xxxx

# Estados posibles:
# - initial: Registrando
# - healthy: Respondiendo correctamente
# - unhealthy: Fallando health checks
# - unused: No pertenece al TG
# - draining: Deregistrando
```

## Auto Scaling con ALB

```bash
# Crear ASG con ALB integration
aws autoscaling create-auto-scaling-group \
  --auto-scaling-group-name asg-web-tier \
  --launch-template LaunchTemplateId=lt-xxxx,Version=1 \
  --min-size 2 \
  --max-size 6 \
  --desired-capacity 2 \
  --vpc-zone-identifier "subnet-xxxx,subnet-yyyy,subnet-zzzz" \
  --target-group-arns arn:aws:elasticloadbalancing:us-east-1:123456789012:targetgroup/tg-web/xxxx \
  --health-check-type ELB \
  --health-check-grace-period 60 \
  --instance-maintenance-policy MinHealthyPercentage=100
```

## Comandos Útiles

```bash
# Listar todos los load balancers
aws elbv2 describe-load-balancers

# Ver descripción de ALB
aws elbv2 describe-load-balancers \
  --names mi-alb \
  --query 'LoadBalancers[0]'

# Obtener DNS name
aws elbv2 describe-load-balancers \
  --names mi-alb \
  --query 'LoadBalancers[0].DNSName' \
  --output text

# Ver reglas de listener
aws elbv2 describe-rules \
  --listener-arn arn:aws:elasticloadbalancing:us-east-1:123456789012:listener/app/mi-alb/xxxx/yyyy

# Ver métricas
aws cloudwatch get-metric-statistics \
  --namespace AWS/ApplicationELB \
  --metric-name TargetResponseTime \
  --dimensions Name=LoadBalancer,Value=app/mi-alb/xxxx \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-01T01:00:00Z \
  --period 300 \
  --statistics Average
```
