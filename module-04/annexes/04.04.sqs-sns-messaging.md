## Comparativa SQS vs SNS

```
┌─────────────────────────────────────────────────────────────┐
│              Messaging Patterns                             │
│                                                             │
│  POINT-TO-POINT (SQS)          PUB/SUB (SNS)                │
│  ┌──────────┐                   ┌───────────┐               │
│  │ Producer │ ──── Queue ────▶  │ Consumer1 │               │
│  └──────────┘                   └───────────┘               │
│       │                          ┌──────────┐               │
│       │                          │ Consumer2│               │
│       │                          └──────────┘               │
│       │                          ┌───────────┐              │
│       └─────────────────────────▶│ Consumer3 │              │
│                                  └───────────┘              │
└─────────────────────────────────────────────────────────────┘
```

| Aspecto | SQS | SNS |
|---------|-----|-----|
| **Pattern** | Point-to-point (queue) | Pub/Sub (topic) |
| **Consumers** | Single consumer (por mensaje) | Múltiples subscribers |
| **Delivery** | Polling (consumer pide) | Push ( SNS envía) |
| **Use case** | Task queues, async jobs | Notifications, fan-out |
| **Message lifecycle** | Consumer/delete | Fire-and-forget |
| **Ordering** | Standard: no, FIFO: sí | No guarantee |

## Amazon SQS

### Tipos de Colas

| Característica | Standard | FIFO |
|----------------|----------|------|
| **Throughput** | Unlimited | 300 msg/sec (batch: 3000) |
| **Ordering** | Best-effort | Exactly-once, ordered |
| **Delivery** | At-least-once | Exactly-once |
| **Duplicates** | Possible | Deduplicated (5 min) |
| **Price** | Pay per request | Pay per request (+50%) |

### CLI: Crear y Usar Colas

```bash
# 1. Crear Standard Queue
aws sqs create-queue \
  --queue-name mi-cola-estandar \
  --attributes '{"VisibilityTimeout": "30", "MessageRetentionPeriod": "345600"}'

# 2. Crear FIFO Queue
aws sqs create-queue \
  --queue-name ordenes-procesamiento.fifo \
  --attributes '{
    "FifoQueue": "true",
    "ContentBasedDeduplication": "true",
    "VisibilityTimeout": "30",
    "MessageRetentionPeriod": "86400"
  }'

# 3. Listar colas
aws sqs list-queues

# 4. Obtener URL de cola
aws sqs get-queue-url \
  --queue-name mi-cola-estandar

# Output: https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar
```

### Enviar Mensajes

```bash
# Enviar mensaje simple
aws sqs send-message \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --message-body '{"order_id": "123", "amount": 99.99}'

# Enviar con delay
aws sqs send-message \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --message-body '{"task": "process"}' \
  --delay-seconds 60

# Enviar batch (hasta 10 mensajes)
aws sqs send-message-batch \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --entries '[{"Id": "1", "MessageBody": "msg1"}, {"Id": "2", "MessageBody": "msg2"}]'
```

### Recibir y Procesar Mensajes

```bash
# Recibir mensajes (polling)
aws sqs receive-message \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --max-number-of-messages 10 \
  --wait-time-seconds 20

# Respuesta ejemplo:
# {
#   "Messages": [
#     {
#       "MessageId": "uuid-xxxx",
#       "ReceiptHandle": "AQEB...",
#       "MD5OfBody": "...",
#       "Body": "{\"order_id\": \"123\"}",
#       "Attributes": {"ApproximateReceiveCount": "1"}
#     }
#   ]
# }
```

### Eliminar Mensajes

```bash
# Eliminar mensaje específico (requiere ReceiptHandle)
aws sqs delete-message \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --receipt-handle "AQEB..."

# Eliminar múltiples mensajes
aws sqs delete-message-batch \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-estandar \
  --entries '[{"Id": "1", "ReceiptHandle": "handle1"}, {"Id": "2", "ReceiptHandle": "handle2"}]'
```

## Dead Letter Queue (DLQ)

Cola para mensajes que no pueden ser procesados después de varios intentos.

```bash
# 1. Crear DLQ
aws sqs create-queue \
  --queue-name mi-cola-dlq.fifo \
  --attributes '{"FifoQueue": "true"}'

# 2. Configurar cola principal con DLQ
aws sqs set-queue-attributes \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola-principal.fifo \
  --attributes '{
    "RedrivePolicy": "{\"deadLetterTargetArn\": \"arn:aws:sqs:us-east-1:123456789012:mi-cola-dlq.fifo\", \"maxReceiveCount\": 3}"
  }'
```

## Amazon SNS

### Conceptos

```
┌─────────────────────────────────────────────────────────────┐
│                    Amazon SNS                               │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Topic                                                 │   │
│  │  • Channel para publicar mensajes                    │   │
│  │  • Múltiples subscribers                            │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  Publishers                                          │   │
│  │  • Any AWS service, app, Lambda                      │   │
│  ├─────────────────────────────────────────────────────┤   │
│  │  Subscribers                                         │   │
│  │  • SQS, Lambda, HTTP, Email, SMS, Push              │   │
│  └─────────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────────┘
```

### CLI: Crear y Usar SNS

```bash
# 1. Crear Topic
aws sns create-topic \
  --name mi-topic \
  --attributes '{"DisplayName": "Mi App Notifications"}'

# 2. Crear FIFO Topic
aws sns create-topic \
  --name ordenes-topic.fifo \
  --attributes '{
    "FifoTopic": "true",
    "ContentBasedDeduplication": "true"
  }'

# 3. Listar topics
aws sns list-topics

# 4. Subscribe a SQS queue
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic \
  --protocol sqs \
  --notification-endpoint arn:aws:sqs:us-east-1:123456789012:mi-cola-sqs

# 5. Subscribe a Lambda
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic \
  --protocol lambda \
  --notification-endpoint arn:aws:lambda:us-east-1:123456789012:function:mi-funcion

# 6. Subscribe a HTTP/HTTPS
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic \
  --protocol https \
  --notification-endpoint https://api.mi-dominio.com/webhook
```

### Publicar Mensajes

```bash
# Publicar mensaje simple
aws sns publish \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic \
  --message '{"user_id": "123", "action": "order_placed"}'

# Publicar con atributos (para filtrado)
aws sns publish \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic \
  --message '{"order_id": "456"}' \
  --message-attributes '{
    "priority": {"DataType": "String", "StringValue": "high"},
    "region": {"DataType": "String", "StringValue": "us-east-1"}
  }'

# Publicar a FIFO topic
aws sns publish \
  --topic-arn arn:aws:sns:us-east-1:123456789012:ordenes-topic.fifo \
  --message '{"order_id": "789"}' \
  --message-group-id "orders"
```

## SNS + SQS: Fan-Out Pattern

Un publisher, múltiples colas (subscribers).

```bash
# Crear estructura fan-out
# Topic: "orders" → Queue1 (fulfillment), Queue2 (analytics), Queue3 (notifications)

# 1. Crear topic
aws sns create-topic --name orders-fanout

# 2. Crear 3 colas
aws sqs create-queue --queue-name fulfillment-queue
aws sqs create-queue --queue-name analytics-queue
aws sqs create-queue --queue-name notifications-queue

# 3. Suscribir colas al topic
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:orders-fanout \
  --protocol sqs \
  --notification-endpoint arn:aws:sqs:us-east-1:123456789012:fulfillment-queue

aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:orders-fanout \
  --protocol sqs \
  --notification-endpoint arn:aws:sqs:us-east-1:123456789012:analytics-queue

aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:orders-fanout \
  --protocol sqs \
  --notification-endpoint arn:aws:sqs:us-east-1:123456789012:notifications-queue

# 4. Publicar → todas las colas reciben
aws sns publish \
  --topic-arn arn:aws:sns:us-east-1:123456789012:orders-fanout \
  --message '{"order_id": "123"}'
```

## Message Filtering

SNS puede filtrar mensajes a subscribers basándose en message attributes.

```bash
# 1. Configurar topic con filter policy
aws sns create-topic --name filtered-orders

# 2. Subscribe con filter policy
aws sns subscribe \
  --topic-arn arn:aws:sns:us-east-1:123456789012:filtered-orders \
  --protocol sqs \
  --notification-endpoint arn:aws:sqs:us-east-1:123456789012:us-orders-queue

# 3. Set filter policy para subscription
aws sns set-subscription-attributes \
  --subscription-arn arn:aws:sns:us-east-1:123456789012:filtered-orders:subscription-id \
  --attribute-name FilterPolicy \
  --attribute-value '{"region": ["us-east-1", "us-west-2"]}'

# Ejemplo: Solo orders de US van a esta cola
# Orders de EU van a otra cola con filter {"region": ["eu-west-1"]}
```

## SQS + Lambda Integration

Procesar mensajes automáticamente con Lambda trigger.

```bash
# 1. Crear Lambda
aws lambda create-function \
  --function-name process-order \
  --runtime python3.11 \
  --role arn:aws:iam::123456789012:role/lambda-sqs-role \
  --handler process_order.handler \
  --zip-file fileb://function.zip

# 2. Agregar trigger SQS
aws lambda create-event-source-mapping \
  --function-name process-order \
  --event-source-arn arn:aws:sqs:us-east-1:123456789012:mi-cola \
  --batch-size 10 \
  --scaling-config '{"MaximumBatchSize": 10, "ConcurrencyLimit": 5}'
```

**Lambda handler ejemplo:**
```python
import json

def handler(event, context):
    for record in event['Records']:
        message = json.loads(record['body'])
        order_id = message['order_id']
        # Process order...
        print(f"Processing order: {order_id}")
    
    return {'statusCode': 200}
```

## SNS + Lambda Integration

Trigger Lambda cuando llega mensaje a topic.

```bash
# Lambda ya está subscribe al topic (creado anteriormente)

# Lambda handler
def lambda_handler(event, context):
    for record in event['Records']:
        message = json.loads(record['Sns']['Message'])
        subject = record['Sns']['Subject']
        
        print(f"Subject: {subject}")
        print(f"Message: {message}")
    
    return {'statusCode': 200}
```

## Comparativa con Amazon EventBridge

| Aspecto | SNS | EventBridge |
|---------|-----|-------------|
| **Target type** | Specific endpoints | Multiple targets via rules |
| **Schema registry** | No | Sí |
| **SaaS integration** | No | Sí |
| **Event replay** | No | Sí (archival) |
| **Rules engine** | Basic (filter) | Advanced (content, JSON) |
| **Cost** | Pay per message | Pay per event + rule evaluation |

## Troubleshooting

### SQS: Mensaje no se procesa

```bash
# 1. Ver si hay mensajes en cola
aws sqs get-queue-attributes \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola \
  --attribute-names ApproximateNumberOfMessages

# 2. Ver Age of oldest message
aws cloudwatch get-metric-statistics \
  --namespace AWS/SQS \
  --metric-name ApproximateAgeOfOldestMessage \
  --dimensions Name=QueueName,Value=mi-cola \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-01T01:00:00Z \
  --period 300 \
  --statistics Maximum

# Posibles causas:
# - VisibilityTimeout muy bajo (procesamiento toma mucho)
# - MaxReceiveCount alcanzado (va a DLQ)
# - Lambda errors (ver CloudWatch logs)
```

### SNS: Mensaje no se entrega

```bash
# 1. Ver CloudWatch metrics para topic
aws cloudwatch get-metric-statistics \
  --namespace AWS/SNS \
  --metric-name NumberOfMessagesPublished \
  --dimensions Name=TopicName,Value=mi-topic \
  --start-time 2024-01-01T00:00:00Z \
  --end-time 2024-01-01T01:00:00Z \
  --period 300 \
  --statistics Sum

# 2. Ver delivery status (si está configurado)
aws sns get-topic-attributes \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic

# Posibles causas:
# - Subscription no confirmada (HTTP/S)
# - Topic policy deniega
# - DLQ configurado y lleno
```

## Comandos Relacionados

```bash
# SQS: Purge queue (eliminar todos mensajes)
aws sqs purge-queue \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola

# SQS: Set queue attributes
aws sqs set-queue-attributes \
  --queue-url https://sqs.us-east-1.amazonaws.com/123456789012/mi-cola \
  --attributes '{"VisibilityTimeout": "60", "ReceiveMessageWaitTimeSeconds": "20"}'

# SNS: Delete topic
aws sns delete-topic \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic

# SNS: List subscriptions
aws sns list-subscriptions-by-topic \
  --topic-arn arn:aws:sns:us-east-1:123456789012:mi-topic
```
