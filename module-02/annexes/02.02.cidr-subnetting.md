# CIDR y Subnetting - Guía Rápida

> **Tiempo de lectura:** ~8 min | **Referencia:** [AWS VPC Documentation](https://docs.aws.amazon.com/vpc/latest/userguide/VPC_Subnets.html)

## ¿Qué es CIDR?

CIDR (Classless Inter-Domain Routing) es un método para designar rangos de direcciones IP sin usar las clases tradicionales (A, B, C).

**Sintaxis:** `IP_BASE/MÁSCARA`
```
10.0.0.0/16
│└── IP base
     └── Máscara: cuántos bits definen la red
```

### Nomenclatura

| Sufijo | Bits de red | Direcciones totales | Hosts usable* |
|--------|-------------|---------------------|---------------|
| `/24` | 24 | 256 | 254 |
| `/25` | 25 | 128 | 126 |
| `/26` | 26 | 64 | 62 |
| `/27` | 27 | 32 | 30 |
| `/28` | 28 | 16 | 14 |
| `/29` | 29 | 8 | 6 |
| `/30` | 30 | 4 | 2 |
| `/31` | 31 | 2 | 0 (special) |
| `/32` | 32 | 1 | 1 |

**\*En AWS, las primeras 4 IPs y la última de cada subnet están reservadas.**

### Equivalencia Decimal

| Sufijo | Máscara decimal |
|--------|-----------------|
| `/24` | 255.255.255.0 |
| `/25` | 255.255.255.128 |
| `/26` | 255.255.255.192 |
| `/27` | 255.255.255.224 |
| `/28` | 255.255.255.240 |
| `/29` | 255.255.255.248 |

## Ejemplo Práctico en AWS

```
VPC CIDR: 10.0.0.0/16  (65,536 direcciones: 10.0.0.0 - 10.0.255.255)

Subnet Pública:  10.0.1.0/24
  Rango: 10.0.1.0 - 10.0.1.255 (256 direcciones)
  Usable: 10.0.1.4 - 10.0.1.254 (251 direcciones)

Subnet Privada:  10.0.2.0/24
  Rango: 10.0.2.0 - 10.0.2.255 (256 direcciones)
  Usable: 10.0.2.4 - 10.0.2.254 (251 direcciones)
```

### Cómo Calcular el Rango

```
IP: 10.0.1.0/24

- Red: 10.0.1.0 (los primeros 24 bits fijo)
- Broadcast: 10.0.1.255 (última IP de la subnet)
- Primera usable: 10.0.1.1 (normalmente la gateway)
- Última usable: 10.0.1.254
```

## Bloques de IP Privada (RFC 1918)

AWS recomienda usar estos rangos para VPCs:

| Clase | Rango público | CIDR | Tamaño típico VPC |
|-------|---------------|------|-------------------|
| A | 10.0.0.0 - 10.255.255.255 | 10.0.0.0/8 | Muy grande |
| B | 172.16.0.0 - 172.31.255.255 | 172.16.0.0/12 | Grande |
| C | 192.168.0.0 - 192.168.255.255 | 192.168.0.0/16 | Mediano |

**En AWS se usan típicamente:** `10.0.0.0/16` para la VPC principal.

---

## Reglas para VPCs en AWS

1. **Tamaño mínimo:** `/28` (16 direcciones)
2. **Tamaño máximo:** `/16` (65,536 direcciones)
3. **CIDRs prohibidos:** 169.254.0.0/16 (link-local), 224.0.0.0/4 (multicast)
4. **Peering:** Los CIDRs de dos VPCs no deben solaparse

## Comandos AWS CLI para Verificar

```bash
# Verificar CIDR de una VPC
aws ec2 describe-vpcs --vpc-ids vpc-xxxxx --query 'Vpcs[0].CidrBlock'

# Listar subnets con sus CIDRs
aws ec2 describe-subnets --query 'Subnets[*].{ID:SubnetId,CIDR:CidrBlock,VPC:VpcId}'

# Verificar si CIDRs se solapan (para Peering)
aws ec2 describe-vpc-peering-connections
```
