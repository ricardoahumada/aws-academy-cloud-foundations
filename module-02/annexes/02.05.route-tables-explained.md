# Route Tables - Guía de Referencia

> **Tiempo de lectura:** ~7 min | **Referencia:** [AWS Route Tables Documentation](https://docs.aws.amazon.com/vpc/latest/userguide/VPC_Route_Tables.html)

---

## Concepto Básico

Una Route Table define **a dónde van los paquetes de red** cuando salen de una subnet.

```
┌─────────────┐      ┌─────────────────┐      ┌─────────────┐
│   Instances │ ───► │  Route Table     │ ───► │ Destination │
│   enSubnet │      │  de la Subnet    │      │ (IGW, NAT,  │
└─────────────┘      └─────────────────┘      │ Local, etc)  │
                                             └─────────────┘
```

**Cada subnet DEBE tener asociada exactamente una Route Table.**

---

## Estructura de una Ruta

| Campo | Descripción | Ejemplo |
|-------|-------------|---------|
| **Destination** |IP o CIDR de destino | `0.0.0.0/0` (todo) o `10.0.0.0/16` (red local) |
| **Target** | Donde enviar el tráfico | `igw-xxxx`, `nat-xxxx`, `local` |

---

## Destinos Comunes en AWS

| Destination | Target | Cuándo usarlo |
|-------------|--------|---------------|
| `10.0.0.0/16` | `local` | Tráfico entre subnets de la misma VPC |
| `0.0.0.0/0` | `igw-xxxx` | Subnet pública que necesita Internet |
| `0.0.0.0/0` | `nat-xxxx` | Subnet privada que sale a Internet |
| `10.1.0.0/16` | `pcx-xxxx` | Tráfico a VPC en Peering |
| `172.16.0.0/16` | `tgw-xxxx` | Tráfico via Transit Gateway |

---

## Ejemplo: Subnet Pública

```
VPC: 10.0.0.0/16

Destination       Target         Uso
─────────────────────────────────────────────────────
10.0.0.0/16       local          Tráfico interno de la VPC
0.0.0.0/0         igw-xxxxxxxxx  Todo lo demás → Internet
```

**Resultado:** Instancias en esta subnet pueden:
- Comunicarse entre sí (10.0.x.x)
- Acceder a Internet (0.0.0.0/0 → IGW)

---

## Ejemplo: Subnet Privada

```
VPC: 10.0.0.0/16

Destination       Target         Uso
─────────────────────────────────────────────────────
10.0.0.0/16       local          Tráfico interno de la VPC
0.0.0.0/0         nat-xxxxxxxxx  Todo lo demás → NAT Gateway
```

**Resultado:** Instancias en esta subnet pueden:
- Comunicarse entre sí (10.0.x.x)
- Acceder a Internet SOLO a través del NAT Gateway
- **NO recibir conexiones entrantes desde Internet**

## Cómo AWS Evalúa las Rutas

**Regla:** AWS usa la ruta más específica que coincida (Longest Prefix Match).

```
Destination en paquete: 10.0.1.50

Rutas disponibles:
  10.0.0.0/16  → local      (coincide: 16 bits)
  10.0.1.0/24  → igw        (coincide: 24 bits) ✓ GANADOR

→ El tráfico va a IGW porque /24 es más específico que /16
```

### Prioridad: Local vs Específico

```
Paquete a: 10.0.2.100

Rutas:
  10.0.0.0/16  → local   (coincide)
  
→ Siempre usa /16 porque es la única que coincide
→ No hay ambigüedad porque "local" solo existe para el CIDR de la VPC
```

## Route Propagation (Propagación de Rutas)

Cuando usas **AWS Site-to-Site VPN** o **Direct Connect**, las rutas de la red on-premise pueden propagarse automáticamente.

```
┌─────────────────────────────────────────────────────────┐
│                    Tu Red On-Premise                     │
│                   192.168.1.0/24                        │
└─────────────────────┬───────────────────────────────────┘
                      │ VPN/DX
                      ▼
┌─────────────────────────────────────────────────────────┐
│                    VPC (10.0.0.0/16)                    │
│                                                         │
│  Route Table con propagation enabled:                    │
│  ─────────────────────────────────────────────────────  │
│  10.0.0.0/16    → local                                 │
│  192.168.1.0/24 → vpn-xxxx (propagated)                 │
└─────────────────────────────────────────────────────────┘
```

**Habilitar propagation:**
1. Ve a VPC > Route Tables
2. Selecciona la RT
3. Actions > Edit route propagation
4. Enable

## Comandos AWS CLI

```bash
# Listar todas las Route Tables
aws ec2 describe-route-tables

# Ver una RT específica
aws ec2 describe-route-tables --route-table-ids rt-xxxx

# Ver qué subnet está asociada a qué RT
aws ec2 describe-route-tables --query 'RouteTables[*].{
  RT: RouteTableId,
  Subnets: Associations[*].SubnetId,
  Routes: Routes[*].{Dest:DestinationCidrBlock,Target:GatewayId}
}'

# Crear ruta hacia IGW
aws ec2 create-route --route-table-id rt-xxxx \
  --destination-cidr-block 0.0.0.0/0 \
  --gateway-id igw-xxxx

# Crear ruta hacia NAT Gateway
aws ec2 create-route --route-table-id rt-xxxx \
  --destination-cidr-block 0.0.0.0/0 \
  --nat-gateway-id nat-xxxx
```

## Troubleshooting Rápido

| Problema | Verificación |
|----------|--------------|
| Instancia no tiene Internet | ¿RT tiene `0.0.0.0/0` → IGW? |
| Instancia en privada no sale | ¿RT tiene `0.0.0.0/0` → NAT? |
| No conecta a otra VPC | ¿RT tiene ruta al Peering? |
| Tráfego no llega | ¿Security Group permite? ¿NACL permite? |

```bash
# Verificar que una subnet tiene RT asociada
aws ec2 describe-route-tables \
  --filters "Name=association.subnet-id,Values=subnet-xxxxx"

# Verificar attributes de la VPC (para IGW routing)
aws ec2 describe-vpcs --vpc-ids vpc-xxxx --query 'Vpcs[0]'
```
