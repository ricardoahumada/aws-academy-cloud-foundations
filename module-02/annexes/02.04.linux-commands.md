## Comandos Básicos de Navegación

```bash
# Dónde estás
pwd

# Listar archivos
ls                    # Simple
ls -la                # Detallado (incluye ocultos)
ls -lh                # Tamaño legible (human readable)
ls -lt                # Por fecha de modificación

# Cambiar directorio
cd /directorio        # Ir a directorio absoluto
cd ./subdir           # Ir a subdirectorio relativo
cd ..                 # Ir al directorio padre
cd ~                 # Ir a tu home
cd -                 # Ir al directorio anterior

#路径 absoluta vs relativa
#  /home/ec2-user/app    → absoluta (comienza con /)
#  ./config/app.conf     → relativa (comienza con . o ..)
```

## Gestión de Archivos y Directorios

```bash
# Crear archivos y directorios
touch archivo.txt          # Crear archivo vacío
mkdir mi-carpeta           # Crear directorio
mkdir -p dir1/dir2/dir3    # Crear directorios anidados

# Copiar
cp archivo.txt respaldo.txt        # Copiar archivo
cp archivo.txt /tmp/              # Copiar a otro directorio
cp -r carpeta/ backup/            # Copiar directorio recursivamente

# Mover/Renombrar
mv archivo.txt nuevo-nombre.txt   # Renombrar
mv archivo.txt /tmp/               # Mover a otro directorio

# Eliminar
rm archivo.txt             # Eliminar archivo
rm -rf carpeta/            # Eliminar directorio y todo su contenido
```

## Permisos de Archivos

```bash
# Ver permisos (ls -l muestra:)
# -rw-r--r--  1 ec2-user ec2-user  1234 Apr 15 10:30 archivo.txt
# │└┬┘│└┬┘│└┬┘│ └┬┘
# │ │  │   │   │   └── Grupo
# │ │  │   │   └─────── Otros (rwx)
# │ │  │   └────────── Owner (rwx)
# │ │  └────────────── Grupo (rwx)
# │ └───────────────── Owner
# └───────────────────── Tipo (-=archivo, d=directorio)

# Tipos de permisos
r (read)    → 4  → Leer
w (write)   → 2  → Escribir/Modificar
x (execute) → 1  → Ejecutar/Entrar al directorio

# Cambiar permisos
chmod 755 archivo         # rwxr-xr-x (dueño=todo, grupo=rx, otros=rx)
chmod 644 archivo         # rw-r--r-- (dueño=rw, grupo=r, otros=r)
chmod +x script.sh        # Agregar execute al archivo
chmod -x script.sh         # Quitar execute

# Cambiar dueño
sudo chown ec2-user:ec2-user archivo.txt
sudo chown -R ec2-user:ec2-user carpeta/
```

## Visualizar y Editar Archivos

```bash
# Visualizar archivos
cat archivo.txt           # Ver todo el contenido
less archivo.txt          # Ver con paginación (q=salir, /buscar)
head -20 archivo.txt      # Ver primeras 20 líneas
tail -20 archivo.txt      # Ver últimas 20 líneas
tail -f /var/log/syslog   # Ver logs en tiempo real

# Buscar dentro de archivos
grep "texto" archivo.txt          # Buscar línea con "texto"
grep -i "texto" archivo.txt      # Case insensitive
grep -r "texto" carpeta/         # Buscar recursivamente
grep -n "texto" archivo.txt      # Mostrar número de línea

# Editores
nano archivo.txt          # Editor simple (ctrl+x para salir)
vim archivo.txt            # Editor avanzado (más potente)
```

## Gestión de Procesos y Servicios

```bash
# Ver procesos
ps aux                    # Todos los procesos
ps aux | grep httpd       # Buscar proceso específico
top                       # Monitor de procesos (q=salir)
htop                      # Versión mejorada de top

# Gestionar servicios (systemd)
sudo systemctl status httpd        # Ver estado
sudo systemctl start httpd         # Iniciar
sudo systemctl stop httpd          # Detener
sudo systemctl restart httpd        # Reiniciar
sudo systemctl enable httpd        # Iniciar al boot
sudo systemctl disable httpd       # No iniciar al boot

# Gestionar servicios (SysV init)
sudo service httpd status
sudo service httpd start
sudo service httpd stop
sudo service httpd restart

# Comandos antiguos (Amazon Linux 2 usa systemd)
sudo /etc/init.d/httpd status
```

## Instalación de Paquetes

### Amazon Linux 2 / RHEL / CentOS

```bash
# Actualizar sistema
sudo yum update -y

# Instalar paquete
sudo yum install -y httpd nginx python3 mysql-server

# Verificar instalación
which httpd          # Debería mostrar ruta
httpd -v             # Versión

# Eliminar paquete
sudo yum remove -y httpd
```

### Ubuntu / Debian

```bash
# Actualizar sistema
sudo apt update
sudo apt upgrade -y

# Instalar paquete
sudo apt install -y apache2 nginx mysql-server

# Eliminar paquete
sudo apt remove -y apache2
```

## Gestión de Red

```bash
# Ver información IP
ip addr                # Ver direcciones IP
ip route               # Ver tabla de rutas
ifconfig               # Versión legacy

# Probar conectividad
ping 8.8.8.8           # Ping a Google DNS
ping google.com        # Ping por nombre
curl https://google.com # Hacer request HTTP

# Ver puertos escuchando
sudo netstat -tlnp      # Todos los puertos TCP
sudo ss -tlnp          # Versión moderna
sudo lsof -i -P        # Ver archivos abiertos de red

# Puerto específico
sudo netstat -tlnp | grep :80
sudo ss -tlnp | grep :22
```

## Comandos para EC2 User Data (Scripting)

```bash
#!/bin/bash
# Esta sección muestra los comandos más comunes en User Data

# Actualizar sistema
yum update -y

# Instalar Apache
yum install -y httpd

# Iniciar y habilitar servicio
systemctl start httpd
systemctl enable httpd

# Crear contenido web
echo "<h1>Hola desde EC2</h1>" > /var/www/html/index.html

# Cambiar permisos
chmod 644 /var/www/html/index.html

# Ver logs de Apache
tail -f /var/log/httpd/access_log
tail -f /var/log/httpd/error_log

# Reiniciar Apache después de cambios
systemctl restart httpd
```

## Troubleshooting Rápido

| Problema | Comando |
|----------|---------|
| Archivo no encontrado | `ls -la /ruta/` |
| Permiso denegado | `ls -la /ruta/archivo` + `chmod` |
| Servicio no inicia | `sudo systemctl status httpd` + `journalctl -xe` |
| Puerto no responde | `sudo netstat -tlnp \| grep :80` |
| No puedo hacer ping | `ping 8.8.8.8` (verificar SG/NACL) |
| User data no ejecutó | `cat /var/log/cloud-init.log` |
| Espacio bajo | `df -h` + `du -sh /*` |
