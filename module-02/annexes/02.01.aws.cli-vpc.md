# AWS CLI para VPC - Comandos Esenciales

> **Tiempo de lectura:** ~8 min | **Referencia:** [AWS CLI VPC Reference](https://docs.aws.amazon.com/cli/latest/reference/ec2/)

---

## Configuración Inicial

```bash
# Configurar credenciales default
aws configure

# Configurar región
aws configure set region us-east-1

# Verificar identidad actual (súper útil!)
aws sts get-caller-identity

# Ver configuración
aws configure list
```

## VPC

### Crear y Listar

```bash
# Crear VPC con CIDR /16
aws ec2 create-vpc \
  --cidr-block 10.0.0.0/16 \
  --tag-specifications ResourceType=vpc,Tags=[{Key=Name,Value=mi-vpc}]

# Listar VPCs
aws ec2 describe-vpcs

# Describe con query específica
aws ec2 describe-vpcs --query 'Vpcs[*].{ID:VpcId,CIDR:CidrBlock,Name:Tags[?Key==Name]|[0].Value}'
```

### Modificar VPC

```bash
# Habilitar DNS hostname en VPC
aws ec2 modify-vpc-attribute \
  --vpc-id vpc-xxxxx \
  --enable-dns-hostnames "{\"Value\": true}"

# Habilitar DNS support
aws ec2 modify-vpc-attribute \
  --vpc-id vpc-xxxxx \
  --enable-dns-support "{\"Value\": true}"
```

## Subnets

### Crear Subnet

```bash
# Crear subnet pública
aws ec2 create-subnet \
  --vpc-id vpc-xxxxx \
  --cidr-block 10.0.1.0/24 \
  --availability-zone us-east-1a \
  --tag-specifications ResourceType=subnet,Tags=[{Key=Name,Value=mi-subnet-publica}]

# Crear subnet privada
aws ec2 create-subnet \
  --vpc-id vpc-xxxxx \
  --cidr-block 10.0.2.0/24 \
  --availability-zone us-east-1a \
  --tag-specifications ResourceType=subnet,Tags=[{Key=Name,Value=mi-subnet-privada}]

# Habilitar auto-assign public IP (subnet pública)
aws ec2 modify-subnet-attribute \
  --subnet-id subnet-xxxxx \
  --map-public-ip-on-launch
```

### Listar y Describir

```bash
# Listar todas las subnets
aws ec2 describe-subnets

# Listar subnets de una VPC
aws ec2 describe-subnets --filters Name=vpc-id,Values=vpc-xxxxx

# Filtrar por tag
aws ec2 describe-subnets --filters Name=tag:Name,Values=*publica*
```

## Internet Gateway

### Crear y Adjuntar

```bash
# Crear Internet Gateway
aws ec2 create-internet-gateway \
  --tag-specifications ResourceType=internet-gateway,Tags=[{Key=Name,Value=mi-igw}]

# Adjuntar a VPC
aws ec2 attach-internet-gateway \
  --internet-gateway-id igw-xxxxx \
  --vpc-id vpc-xxxxx

# Listar IGWs
aws ec2 describe-internet-gateways

# Detach IGW (para eliminar)
aws ec2 detach-internet-gateway \
  --internet-gateway-id igw-xxxxx \
  --vpc-id vpc-xxxxx
```

## NAT Gateway

### Crear

```bash
# Crear NAT Gateway en subnet pública
aws ec2 create-nat-gateway \
  --subnet-id subnet-publica \
  --allocation-id eip-xxxxx \
  --tag-specifications ResourceType=natgateway,Tags=[{Key=Name,Value=mi-nat}]

# La respuesta incluye NatGatewayId
# Esperar hasta que estado sea "available"
aws ec2 describe-nat-gateways \
  --nat-gateway-ids nat-xxxxx

# Crear Elastic IP si no tienes
aws ec2 allocate-address --domain vpc
```

### Eliminar

```bash
# Liberar EIP
aws ec2 release-address --allocation-id eip-xxxxx

# Eliminar NAT Gateway
aws ec2 delete-nat-gateway --nat-gateway-id nat-xxxxx
```

## Route Tables

### Crear y Asociar

```bash
# Crear Route Table para VPC
aws ec2 create-route-table \
  --vpc-id vpc-xxxxx \
  --tag-specifications ResourceType=route-table,Tags=[{Key=Name,Value=rt-publica}]

# Crear ruta a IGW
aws ec2 create-route \
  --route-table-id rt-xxxxx \
  --destination-cidr-block 0.0.0.0/0 \
  --gateway-id igw-xxxxx

# Crear ruta a NAT Gateway
aws ec2 create-route \
  --route-table-id rt-xxxxx \
  --destination-cidr-block 0.0.0.0/0 \
  --nat-gateway-id nat-xxxxx

# Asociar subnet a Route Table
aws ec2 associate-route-table \
  --route-table-id rt-xxxxx \
  --subnet-id subnet-xxxxx

# Verificar asociación
aws ec2 describe-route-tables \
  --route-table-id rt-xxxxx \
  --query 'RouteTables[0].Associations'
```

### Rutas Locales

```bash
# La ruta local es automática y NO se puede modificar o eliminar
# Solo existe para el CIDR de la VPC

# Ver todas las rutas incluyendo local
aws ec2 describe-route-tables \
  --route-table-id rt-xxxxx \
  --query 'RouteTables[0].Routes'
```

## Security Groups

### Crear y Configurar

```bash
# Crear Security Group
aws ec2 create-security-group \
  --group-name mi-sg-bastion \
  --description "Security group para bastion host" \
  --vpc-id vpc-xxxxx

# Agregar regla inbound: SSH desde cualquier lugar
aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 22 \
  --cidr 0.0.0.0/0

# Agregar regla inbound: HTTP desde VPC
aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 80 \
  --cidr 10.0.0.0/16

# Agregar regla para permitir tráfico entre SGs
aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 0-65535 \
  --source-group sg-otronumero

# Revocar regla
aws ec2 revoke-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 22 \
  --cidr 0.0.0.0/0

# Listar reglas de un SG
aws ec2 describe-security-groups --group-ids sg-xxxxx
```

## Network ACLs (NACLs)

### Crear y Asociar

```bash
# Crear NACL
aws ec2 create-network-acl \
  --vpc-id vpc-xxxxx \
  --tag-specifications ResourceType=network-acl,Tags=[{Key=Name,Value=mi-nacl}]

# Crear regla inbound (Allow SSH)
aws ec2 create-network-acl-entry \
  --network-acl-id acl-xxxxx \
  --rule-number 100 \
  --protocol tcp \
  --rule-action allow \
  --egress false \
  --port-range From=22,To=22 \
  --cidr-block 10.0.0.0/16

# Crear regla outbound (Allow respuesta SSH)
aws ec2 create-network-acl-entry \
  --network-acl-id acl-xxxxx \
  --rule-number 100 \
  --protocol tcp \
  --rule-action allow \
  --egress true \
  --port-range From=1024,To=65535 \
  --cidr-block 10.0.0.0/16

# Asociar NACL a subnet
aws ec2 associate-subnet-network-acl \
  --subnet-id subnet-xxxxx \
  --network-acl-id acl-xxxxx

# Ver NACL associations
aws ec2 describe-network-acls --network-acl-ids acl-xxxxx
```

## Cleanup (Eliminar Recursos)

```bash
# NO ejecutar esto en producción sin confirmar!
# Es solo para limpiar labs/test

# Eliminar NAT Gateway (primero desasociar EIP)
aws ec2 delete-nat-gateway --nat-gateway-id nat-xxxxx

# Eliminar rutas primero
aws ec2 delete-route --route-table-id rt-xxxxx --destination-cidr-block 0.0.0.0/0

# Desasociar subnet de RT
aws ec2 disassociate-route-table --association-id rtbassoc-xxxxx

# Eliminar RT
aws ec2 delete-route-table --route-table-id rt-xxxxx

# Detach y eliminar IGW
aws ec2 detach-internet-gateway --internet-gateway-id igw-xxxxx --vpc-id vpc-xxxxx
aws ec2 delete-internet-gateway --internet-gateway-id igw-xxxxx

# Eliminar subnets
aws ec2 delete-subnet --subnet-id subnet-xxxxx

# Eliminar VPC
aws ec2 delete-vpc --vpc-id vpc-xxxxx
```
