# SSH y Conectividad - Guía Práctica

> **Tiempo de lectura:** ~7 min | **Referencia:** [AWS Connect to Linux Instance](https://docs.aws.amazon.com/AWSEC2/latest/UserGuide/AccessingInstancesLinux.html)

---

## Requisitos Previos

1. **Key Pair (.pem):** Descargado al crear el par de claves
2. **IP pública o endpoint:** De la instancia EC2
3. **Seguridad:** Security Group debe permitir SSH (puerto 22)

---

## Windows: OpenSSH (Recomendado)

### Paso 1: Verificar OpenSSH

```powershell
# En Windows Terminal o PowerShell
ssh -V

# Si no está instalado, instálalo desde:
# Settings > Apps > Optional features > OpenSSH Client
```

### Paso 2: Conectar

```powershell
# Usar IP pública
ssh -i "ruta\a\mi-keypair.pem" ec2-user@54.123.45.67

# Si usaste Amazon Linux 2
ssh -i "ruta\a\mi-keypair.pem" ec2-user@ec2-54-123-45-67.compute-1.amazonaws.com

# Si conectas desde Windows Terminal con WSL
ssh -i ~/path/to/key.pem ec2-user@54.123.45.67
```

### Solución de Problemas Windows

```powershell
# Error:Permissions are too open
icacls mi-keypair.pem /inheritance:r
icacls mi-keypair.pem /grant:r "%USERNAME%:(R)"

# Verificar permisos
dir mi-keypair.pem
# Deben mostrar SOLO tu usuario con acceso
```

## Windows: PuTTY

### Paso 1: Convertir .pem a .ppk

```
1. Abrir PuTTYgen
2. Load > seleccionar mi-keypair.pem
3. Save Private Key > mi-keypair.ppk
```

### Paso 2: Conectar con PuTTY

```
1. Host Name: ec2-user@54.123.45.67
2. Category > Connection > SSH > Auth
   - Private key file: Browse > seleccionar mi-keypair.ppk
3. Category > Session
   - Host Name: ec2-user@54.123.45.67
   - Saved Sessions: mi-bastion > Save
4. Open
```

### Solución de Problemas PuTTY

| Error | Causa | Solución |
|-------|-------|----------|
| `Server refused our key` | .ppk incorrecto | Regenerar .ppk con PuTTYgen |
| `Connection timed out` | SG no permite | Verificar regla inbound puerto 22 |
| `Connection refused` | Firewall Windows | Desactivar temporalmente Windows Firewall |

## Transferencia de Archivos

### SCP (Linux/macOS/Windows Terminal)

```bash
# Subir archivo a la instancia
scp -i mi-keypair.pem archivo.txt ec2-user@54.123.45.67:/home/ec2-user/

# Descargar archivo desde la instancia
scp -i mi-keypair.pem ec2-user@54.123.45.67:/path/remoto /path/local

# Subir directorio completo
scp -i mi-keypair.pem -r mi-directorio/ ec2-user@54.123.45.67:/home/ec2-user/
```

### SFTP

```bash
# Conectar via SFTP
sftp -i mi-keypair.pem ec2-user@54.123.45.67

# Comandos SFTP
sftp> put archivo.txt        # Subir
sftp> get archivo.txt        # Descargar
sftp> ls                     # Listar
sftp> exit                   # Salir
```

## Troubleshooting de Conectividad

### Error: Connection Timeout

```powershell
# 1. Verificar IP pública
aws ec2 describe-instances \
  --instance-ids i-xxxxx \
  --query 'Reservations[0].Instances[0].PublicIpAddress'

# 2. Verificar Security Group (desde consola o CLI)
aws ec2 describe-security-groups \
  --group-ids sg-xxxxx \
  --query 'SecurityGroups[0].IpPermissions'

# 3. Verificar que subnet tiene auto-assign public IP
aws ec2 describe-subnets \
  --subnet-ids subnet-xxxxx \
  --query 'Subnets[0].MapPublicIpOnLaunch'
```

### Error: Permission Denied (publickey)

```bash
# Verificar que el usuario es correcto
# Amazon Linux: ec2-user
# Ubuntu: ubuntu
# RHEL: ec2-user o root
# CentOS: centos

# Si usaste wrong username:
ssh -i mi-key.pem ubuntu@54.123.45.67   # Esto falla en Amazon Linux
ssh -i mi-key.pem ec2-user@54.123.45.67  # Esto funciona
```

### Verificar desde la Instancia

```bash
# En AWS Console, usar Session Manager o EC2 Instance Connect
# Si puedes acceder a la instancia:

# Ver logs de SSH
sudo journalctl -u sshd

# Ver configuración de SSHD
sudo cat /etc/ssh/sshd_config

# Ver usuarios válidos
cat /etc/passwd
```

## Security Groups: Configuración SSH

### Mínimo (desde cualquier lugar - lab/testing)

```json
{
  "IpProtocol": "tcp",
  "FromPort": 22,
  "ToPort": 22,
  "IpRanges": [{"CidrIp": "0.0.0.0/0"}]
}
```

### Recomendado (desde tu IP only)

```bash
# Obtener tu IP pública
curl ifconfig.me

# Crear regla solo con tu IP
aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 22 \
  --cidr TU_IP/32
```

### Seguridad Adicional

```bash
# Para bastion: crear SG específico que solo permita SSH desde tu IP
aws ec2 create-security-group \
  --group-name bastion-sg \
  --description "SSH solo desde mi IP" \
  --vpc-id vpc-xxxxx

aws ec2 authorize-security-group-ingress \
  --group-id sg-xxxxx \
  --protocol tcp \
  --port 22 \
  --cidr TU_IP/32
```
