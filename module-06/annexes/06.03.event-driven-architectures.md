## Conceptos Fundamentales

### ¿Qué es EventBridge?

```
┌──────────────┐     ┌──────────────┐     ┌──────────────┐
│   Event      │     │              │     │     Rule     │     ┌──────────────┐
│   Source     │────►│  EventBridge │────►│   (Filter)   │────►│    Target    │
│   (AWS/SaaS) │     │    Bus       │     │              │     │  (Lambda/SQS)│
└──────────────┘     └──────────────┘     └──────────────┘     └──────────────┘
                            │
                            │ Default Event Bus (aws.events)
                            │
                     ┌──────┴──────┐
                     │ Custom Bus  │
                     │ (Tu cuenta) │
                     └─────────────┘
```

### Componentes

| Componente | Descripción |
|------------|-------------|
| **Event** | JSON con datos de lo que ocurrió |
| **Event Bus** | Canal que recibe eventos |
| **Rule** | Filtro que matching events → targets |
| **Target** | Destino final (Lambda, SQS, SNS, etc.) |

## Tipos de Event Buses

### Default Event Bus

```bash
# Eventos de servicios AWS
aws events put-events \
    --entries '[{"Source": "my.app", "DetailType": "user.signup", "Detail": "{\"userId\": \"123\"}"}]'
```

### Custom Event Bus

```bash
# Crear custom event bus
aws events create-event-bus \
    --name my-event-bus \
    --description "My custom event bus"

# Enviar evento a custom bus
aws events put-events \
    --entries '[{"EventBusName": "my-event-bus", "Source": "my.app", "DetailType": "order.created", "Detail": "{\"orderId\": \"456\"}"}]'
```

### Partner Event Bus

Para SaaS partners (Datadog, Zendesk, etc.):

```bash
# Crear partner event bus (recibe de SaaS)
aws events create-event-bus \
    --name datadog-events \
    --event-source-name aws.partner/datadog.com/my-account
```

## Estructura de un Event

```json
{
    "id": "cdc18c94-2c5c-4d8d-9184-7cc88f8945a3",     // Unique ID
    "version": "0",                                    // Event schema version
    "account": "123456789012",                         // AWS Account ID
    "time": "2026-04-17T10:30:00Z",                    // Timestamp
    "region": "us-east-1",                             // Region
    "detail-type": "EC2 Instance State-change Notification",  // Event type
    "source": "aws.ec2",                               // Service source
    "resources": ["arn:aws:ec2:us-east-1:123456789:instance/i-123456"],  // Related resources
    "detail": {                                        // Event-specific data
        "instance-id": "i-123456",
        "state": "running"
    }
}
```

## Rules y Pattern Matching

### Event Pattern (EventBridge Rule)

```json
{
  "source": ["aws.ec2"],
  "detail-type": ["EC2 Instance State-change Notification"],
  "detail": {
    "state": ["running", "stopped"],
    "instance-id": [{ "prefix": "i-0" }]
  }
}
```

### Custom Pattern Examples

**Pattern: S3 Object Created**

```json
{
  "source": ["aws.s3"],
  "detail-type": ["Object Created"],
  "detail": {
    "bucket": {
      "name": ["my-app-bucket"]
    },
    "object": {
      "key": [{ "suffix": ".json" }]
    }
  }
}
```

**Pattern: DynamoDB Stream**

```json
{
  "source": ["aws.dynamodb"],
  "detail-type": ["DynamoDB Stream Record"],
  "detail": {
    "eventName": ["INSERT", "MODIFY"],
    "tableName": ["UsersTable"]
  }
}
```

## Targets

### Lambda como Target

```bash
# Crear regla con Lambda target
aws events put-rule \
    --name my-lambda-rule \
    --event-pattern '{"source": ["my.app"]}' \
    --description "Trigger Lambda on my events"

# Añadir Lambda como target
aws events put-targets \
    --rule my-lambda-rule \
    --targets '[{"Id": "1", "Arn": "arn:aws:lambda:us-east-1:123456789:function:my-function"}]'
```

### SQS como Target

```bash
# Crear regla con SQS target
aws events put-rule \
    --name my-sqs-rule \
    --event-pattern '{"source": ["my.app"]}'

aws events put-targets \
    --rule my-sqs-rule \
    --targets '[{"Id": "1", "Arn": "arn:aws:sqs:us-east-1:123456789:my-queue", "SqsParameters": {"MessageGroupId": "default"}}]'
```

### SNS como Target

```bash
aws events put-targets \
    --rule my-sns-rule \
    --targets '[{"Id": "1", "Arn": "arn:aws:sns:us-east-1:123456789:my-topic"}]'
```

## Lambda Handler para EventBridge

```python
import json

def lambda_handler(event, context):
    """
    EventBridge invoca Lambda con el evento completo en 'detail'
    """
    # Información del envelope
    version = event.get('version', '0')
    id = event['id']
    source = event['source']
    detail_type = event['detail-type']
    account = event['account']
    time = event['time']
    region = event['region']
    
    # El contenido del evento está en 'detail'
    detail = event.get('detail', {})
    
    print(f'Received event: {detail_type} from {source}')
    print(f'Region: {region}, Account: {account}')
    print(f'Event ID: {id}, Time: {time}')
    
    # Procesar según el tipo de evento
    if source == 'aws.ec2':
        instance_id = detail.get('instance-id')
        state = detail.get('state')
        print(f'EC2 Instance {instance_id} is {state}')
        
    elif source == 'aws.dynamodb':
        table_name = detail.get('tableName')
        event_name = detail.get('eventName')
        print(f'DynamoDB: {event_name} on {table_name}')
        
    elif source == 'my.app':
        # Custom event from your application
        custom_data = detail.get('customField')
        print(f'Custom event: {custom_data}')
    
    return {
        'statusCode': 200,
        'body': json.dumps({'processed': True})
    }
```

## EventBridge Scheduler (EventBridge Scheduler)

### Crear Schedule (Cron)

```bash
# Crear regla programada (cada hora)
aws events put-rule \
    --name daily-cleanup \
    --schedule-expression "cron(0 12 * * ? *)" \
    --description "Run cleanup at noon daily"

# Target: Lambda
aws events put-targets \
    --rule daily-cleanup \
    --targets '[{"Id": "1", "Arn": "arn:aws:lambda:us-east-1:123456789:function:cleanup-function"}]'
```

### Rate Expressions

```bash
# Cada 5 minutos
--schedule-expression "rate(5 minutes)"

# Cada hora
--schedule-expression "rate(1 hour)"

# Cada día
--schedule-expression "rate(1 day)"
```

### Cron Expressions

```bash
# Lunes a viernes a las 9am UTC
--schedule-expression "cron(0 9 ? * MON-FRI *)"

# Primer día del mes a medianoche
--schedule-expression "cron(0 0 1 * ? *)"

# Cada 15 minutos
--schedule-expression "cron(0/15 * * * ? *)"
```

## Dead Letter Queue (DLQ)

```bash
# Crear DLQ
aws sqs create-queue \
    --queue-name my-lambda-dlq

# Obtener ARN
aws sqs get-queue-attributes \
    --queue-url https://sqs.us-east-1.amazonaws.com/123456789/my-lambda-dlq \
    --attribute-names QueueArn

# Actualizar Lambda con DLQ
aws lambda update-function-configuration \
    --function-name my-function \
    --dead-letter-config TargetArn=arn:aws:sqs:us-east-1:123456789:my-lambda-dlq
```

### Retry Configuration

```bash
# Configurar retry en EventBridge
aws events put-rule \
    --name my-rule \
    --event-pattern '{"source": ["my.app"]}' \
    --retry-policy MaximumRetryAttempts=3,MinimumRetryDelay=60
```

## EventBridge vs SNS vs SQS

| Característica | EventBridge | SNS | SQS |
|----------------|-------------|-----|-----|
| **Modelo** | Event bus con filtering | Pub/Sub | Message queue |
| **Filtering** | Yes (Rules) | No | No |
| **Scheduling** | Yes (Scheduler) | No | No |
| **Partner SaaS** | Yes | No | No |
| **Delivery** | Fire-and-forget | At-least-once | At-least-once |
| **Ordering** | Per event source | Per topic | Per queue |
| **DLQ** | Yes | Yes (optional) | Built-in |
| **Cost** | Per event + rules | Per message | Per message |

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| Lambda no se invoca | Rule no existe | Verificar `aws events list-rules` |
| Event dropped | DLQ no configurado | Agregar DLQ con `dead-letter-config` |
| Too many invocations | Pattern muy amplio | Refinar event pattern |
| Events not matching | Pattern incorrecto | Usar `test-event-pattern` |
| Schedule not firing | Timezone UTC | Verificar `cron` en UTC |
