## Conceptos Fundamentales

### Estructura de una Lambda Function

```
┌─────────────────────────────────────────────────────────────┐
│                    Lambda Function                          │
│                                                             │
│  Handler: lambda_handler(event, context)                   │
│                                                             │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Event (JSON input)                                 │   │
│  │  - API Gateway payload                             │   │
│  │  - S3 object created                                │   │
│  │  - SNS message                                     │   │
│  │  - Scheduled trigger                               │   │
│  └─────────────────────────────────────────────────────┘   │
│                           │                                 │
│                           ▼                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Tu Código                                           │   │
│  │  - Procesa el event                                  │   │
│  │  - Business logic                                    │   │
│  │  - Llamadas a otros servicios AWS                     │   │
│  └─────────────────────────────────────────────────────┘   │
│                           │                                 │
│                           ▼                                 │
│  ┌─────────────────────────────────────────────────────┐   │
│  │  Response (JSON output)                              │   │
│  │  - statusCode: 200                                   │   │
│  │  - body: "..."                                      │   │
│  │  - headers: {...}                                   │   │
│  └─────────────────────────────────────────────────────┘   │
│                                                             │
│  Context object: metadata de la ejecución                  │
│  - context.function_name                                    │
│  - context.request_id                                      │
│  - context.remaining_time                                   │
└─────────────────────────────────────────────────────────────┘
```

### El Objeto Context

El objeto `context` proporciona información sobre la ejecución actual:

```python
import json

def lambda_handler(event, context):
    # Información del context
    function_name = context.function_name
    request_id = context.request_id
    remaining_time = context.get_remaining_time_in_millis()
    invoked_function_arn = context.invoked_function_arn
    log_group_name = context.log_group_name
    log_stream_name = context.log_stream_name
    memory_limit = context.memory_limit_in_mb
    
    # Calcular tiempo restante
    if remaining_time < 5000:  # 5 segundos
        return {
            'statusCode': 500,
            'body': json.dumps('Insufficient execution time')
        }
    
    # Tu lógica de negocio
    return {
        'statusCode': 200,
        'body': json.dumps(f'Processed by {function_name}')
    }
```

## Patrones de Handlers

### Proxy Integration (API Gateway)

```python
def lambda_handler(event, context):
    """
    Formato de event con API Gateway Proxy Integration
    """
    method = event['httpMethod']
    path = event['path']
    headers = event.get('headers', {})
    query_params = event.get('queryStringParameters', {})
    body = event.get('body', '')
    
    if method == 'GET' and path == '/users':
        return {
            'statusCode': 200,
            'body': json.dumps({'users': []}),
            'headers': {'Content-Type': 'application/json'}
        }
    
    if method == 'POST' and path == '/users':
        data = json.loads(body)
        return {
            'statusCode': 201,
            'body': json.dumps({'created': data}),
            'headers': {'Content-Type': 'application/json'}
        }
    
    return {
        'statusCode': 400,
        'body': json.dumps({'error': 'Not found'}),
        'headers': {'Content-Type': 'application/json'}
    }
```

### S3 Event Handler

```python
import json
import boto3

s3 = boto3.client('s3')

def lambda_handler(event, context):
    """
    Event from S3 bucket (object created)
    """
    # Extraer bucket y key del event
    bucket = event['Records'][0]['s3']['bucket']['name']
    key = event['Records'][0]['s3']['object']['key']
    
    # Decode URL-encoded characters
    import urllib.parse
    key = urllib.parse.unquote_plus(key)
    
    print(f'Processing file: {key} from bucket: {bucket}')
    
    # Get object metadata
    response = s3.head_object(Bucket=bucket, Key=key)
    content_type = response.get('ContentType', 'application/octet-stream')
    size = response.get('ContentLength', 0)
    
    return {
        'statusCode': 200,
        'body': json.dumps({
            'bucket': bucket,
            'key': key,
            'size': size,
            'content_type': content_type
        })
    }
```

## Logging y Debugging

```python
import json
import logging

# Configurar logging
logger = logging.getLogger()
logger.setLevel(logging.INFO)

def lambda_handler(event, context):
    # Logging estructurado
    logger.info(f'Event received: {json.dumps(event)}')
    logger.info(f'Request ID: {context.request_id}')
    logger.info(f'Function memory: {context.memory_limit_in_mb}MB')
    
    try:
        # Tu lógica
        result = process_event(event)
        logger.info(f'Result: {result}')
        return result
    except Exception as e:
        logger.error(f'Error processing event: {str(e)}')
        raise
```

## Troubleshooting

| Error | Causa | Solución |
|-------|-------|----------|
| `Runtime.ExitError` | Handler no retorna respuesta | Verificar return statement |
| `Process exited before completing request` | Timeout | Aumentar timeout o optimizar código |
| `Cold start too slow` | Runtime pesado (Java/C#) | Usar Python/Node.js, Provisioned Concurrency |
| `ImportError` | Dependencias faltantes | Crear Lambda Layer o bundle en ZIP |
| `AccessDenied` | Permisos IAM | Agregar role con policy necesaria |

## Comandos AWS CLI

```bash
# Crear función Lambda
aws lambda create-function \
    --function-name my-function \
    --runtime python3.11 \
    --role arn:aws:iam::123456789:role/lambda-role \
    --handler lambda_handler.lambda_handler \
    --zip-file fileb://function.zip \
    --timeout 30 \
    --memory-size 256

# Invocar función (test)
aws lambda invoke \
    --function-name my-function \
    --payload '{"name": "World"}' \
    --log-type Tail \
    response.json

# Ver logs
aws logs tail /aws/lambda/my-function --follow

# Actualizar código
aws lambda update-function-code \
    --function-name my-function \
    --zip-file fileb://updated_function.zip

# Configurar variables de entorno
aws lambda update-function-configuration \
    --function-name my-function \
    --environment Variables="{DB_HOST=prod.db.com,LOG_LEVEL=DEBUG}"

# Añadir Layer
aws lambda update-function-configuration \
    --function-name my-function \
    --layers arn:aws:lambda:us-east-1:123456789:layer:my-layer:1
```
