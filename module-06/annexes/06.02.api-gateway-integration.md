## Lambda Proxy Integration

### Características
- Lambda recibe el request completo del cliente
- Lambda retorna la respuesta directamente
- Configuración mínima
- Recomendado para la mayoría de casos

### Flujo

```
Cliente → API Gateway → Lambda Proxy → Lambda Function
    ↑                        ↓
    └──────── Response ←─────┘
```

### Event que Lambda recibe (Proxy)

```python
def lambda_handler(event, context):
    # Campos del event de API Gateway
    resource = event['resource']           # '/users/{id}'
    path = event['path']                   # '/users/123'
    http_method = event['httpMethod']       # 'GET'
    headers = event['headers']              # {...}
    query_params = event.get('queryStringParameters')  # None or {...}
    path_params = event.get('pathParameters')           # {'id': '123'}
    body = event.get('body')               # '{"name": "John"}' or None
    is_base64 = event.get('isBase64Encoded', False)     # False
    
    # Tu lógica
    return {
        'statusCode': 200,
        'body': json.dumps({'message': 'Hello'}),
        'headers': {'Content-Type': 'application/json'}
    }
```

### Response del Lambda (Proxy)

```python
{
    'statusCode': 200,           # Required
    'body': '...',                # Required (string)
    'headers': {                  # Optional
        'Content-Type': 'application/json',
        'Access-Control-Allow-Origin': '*'
    },
    'isBase64Encoded': False      # Optional
}
```

## Lambda Non-Proxy (Custom) Integration

### Cuándo usar
- Necesitas transformar request/response
- Validación custom antes de Lambda
- Mapeo de errores custom
- Integración con múltiples backends

### Flujo

```
Cliente → API Gateway → Mapping Template → Lambda
    ↑                           ↓
    │                    (transform request)
    │                           ↓
    └─────────── Mapping Template ←
                  (transform response)
```

### Configuración en Console

1. Method > Integration Request
2. Integration type: Lambda Function
3. **Desmarcar** "Use Lambda Proxy Integration"
4. Configurar Mapping Templates

### Mapping Template (Request)

```json
{
    "method": "$input.params('httpMethod')",
    "body": "$input.json('$.body')",
    "userId": "$input.params('id')",
    "timestamp": "$input.json('$.timestamp')"
}
```

### Mapping Template (Response)

```json
{
    "statusCode": 200,
    "message": "$input.path('$.body.message')",
    "data": "$input.json('$.body.data')"
}
```

## HTTP Integration

### HTTP Proxy

```bash
# Configurar en API Gateway
aws apigatewayv2 create-integration \
    --api-id api-id \
    --integration-type HTTP_PROXY \
    --integration-uri https://api.example.com/users \
    --integration-method ANY \
    --payload-format 1.0
```

### HTTP Custom Integration

Útil para agregar autenticación o transformación:

```
Cliente → API Gateway → (add auth header) → Backend HTTP
```

## AWS Service Proxy (Sin Lambda)

### DynamoDB Proxy

Permite acceso directo a DynamoDB sin Lambda:

```bash
# Configuración en API Gateway Console:
# Integration type: AWS Service
# AWS Region: us-east-1
# AWS Service: DynamoDB
# HTTP Method: POST
# Action: PutItem
```

**Mapping Template para PutItem:**

```json
{
    "TableName": "UsersTable",
    "Item": {
        "userId": {"S": "$input.params('userId')"},
        "name": {"S": "$input.json('$.name')"},
        "email": {"S": "$input.json('$.email')"}
    }
}
```

**Response:**

```json
#if ($input.path('$.HttpStatusCode') == 200)
    #set($context.responseOverride.status = 201)
{
    "userId": "$input.path('$.Item.userId.S')",
    "status": "created"
}
#else
{
    "error": "$input.path('$.Error.Message')"
}
#end
```

### S3 Proxy

# GET Object from S3

# AWS Service: S3

## API Gateway REST vs HTTP API

| Característica | REST API | HTTP API |
|----------------|----------|----------|
| **Proxy Integration** | Lambda proxy + custom | Lambda proxy only |
| **WebSocket** | Yes | No (usa API Gateway WebSocket APIs) |
| **Swagger/OpenAPI** | Import/Export | Import only |
| **CORS** | Manual | Automatic |
| **Latencia** | Higher | Lower (~10ms vs ~50ms) |
| **Costos** | Higher | ~70% cheaper |
| **Autenticación** | IAM, Lambda, Cognito | JWT, Cognito, IAM |

## CORS Configuration

### Respuesta de OPTIONS (Preflight)

```python
def lambda_handler(event, context):
    if event['httpMethod'] == 'OPTIONS':
        return {
            'statusCode': 200,
            'headers': {
                'Access-Control-Allow-Origin': '*',
                'Access-Control-Allow-Headers': 'Content-Type,X-Amz-Date,Authorization,X-Api-Key',
                'Access-Control-Allow-Methods': 'GET,POST,PUT,DELETE,OPTIONS'
            },
            'body': ''
        }
    
    # Tu lógica normal
    return {
        'statusCode': 200,
        'headers': {
            'Access-Control-Allow-Origin': '*',
            'Content-Type': 'application/json'
        },
        'body': json.dumps({'message': 'Hello'})
    }
```

## Throttling y Usage Plans

```bash
# Configurar throttling por stage
aws apigateway update-stage \
    --rest-api-id api-id \
    --stage-name prod \
    --patch-ops op=replace,path=/throttling/burstLimit,value=100

# Crear Usage Plan
aws apigateway create-usage-plan \
    --name "Basic Plan" \
    --api-stages apiId=api-id,stage=prod \
    --quota settings=limit=1000,period=DAY

# Crear API Key
aws apigateway create-api-key \
    --name "user1-key" \
    --enabled

# Asociar a Usage Plan
aws apigateway create-usage-plan-key \
    --usage-plan-id plan-id \
    --key-id key-id \
    --key-type API_KEY
```

## Troubleshooting

| Error | Causa | Solución |
|-------|-------|----------|
| `403 Forbidden` | Missing API Key | Incluir header `X-API-Key` |
| `502 Bad Gateway` | Lambda timeout | Aumentar timeout o Lambda |
| `500 Internal Error` | Lambda error | Ver logs en CloudWatch |
| `403 CORS` | Missing headers | Configurar CORS en API Gateway |
| `404 Not Found` | Wrong stage | Verificar URL: `/prod/resource` |

## Deployment y Stages

```bash
# Deploy a stage
aws apigateway create-deployment \
    --rest-api-id api-id \
    --stage-name dev

# Actualizar (re-deploy)
aws apigateway create-deployment \
    --rest-api-id api-id \
    --stage-name prod \
    --description "Updated with new Lambda"

# URL de invocación
# REST API: https://{api-id}.execute-api.{region}.amazonaws.com/{stage}
# HTTP API: https://{api-id}.execute-api.{region}.amazonaws.com/{stage}
```
