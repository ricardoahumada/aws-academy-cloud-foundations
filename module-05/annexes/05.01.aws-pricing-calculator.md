# AWS Pricing Calculator - Guía de Uso

> **Tiempo de lectura:** ~10 min | **Referencia:** [AWS Pricing Calculator](https://calculator.aws/)

---

## Concepto Básico

AWS Pricing Calculator permite estimar el costo de arquitecturas en AWS antes de implementarlas. Es esencial para presupuestos y decisiones de compra.

**Casos de uso:**
- Estimar costo de arquitectura nueva
- Comparar escenarios (On-Demand vs RI vs SP)
- Planificar presupuestos de proyecto
- Validar estimación de migración

## Ejemplos Prácticos

### Ejemplo 1: Arquitectura Web Básica

**Arquitectura a estimar:**
- 2x EC2 t3.micro (desarrollo)
- 1x RDS db.t3.micro (Single-AZ)
- 10GB S3 Standard
- 100GB datos transferidos

**Pasos:**

```bash
# NO aplica - es GUI
# Solo para referencia de pasos
```

**Configuración paso a paso:**

```
1. Click "Add service" > "Amazon EC2"
   
   Configure:
   - Region: US East (N. Virginia)
   - Operating system: Linux
   - Instance type: t3.micro
   - Instance count: 2
   - Usage pattern: Hours per month = 730
   - Pricing type: On-Demand
   
   Resultado: ~$15.20/month

2. Click "Add service" > "Amazon RDS"
   
   Configure:
   - Region: US East (N. Virginia)
   - Engine: MySQL
   - Instance class: db.t3.micro
   - Deployment: Single-AZ
   - Storage: 20 GB
   - Usage: 730 hours/month
   
   Resultado: ~$28.00/month

3. Click "Add service" > "Amazon S3"
   
   Configure:
   - Region: US East (N. Virginia)
   - Storage class: Standard
   - Storage: 10 GB
   - PUT/COPY requests: 1,000
   - GET and other requests: 10,000
   
   Resultado: ~$0.30/month

4. Click "Add service" > "Data Transfer"
   
   Configure:
   - Outbound Data Transfer: 100 GB
   
   Resultado: ~$9.00/month

TOTAL ESTIMADO: ~$52.50/month
```

### Ejemplo 2: Comparar On-Demand vs Reserved

**Misma arquitectura, comparar opciones:**

```
1. Agregar EC2 igual que antes
   - On-Demand: $7.60/instance/month

2. Click "Add new feature" o modificar:
   - Pricing type: Reserved
   - Term: 1 Year
   - Payment: No Upfront
   
   Resultado: ~$4.50/instance/month (40% ahorro)
```

### Ejemplo 3: Website con Auto Scaling

```
EC2 Auto Scaling Group:
- Min: 2 instances
- Max: 10 instances  
- Average: 4 instances
- Instance: t3.medium Linux
- On-Demand: 20%, Spot: 80%

Configuración:
- Modify EC2 > Step 3: Select "Spot Instance"
- Fleet composition: 20% On-Demand, 80% Spot
- Fleet strike price: $0.015 (vs $0.042 On-Demand)

Resultado:
- On-Demand portion: 0.8 instance = $24.20/month
- Spot portion: 3.2 instances = $15.30/month
- Total: ~$39.50/month (vs $64.80 full On-Demand)
Ahorro: ~39%
```

## Comandos AWS CLI para Precios

### Obtener Lista de Precios EC2

```bash
# Precio On-Demand por tipo de instancia
aws ec2 describe-instance-type-offerings \
  --location-type availability-zone \
  --filters Name=location,Values=us-east-1a

# Precio específico (usar jq para filtrar)
aws ec2 describe-spot-price-history \
  --instance-types t3.micro \
  --product-descriptions "Linux/UNIX" \
  --start-time 2026-04-01T00:00:00Z \
  --end-time 2026-04-17T00:00:00Z
```

### Estimar con CLI (Básico)

```bash
# No existe comando directo para calculator
# Pero podemos ver pricing via:
aws pricing get-products \
  --service-code AmazonEC2 \
  --filters Type=TERM_MATCH,Field=instanceType,Value=t3.micro \
  --filters Type=TERM_MATCH,Field=location,Value="US East (N. Virginia)" \
  --filters Type=TERM_MATCH,Field=operatingSystem,Value=Linux \
  --region us-east-1
```

## Errores Comunes y Soluciones

| Error | Causa | Solución |
|-------|-------|----------|
| Precio muy bajo | Region incorrecta | Verificar region del calculator |
| No incluye transferencia | Olvidó Data Transfer | Agregar servicio explícitamente |
| RDS double-charged | Incluyó Multi-AZ extra | Single-AZ es más barato |
| Free Tier no considerado | Estimación mensual alta | 750h/mes es free tier |

---

## Tips para Estimaciones Precisas

1. **Incluir TODOS los servicios** - No solo EC2
2. **Data transfer es fácil de olvidar** - Siempre agregarlo
3. **Usar region correcta** - Precios varían por region
4. **Comparar Reserved vs On-Demand** - Siempre hacer la comparación
5. **Guardar estimaciones** - Crear cuenta para guardar
