# Tagging Strategies - Guía de Implementación

> **Tiempo de lectura:** ~10 min | **Referencia:** [AWS Tagging Strategies](https://docs.aws.amazon.com/general/latest/gr/aws_tagging.html)

---

## Concepto Básico

Los **tags** son pares clave-valor que se asignan a recursos AWS para:
- Organizar recursos
- Rastrear costos por departamento/proyecto
- Control de acceso (IAM policies)
- Automatización (Resource Groups)
- Seguridad y cumplimiento

**Ejemplo de tag:**
```
Key: Environment
Value: production
```

---

## Tags Estándar (Tags Recomendados)

AWS recomienda un conjunto de tags estándar:

| Tag Clave | Descripción | Valores ejemplo |
|-----------|-------------|-----------------|
| `Name` | Nombre descriptivo | `web-server-01`, `api-prod` |
| `Environment` | Ambiente | `development`, `staging`, `production` |
| `Department` | Departamento | `engineering`, `marketing`, `finance` |
| `Project` | Proyecto | `ecommerce`, `analytics`, `mobile-app` |
| `CostCenter` | Centro de costo | `CC-12345` |
| `Owner` | Responsable | `juan@empresa.com`, `team-infra` |
| `ManagedBy` | Quién administra | `terraform`, `cloudformation`, `manual` |

## Estrategias de Naming

### Convención de Nombres

```
# Formato recomendado
{Environment}-{Service}-{Project}-{Number}

Ejemplos:
- prod-web-ecommerce-01
- dev-api-analytics-03
- staging-db-reporting-01
```

### Mejores Prácticas

```bash
# Usar consistente casing
Environment: production  # NO: Environment: Production
                      # NO: environment: production

# Usar valores consistentes
Environment: production  # SI
Environment: prod        # NO (mix!)

# Tags cortos para valores frecuentes
Dept: eng               # Abreviaciones ok para teams conocidos
Project: analytics      # Nombres completos para projects

# No usar caracteres especiales
Name: Server_01         # NO: spaces, underscores inconsistentes
Name: server-01         # SI: hyphens son estándar AWS
```

## Implementación por Servicio

### EC2 Instances

```bash
# Al crear instancia vía CLI
aws ec2 run-instances \
  --image-id ami-xxxxx \
  --instance-type t3.micro \
  --tag-specifications \
    ResourceType=instance,Tags=[
      {Key=Name,Value=web-server-01},
      {Key=Environment,Value=production},
      {Key=Project,Value=ecommerce},
      {Key=Owner,Value=juan@empresa.com}
    ]
```

### S3 Buckets

```bash
# Tags para buckets
aws s3api put-bucket-tagging \
  --bucket mi-bucket \
  --tagging 'TagSet=[{Key=Environment,Value=production},{Key=Project,Value=analytics}]'
```

### RDS Databases

```bash
# Tags para RDS
aws rds add-tags-to-resource \
  --resource-name arn:aws:rds:us-east-1:123456789012:db:mi-rds \
  --tags Key=Environment,Value=production Key=BackupPolicy,Value=daily
```

### Múltiples Recursos

```bash
# Resource Groups > Tag Editor para bulk operations
# O CLI con script:

for resource in $(aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Environment,Values=untagged \
  --query ResourceTagMappingList[].ResourceARN --output text); do
  aws ec2 create-tags \
    --resources $resource \
    --tags Key=Environment,Value=production
done
```

## AWS Organizations - Tagging Policies

### Crear Tagging Policy

```json
{
  "tags": [
    {
      "key": "Environment",
      "values": ["development", "staging", "production"],
      "enforced": true
    },
    {
      "key": "Department",
      "values": ["*"],
      "enforced": false
    }
  ]
}
```

### Aplicar a OU o Cuenta

```
1. AWS Organizations > Policies > Tagging Policy
2. Create policy
3. Attach to:
   - Root (todas las cuentas)
   - Specific OU
   - Specific account
4. Policy options:
   - Enforce: Bloquea recursos sin tag
   - Warn: Solo alerta
   - Don't enforce: Solo sugieres
```

## Resource Groups

### Crear Resource Group por Tag

```bash
# Crear grupo para todos los recursos production
aws resourcegroups create-group \
  --name production-resources \
  --resource-query '{
    "Type": "TAG_FILTERS_1_0",
    "Query": "{\"TagFilters\":[{\"Key\":\"Environment\",\"Values\":[\"production\"]}]}"
  }'

# Listar recursos en grupo
aws resourcegroups list-group-resources \
  --group-name production-resources
```

### CLI para Recursos por Tag

```bash
# Obtener todos los recursos con cierto tag
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Environment,Values=production \
  --query ResourceTagMappingList[].{ARN:ResourceARN,Tags:Tags}
```

## Cost Allocation Tags

### Activar Tags en Billing

```
1. AWS Billing Dashboard > Cost Allocation Tags
2. Activar tags que quieres ver en billing:
   - environment
   - department
   - project
3. Wait 24h para que tags aparezcan
```

### Reportes con Tags

```

- Group by: department
- Filter: environment=production
```

## Troubleshooting

| Problema | Causa | Solución |
|----------|-------|----------|
| Tags no aparecen en Cost Explorer | No activados en Billing | Activar Cost Allocation Tags |
| Tags no persisten | Tag en wrong resource type | Usar `--tag-specifications` en create |
| Resource Group vacío | Tag key mismatch | Verificar exact spelling del tag key |
| Tag policy blocking | Enforced tag policy | Agregar tags requeridos antes de crear |

## Comandos Relacionados

```bash
# Listar todos los tags de un recurso
aws ec2 describe-tags \
  --filters Name=resource-id,Values=i-xxxxx

# Listar recursos por tag
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Project,Values=ecommerce

# Bulk tag conjq
aws resourcegroupstaggingapi get-resources \
  --tag-filters Key=Environment,Values=development \
  --query 'ResourceTagMappingList[].ResourceARN' \
  --output text | xargs -I {} aws ec2 create-tags \
    --resources {} \
    --tags Key=Owner,Value=team-dev
```
