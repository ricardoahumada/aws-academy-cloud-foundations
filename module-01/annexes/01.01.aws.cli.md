## Configuración Inicial

### Instalación y Configuración

```bash
# Configurar credenciales y región por defecto
aws configure

# Configurar con salida JSON (por defecto)
aws configure set default.output json

# Configurar región específica
aws configure set default.region us-east-1

# Verificar configuración
aws configure list

# Verificar identidad actual (¡comando muy útil!)
aws sts get-caller-identity
```

### Autenticación

```bash
# Configurar perfil named
aws configure --profile mi-perfil

# Usar perfil específico
aws s3 ls --profile mi-perfil

# Asumir un rol (genera credenciales temporales)
aws sts assume-role --role-arn arn:aws:iam::123456789012:role/MyRole --role-session-name MiSesion
```

## Amazon S3

> **Referencia**: [S3 Command Reference](https://docs.aws.amazon.com/cli/latest/reference/s3/)

### Operaciones con Buckets

```bash
# Listar todos los buckets
aws s3 ls

# Listar contenido de un bucket
aws s3 ls s3://mi-bucket/

# Crear bucket
aws s3 mb s3://mi-nuevo-bucket

# Eliminar bucket vacío
aws s3 rb s3://mi-bucket

# Eliminar bucket con contenido
aws s3 rb s3://mi-bucket --force
```

### Operaciones con Objetos

```bash
# Copiar archivo local a S3
aws s3 cp archivo.txt s3://mi-bucket/

# Copiar archivo de S3 a local
aws s3 cp s3://mi-bucket/archivo.txt ./

# Mover archivo
aws s3 mv s3://mi-bucket/archivo.txt s3://mi-bucket/carpeta/

# Eliminar objeto
aws s3 rm s3://mi-bucket/archivo.txt

# Sincronizar directorio local con bucket
aws s3 sync ./mi-carpeta s3://mi-bucket/carpeta/

# Descargar todo el bucket
aws s3 sync s3://mi-bucket/ ./descargas/
```

### Filtros con --exclude e --include

```bash
# Subir solo archivos .jpg y .png
aws s3 cp ./mi-carpeta s3://mi-bucket/ --recursive \
    --exclude "*" \
    --include "*.jpg" \
    --include "*.png"

# Excluir carpeta .git
aws s3 sync ./proyecto s3://mi-bucket/ --exclude ".git/*"
```

### Gestión de Políticas y Etiquetas

```bash
# Ver política del bucket
aws s3api get-bucket-policy --bucket mi-bucket

# Poner política (desde archivo JSON)
aws s3api put-bucket-policy --bucket mi-bucket --policy file://politica.json

# Versiones de objeto
aws s3api list-object-versions --bucket mi-bucket
```

## Amazon EC2

> **Referencia**: [EC2 Command Reference](https://docs.aws.amazon.com/cli/latest/reference/ec2/)

### Instancias

```bash
# Listar todas las instancias
aws ec2 describe-instances

# Listar instancias con filtros (estado running)
aws ec2 describe-instances --filters "Name=instance-state-name,Values=running"

# Listar instancias específicas (por ID)
aws ec2 describe-instances --instance-ids i-1234567890abcdef0

# Iniciar instancia
aws ec2 start-instances --instance-ids i-1234567890abcdef0

# Detener instancia
aws ec2 stop-instances --instance-ids i-1234567890abcdef0

# Reiniciar instancia
aws ec2 reboot-instances --instance-ids i-1234567890abcdef0

# Terminar instancia
aws ec2 terminate-instances --instance-ids i-1234567890abcdef0
```

### Llaves de Acceso (Key Pairs)

```bash
# Crear nuevo par de llaves y descargar clave privada
aws ec2 create-key-pair --key-name mi-llave --query 'KeyMaterial' --output text > mi-llave.pem

# Listar llaves
aws ec2 describe-key-pairs

# Eliminar llave
aws ec2 delete-key-pair --key-name mi-llave

# Importar llave pública
aws ec2 import-key-pair --key-name mi-llave --public-key-material fileb://~/.ssh/id_rsa.pub
```

### Security Groups

```bash
# Crear security group
aws ec2 create-security-group \
    --group-name mi-sg \
    --description "Mi security group" \
    --vpc-id vpc-12345678

# Listar security groups
aws ec2 describe-security-groups

# Agregar regla de entrada (HTTP)
aws ec2 authorize-security-group-ingress \
    --group-id sg-12345678 \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0

# Agregar regla de entrada (SSH desde IP específica)
aws ec2 authorize-security-group-ingress \
    --group-id sg-12345678 \
    --protocol tcp \
    --port 22 \
    --cidr 203.0.113.0/24

# Eliminar regla
aws ec2 revoke-security-group-ingress \
    --group-id sg-12345678 \
    --protocol tcp \
    --port 80 \
    --cidr 0.0.0.0/0
```

### AMIs (Amazon Machine Images)

```bash
# Listar AMIs públicas (Amazon Linux 2023)
aws ec2 describe-images \
    --owners amazon \
    --filters "Name=name,Values=al2023-ami-2023*" "Name=architecture,Values=x86_64"

# Crear AMI desde instancia
aws ec2 create-image \
    --instance-id i-1234567890abcdef0 \
    --name "mi-ami-$(date +%Y%m%d)"

# Deregistrar AMI
aws ec2 deregister-image --image-id ami-12345678
```

### Volumes y Snapshots

```bash
# Crear volumen
aws ec2 create-volume \
    --size 100 \
    --availability-zone us-east-1a \
    --volume-type gp3

# Adjuntar volumen a instancia
aws ec2 attach-volume \
    --volume-id vol-12345678 \
    --instance-id i-1234567890abcdef0 \
    --device /dev/sdf

# Crear snapshot
aws ec2 create-snapshot --volume-id vol-12345678 --description "Backup $(date +%Y-%m-%d)"

# Listar snapshots
aws ec2 describe-snapshots --owner-ids self

# Copiar snapshot a otra región
aws ec2 copy-snapshot \
    --source-region us-east-1 \
    --source-snapshot-id snap-12345678 \
    --destination-region us-west-2
```

### VPC (Redes Virtuales)

```bash
# Crear VPC
aws ec2 create-vpc --cidr-block 10.0.0.0/16

# Crear subred
aws ec2 create-subnet \
    --vpc-id vpc-12345678 \
    --cidr-block 10.0.1.0/24 \
    --availability-zone us-east-1a

# Crear Internet Gateway
aws ec2 create-internet-gateway

# Adjuntar IG a VPC
aws ec2 attach-internet-gateway --vpc-id vpc-12345678 --internet-gateway-id igw-12345678

# Crear Route Table
aws ec2 create-route-table --vpc-id vpc-12345678

# Crear ruta a Internet
aws ec2 create-route \
    --route-table-id rtb-12345678 \
    --destination-cidr-block 0.0.0.0/0 \
    --gateway-id igw-12345678
```

### Addresses Elásticas (EIPs)

```bash
# Asignar IP elástica
aws ec2 allocate-address

# Asociar IP a instancia
aws ec2 associate-address \
    --instance-id i-1234567890abcdef0 \
    --allocation-id eipalloc-12345678

# Liberar IP elástica
aws ec2 release-address --allocation-id eipalloc-12345678
```

## Amazon IAM

> **Referencia**: [IAM Command Reference](https://docs.aws.amazon.com/cli/latest/reference/iam/)

### Usuarios

```bash
# Listar usuarios
aws iam list-users

# Crear usuario
aws iam create-user --user-name mi-usuario

# Crear usuario con acceso programático
aws iam create-user --user-name mi-usuario
aws iam create-access-key --user-name mi-usuario

# Eliminar usuario (primero eliminar sus claves)
aws iam delete-access-key --user-name mi-usuario --access-key-id AKIAIOSFODNN7EXAMPLE
aws iam delete-user --user-name mi-usuario
```

### Grupos y Políticas

```bash
# Crear grupo
aws iam create-group --group-name mi-grupo

# Agregar usuario a grupo
aws iam add-user-to-group --user-name mi-usuario --group-name mi-grupo

# Crear política inline
aws iam put-group-policy \
    --group-name mi-grupo \
    --policy-name mi-politica \
    --policy-document file://politica.json

# Listar políticas adjuntas a grupo
aws iam list-attached-group-policies --group-name mi-grupo
```

### Roles

```bash
# Crear rol (desde archivo de confianza JSON)
aws iam create-role \
    --role-name mi-rol \
    --assume-role-policy-document file://trust-policy.json

# Adjuntar política a rol
aws iam attach-role-policy \
    --role-name mi-rol \
    --policy-arn arn:aws:iam::aws:policy/AmazonS3ReadOnlyAccess

# Listar roles
aws iam list-roles

# Crear instancia perfil para EC2
aws iam create-instance-profile --instance-profile-name mi-perfil
aws iam add-role-to-instance-profile --instance-profile-name mi-perfil --role-name mi-rol

# Eliminar rol
aws iam delete-role --role-name mi-rol
```

### Políticas (Policies)

```bash
# Crear política gestionada
aws iam create-policy \
    --policy-name mi-politica \
    --policy-document file://policy.json

# Listar políticas gestionadas
aws iam list-policies --scope Local

# Obtener política
aws iam get-policy --policy-arn arn:aws:iam::123456789012:policy/mi-politica

# Listar versiones de política
aws iam list-policy-versions --policy-arn arn:aws:iam::123456789012:policy/mi-politica
```

### Contraseñas y MFA

```bash
# Configurar política de contraseñas
aws iam update-account-password-policy \
    --minimum-password-length 12 \
    --require-symbols \
    --require-numbers \
    --require-uppercase-chars \
    --require-lowercase-chars

# Habilitar MFA virtual
aws iam create-virtual-mfa-device --virtual-mfa-device-name mi-mfa

# Habilitar MFA en usuario
aws iam enable-mfa-device \
    --user-name mi-usuario \
    --serial-number arn:aws:iam::123456789012:mfa/mi-usuario \
    --authentication-code1 123456 \
    --authentication-code2 789012
```

### Credenciales de Acceso

```bash
# Listar claves de acceso de usuario
aws iam list-access-keys

# Crear nueva clave
aws iam create-access-key --user-name mi-usuario

# Eliminar clave
aws iam delete-access-key --user-name mi-usuario --access-key-id AKIAIOSFODNN7EXAMPLE

# Actualizar estado de clave (Active/Inactive)
aws iam update-access-key --user-name mi-usuario --access-key-id AKIAIOSFODNN7EXAMPLE --status Inactive
```

## AWS STS

> **Referencia**: [STS Command Reference](https://docs.aws.amazon.com/cli/latest/reference/sts/)

### Identidad y Tokens

```bash
# Ver identidad actual (el comando más útil para verificar credenciales)
aws sts get-caller-identity

# Obtener token de sesión (para usuarios root o IAM)
aws sts get-session-token

# Asumir rol (genera credenciales temporales)
aws sts assume-role \
    --role-arn arn:aws:iam::123456789012:role/DeveloperRole \
    --role-session-name dev-session

# Asumir rol con MFA
aws sts assume-role \
    --role-arn arn:aws:iam::123456789012:role/DeveloperRole \
    --serial-number arn:aws:iam::123456789012:mfa/mi-usuario \
    --token-code 123456 \
    --duration-seconds 3600
```

## Amazon Lambda

> **Referencia**: [Lambda Command Reference](https://docs.aws.amazon.com/cli/latest/reference/lambda/)

### Funciones

```bash
# Listar funciones
aws lambda list-functions

# Obtener configuración de función
aws lambda get-function --function-name mi-funcion

# Crear función
aws lambda create-function \
    --function-name mi-funcion \
    --runtime python3.12 \
    --role arn:aws:iam::123456789012:role/lambda-role \
    --handler lambda_function.lambda_handler \
    --zip-file fileb://codigo.zip

# Invocar función
aws lambda invoke \
    --function-name mi-funcion \
    --payload '{"key": "value"}' \
    response.json

# Invocar función asíncrona
aws lambda invoke \
    --function-name mi-funcion \
    --invocation-type Event \
    --payload '{"key": "value"}' \
    /dev/null

# Actualizar código
aws lambda update-function-code \
    --function-name mi-funcion \
    --zip-file fileb://codigo.zip

# Actualizar configuración
aws lambda update-function-configuration \
    --function-name mi-funcion \
    --environmentVariables '{"ENV": "production"}'

# Eliminar función
aws lambda delete-function --function-name mi-funcion
```

### Versiones y Aliases

```bash
# Publicar versión
aws lambda publish-version --function-name mi-funcion

# Crear alias
aws lambda create-alias \
    --function-name mi-funcion \
    --name production \
    --function-version 1

# Listar alias
aws lambda list-aliases --function-name mi-funcion
```

### Capas (Layers)

```bash
# Listar capas disponibles
aws lambda list-layers

# Listar versiones de una capa
aws lambda list-layer-versions --layer-name mi-capa

# Agregar capa a función
aws lambda update-function-configuration \
    --function-name mi-funcion \
    --layers arn:aws:lambda:us-east-1:123456789012:layer:mi-capa:1
```

## Amazon CloudWatch

> **Referencia**: [CloudWatch Command Reference](https://docs.aws.amazon.com/cli/latest/reference/cloudwatch/)

### Métricas

```bash
# Listar métricas disponibles para un namespace
aws cloudwatch list-metrics --namespace AWS/EC2

# Listar métricas con filtro
aws cloudwatch list-metrics \
    --namespace AWS/EC2 \
    --metric-name CPUUtilization \
    --dimensions Name=InstanceId,Value=i-1234567890abcdef0

# Obtener estadísticas de métrica
aws cloudwatch get-metric-statistics \
    --namespace AWS/EC2 \
    --metric-name CPUUtilization \
    --start-time 2026-04-17T00:00:00Z \
    --end-time 2026-04-17T12:00:00Z \
    --period 3600 \
    --statistics Average,Maximum
```

### Alarmas

```bash
# Crear alarma
aws cloudwatch put-metric-alarm \
    --alarm-name alta-cpu \
    --alarm-description "Alarma cuando CPU > 80%" \
    --metric-name CPUUtilization \
    --namespace AWS/EC2 \
    --statistic Average \
    --period 300 \
    --threshold 80 \
    --comparison-operator GreaterThanThreshold \
    --dimensions Name=InstanceId,Value=i-1234567890abcdef0 \
    --evaluation-periods 2 \
    --alarm-actions arn:aws:sns:us-east-1:123456789012:mi-tema

# Listar alarmas
aws cloudwatch describe-alarms

# Habilitar acciones de alarma
aws cloudwatch enable-alarm-actions \
    --alarm-names alta-cpu

# Deshabilitar acciones de alarma
aws cloudwatch disable-alarm-actions \
    --alarm-names alta-cpu

# Eliminar alarma
aws cloudwatch delete-alarms --alarm-names alta-cpu
```

### Logs

```bash
# Listar grupos de logs
aws logs describe-log-groups

# Listar streams de un grupo
aws logs describe-log-streams --log-group-name /aws/lambda/mi-funcion

# Obtener eventos de log
aws logs get-log-events \
    --log-group-name /aws/lambda/mi-funcion \
    --log-stream-name 2026/04/17/[$LATEST]abc123

# Crear grupo de logs
aws logs create-log-group --log-group-name mi-log-group

# Eliminar grupo de logs
aws logs delete-log-group --log-group-name mi-log-group
```

### Dashboards

```bash
# Crear dashboard
aws cloudwatch put-dashboard \
    --dashboard-name mi-dashboard \
    --dashboard-body file://dashboard.json

# Listar dashboards
aws cloudwatch list-dashboards

# Eliminar dashboard
aws cloudwatch delete-dashboards --dashboard-names mi-dashboard
```

## Amazon VPC

### Endpoints de VPC

```bash
# Crear VPC endpoint (Gateway para S3)
aws ec2 create-vpc-endpoint \
    --vpc-id vpc-12345678 \
    --service-name com.amazonaws.us-east-1.s3 \
    --route-table-ids rtb-12345678

# Listar VPC endpoints
aws ec2 describe-vpc-endpoints

# Eliminar VPC endpoint
aws ec2 delete-vpc-endpoints --vpc-endpoint-ids vpce-12345678
```

### NAT Gateways

```bash
# Crear NAT Gateway
aws ec2 create-nat-gateway \
    --subnet-id subnet-12345678 \
    --allocation-id eipalloc-12345678

# Obtener NAT Gateway info
aws ec2 describe-nat-gateways

# Eliminar NAT Gateway
aws ec2 delete-nat-gateway --nat-gateway-id nat-12345678
```

## Opciones Globales

```bash
# Formato de salida (json, text, table)
aws s3 ls --output table

# Región
aws ec2 describe-instances --region us-west-2

# Perfil
aws ec2 describe-instances --profile mi-perfil

# Endpoint URL personalizado
aws s3 ls --endpoint-url https://s3.amazonaws.com

# Activar debug
aws ec2 describe-instances --debug

# Sin paginación
aws ec2 describe-instances --no-paginate

# Query con JMESPath
aws ec2 describe-instances --query 'Reservations[*].Instances[*].[InstanceId,State.Name]'
```
